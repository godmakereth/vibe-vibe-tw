# 4.2.3 調整樣式與佈局

> **本步目標**：用迭代對話優化頁面的外觀，讓它更符合你的審美

## 迭代調整的心態

第一版代碼很少是完美的，這很正常。

還記得第三章學的"迭代對話"嗎？現在就是用它的時候：

```
第一版 → 看看哪裏不滿意 → 告訴 AI → 拿到新版本 → 再看看 → ...
```

通常 2-3 輪調整就能達到滿意的效果。不要期望一次完美，也不要無限調整。

## 分場景的調整 Prompt 庫

下面是一些常見調整場景的 Prompt，你可以直接複製使用，也可以根據需要修改。

### 顏色調整類

**整體配色太暗：**
```
整體配色太暗了，請改成更清新明亮的風格。
建議：
- 背景用淺灰色（如 #f5f5f5）
- 主色調用亮藍色（如 #3b82f6）
- 按鈕使用漸變或更鮮豔的顏色
```

**想換一個主題色：**
```
請把主色調從藍色改成綠色系。
添加按鈕用深綠色（#059669），完成按鈕用淺綠色。
```

**按鈕顏色不夠醒目：**
```
"添加"按鈕的顏色太淡了，改成更醒目的藍色（#2563eb），
文字改成白色，鼠標懸停時顏色加深。
```

### 佈局調整類

**輸入框和按鈕的位置：**
```
把輸入框和添加按鈕放在同一行。
輸入框佔大部分寬度，按鈕在右邊。
兩者之間留 8px 間距。
```

**頁面整體位置：**
```
頁面內容整體往上移一些，距離頂部 30px 就夠了。
標題和輸入區域之間的間距減小到 16px。
```

**任務列表太擠：**
```
任務列表的每一項之間間距太小，看起來很擠。
請把每個任務項之間的間距增加到 12px，讓視覺更清爽。
```

### 間距調整類

**整體太緊湊：**
```
頁面整體看起來太擁擠。請增加以下間距：
- 頁面內邊距增加到 24px
- 標題下方間距增加到 24px
- 輸入區域和列表之間間距增加到 20px
```

**整體太鬆散：**
```
頁面元素之間間距太大，看起來很空。
請適當減小各部分之間的間距，讓頁面更緊湊。
```

### 尺寸調整類

**輸入框太窄：**
```
輸入框太窄了，打字不方便。
請把輸入框高度增加到 44px，內邊距增加到 12px。
```

**按鈕太小：**
```
按鈕太小，不容易點擊。
請把所有按鈕的最小高度設爲 36px，內邊距增加到 12px 16px。
```

**字體大小調整：**
```
標題字體再大一些（28px），任務內容用 16px，按鈕文字用 14px。
```

### 響應式適配類

**手機上顯示不正常：**
```
頁面在手機上顯示有問題，內容超出屏幕邊緣。
請確保：
- 在窄屏幕（寬度小於 400px）時，頁面寬度自適應
- 按鈕在窄屏幕時可以換行顯示
- 字體大小在手機上不要太小
```

### 風格參考類

**參考特定產品風格：**
```
請把整體風格改成類似蘋果備忘錄的感覺：
- 純白背景
- 圓角更大一些（12px）
- 陰影更柔和
- 字體使用系統默認字體
```

**更現代的感覺：**
```
風格再現代一些：
- 添加輕微的卡片陰影
- 使用圓角（8px）
- 鼠標懸停時有輕微的顏色變化
- 整體使用無襯線字體
```

## 小李的調整過程示例

來看小李如何用迭代對話優化他的待辦清單頁面。

### 第一輪：整體感覺

**小李看到第一版後的想法**：結構對了，但顏色太普通，按鈕太小。

**小李發送**：
```
頁面結構不錯，請做以下調整：
1. 整體配色改成藍綠色系，更清新
2. 添加按鈕改成圓角，顏色更鮮豔
3. 按鈕高度增加，更容易點擊
```

### 第二輪：細節調整

**小李看到新版後的想法**：好多了，但輸入框和列表的間距還可以優化。

**小李發送**：
```
顏色好多了！再調整兩個細節：
1. 輸入框和任務列表之間的間距增加到 20px
2. 每個任務項加一條淺灰色的分隔線
```

### 第三輪：收尾

**小李發送**：
```
整體很好了。最後一個小調整：
- 任務右側的兩個按鈕之間留一點間距（8px）
- "完成"按鈕用綠色，"刪除"按鈕用紅色，顏色對比更明顯
```

經過三輪調整，小李得到了一個滿意的頁面。

## 常見樣式問題速查表

| 問題 | 快速調整 Prompt |
|-----|----------------|
| 輸入框太窄 | 輸入框寬度改成 100%，減去按鈕的寬度 |
| 按鈕太小 | 所有按鈕最小高度 36px，內邊距 12px 16px |
| 列表間距太小 | 任務項之間增加 12px 的間距 |
| 顏色太暗 | 整體配色改成明亮清新的風格 |
| 沒有視覺層次 | 給卡片添加輕微陰影，標題加粗加大 |
| 手機上顯示錯亂 | 添加響應式樣式，寬度使用百分比 |
| 鼠標懸停沒反饋 | 按鈕添加 hover 效果，顏色變深或加陰影 |
| 輸入框太靠邊 | 增加頁面內邊距到 24px |

## "夠好"的標準

什麼時候應該停止調整，進入下一步？用這個清單判斷：

### 必須滿足

- [ ] 所有元素都在正確的位置
- [ ] 可以清楚地看到標題、輸入框、按鈕、任務列表
- [ ] 頁面在手機寬度（375px）下不會變形
- [ ] 沒有明顯的視覺問題（文字溢出、元素重疊）

### 加分項（有更好，沒有也行）

- [ ] 配色符合你的審美
- [ ] 鼠標懸停有交互反饋
- [ ] 整體風格統一
- [ ] 間距看起來舒適

### 不需要追求的

- [ ] 完美的像素級對齊
- [ ] 動畫效果
- [ ] 複雜的視覺效果

::: warning 重要
80 分就夠了，先跑通再優化。如果你已經調整了 3-4 輪，差不多就可以停了。

記住，我們的目標是做出一個**能用的**待辦清單，不是參加設計比賽。
:::

## 調整建議

1. **一次只調整一類問題**：顏色、佈局、間距分開調整，這樣更容易控制
2. **具體描述**：不要說"好看點"，要說"按鈕用 #3b82f6 藍色"
3. **保留好的部分**：如果某些部分已經滿意，在 Prompt 中提到"顏色保持不變"
4. **不超過 5 輪**：如果 5 輪還不滿意，可能需要更清晰地描述你的需求

## 本步檢查點

調整完成後，確認：

- [ ] 整體佈局正確（標題、輸入區、列表區都在）
- [ ] 配色基本滿意
- [ ] 間距和尺寸看起來舒適
- [ ] 手機寬度下也能正常顯示
