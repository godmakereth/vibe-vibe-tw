# 4.2.4 階段性成果檢查

> **本步目標**：確認第一輪完成，準備進入下一輪

## 第一輪完成檢查清單

在進入下一輪之前，請確認以下內容：

### 必須項

- [ ] **頁面能正常顯示**：在瀏覽器中打開，沒有空白或報錯
- [ ] **標題存在**：能看到"我的待辦清單"（或你自定義的標題）
- [ ] **輸入框存在**：有一個可以輸入文字的輸入框
- [ ] **添加按鈕存在**：有一個"添加"按鈕
- [ ] **任務列表存在**：能看到任務列表區域（即使是假數據）
- [ ] **操作按鈕存在**：每個任務有"完成"和"刪除"按鈕

### 加分項

- [ ] 整體樣式你基本滿意
- [ ] 鼠標懸停在按鈕上有視覺反饋
- [ ] 在手機尺寸下也能正常顯示

## 如何測試響應式效果

想知道頁面在手機上看起來怎麼樣？不需要真的用手機，瀏覽器可以模擬：

1. 在瀏覽器中打開你的頁面
2. 按 `F12` 打開開發者工具
3. 點擊工具欄左上角的"設備切換"圖標（長得像手機和平板）
4. 選擇一個手機型號（如 iPhone 12）或者直接拖動頁面寬度

檢查這些情況：

- [ ] 頁面內容沒有超出屏幕
- [ ] 文字大小在手機上也能看清
- [ ] 按鈕在手機上也容易點擊
- [ ] 輸入框寬度適合手機屏幕

::: tip 小技巧
如果手機上顯示有問題，回到 4.2.3 節，使用"響應式適配類"的調整 Prompt。
:::
