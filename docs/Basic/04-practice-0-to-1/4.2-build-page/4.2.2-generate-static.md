# 4.2.2 生成靜態頁面

> **本步目標**：把 Prompt 發送給 AI，獲得第一版靜態頁面

## 發送 Prompt

現在，把你在上一步準備好的 Prompt 發送給 AI IDE。

還記得 4.1.2 節介紹的界面佈局嗎？找到 **對話區**，把 Prompt 粘貼進去，按回車發送。

不同 AI IDE 的對話區位置可能略有不同，但都很好找：

- 通常在界面底部或側邊欄
- 可能標註着 "Chat"、"AI"、"Composer"、"Cascade" 等字樣
- 有些工具可以用快捷鍵打開（如 Cursor 的 `Cmd/Ctrl + L`）

::: tip 找不到對話區？
回顧 [4.1.2 準備你的 AI IDE](../4.1-before-start/4.1.2-prepare-ai-ide.md) 中的界面佈局說明。如果你使用的工具不在常見列表中，可以查看 [1.4.1 AI 編程工具全景圖](../../01-awakening/1.4-tools-guide/1.4.1-env-setup.md) 獲取更多工具信息。
:::

發送後，等待 AI 生成代碼。通常需要 10-30 秒。

## 你應該看到什麼

AI 生成完成後，你應該能看到一個待辦清單頁面，包含以下結構：

```
┌─────────────────────────────────────┐
│                                     │
│        我的待辦清單                  │  ← 標題區
│                                     │
├─────────────────────────────────────┤
│  ┌─────────────────────┐ ┌────┐    │
│  │ 輸入新任務...        │ │添加│    │  ← 輸入區
│  └─────────────────────┘ └────┘    │
├─────────────────────────────────────┤
│                                     │
│  □ 完成項目報告      [完成] [刪除]  │  ← 任務列表區
│  □ 買牛奶            [完成] [刪除]  │    （假數據）
│  □ 給媽媽打電話      [完成] [刪除]  │
│                                     │
└─────────────────────────────────────┘
```

具體檢查這些元素是否存在：

- [ ] 頁面有標題"我的待辦清單"
- [ ] 有一個輸入框
- [ ] 有一個"添加"按鈕
- [ ] 有任務列表，顯示幾條假數據
- [ ] 每個任務有"完成"和"刪除"按鈕
- [ ] 整體佈局居中，看起來比較整齊

## 五種可能的情況及處理

第一次生成的結果可能會有差異。根據你看到的情況，選擇對應的處理方式：

### 情況一：完美符合預期

**現象**：頁面佈局、樣式、元素都符合預期。

**處理**：太棒了！直接跳到 [4.2.4 階段性成果檢查](./4.2.4-checkpoint.md)。

### 情況二：結構對，但樣式不滿意

**現象**：所有元素都在，但顏色太暗、按鈕太小、間距不夠等。

**處理**：進入 [4.2.3 調整樣式與佈局](./4.2.3-adjust-style.md)，用迭代對話調整。

### 情況三：缺少某些元素

**現象**：比如沒有"完成"按鈕，或者沒有假數據。

**處理**：發送追加說明：

```
頁面整體不錯，請補充以下內容：
- 每個任務需要有"完成"按鈕和"刪除"按鈕
- 任務列表先用 3 條假數據展示樣式
```

### 情況四：頁面空白

**現象**：瀏覽器顯示空白頁面，什麼都沒有。

**處理**：這通常是文件保存或預覽問題，按以下順序排查：

1. **確認文件已保存**：在 AI IDE 中按 `Ctrl+S`（Mac 用 `Cmd+S`）
2. **刷新預覽**：按 `Ctrl+Shift+R`（Mac 用 `Cmd+Shift+R`）強制刷新
3. **檢查文件位置**：確認 HTML 文件保存在正確的位置
4. **查看控制檯**：按 `F12` 打開開發者工具，看是否有紅色錯誤

如果以上都不行，讓 AI 幫你檢查：

```
頁面顯示空白，請幫我檢查代碼是否有問題。
如果有問題，請修復並給我完整的正確代碼。
```

### 情況五：代碼報錯

**現象**：瀏覽器控制檯顯示紅色錯誤信息。

**處理**：跳到 [4.5.2 用 AI 幫你修 AI 的代碼](../4.5-debug/4.5.2-fix-with-ai.md)，學習如何讓 AI 幫你修復。

## 理解 AI 生成的代碼結構

你不需要完全理解代碼，但瞭解大致結構會幫助你後續調整。AI 生成的 HTML 文件通常是這樣的：

```html
<!DOCTYPE html>
<html>
<head>
    <!-- 頁面標題和設置 -->
    <title>我的待辦清單</title>
    <style>
        /* 
         * 這裏是 CSS 樣式代碼
         * 控制頁面的顏色、大小、佈局
         */
    </style>
</head>
<body>
    <!-- 這裏是頁面內容 -->
    
    <div class="container">
        <!-- 標題 -->
        <h1>我的待辦清單</h1>
        
        <!-- 輸入區域 -->
        <div class="input-area">
            <input type="text" placeholder="輸入新任務...">
            <button>添加</button>
        </div>
        
        <!-- 任務列表 -->
        <ul class="task-list">
            <li>
                <span>完成項目報告</span>
                <button>完成</button>
                <button>刪除</button>
            </li>
            <!-- 更多任務... -->
        </ul>
    </div>
</body>
</html>
```

簡單理解：

| 代碼部分 | 作用 |
|---------|------|
| `<head>...</head>` | 頁面的"幕後設置"，包括標題和樣式 |
| `<style>...</style>` | CSS 樣式，控制外觀 |
| `<body>...</body>` | 頁面的"正文內容"，用戶看到的部分 |
| `<div>` | 一個容器，用來分組和佈局 |
| `<input>` | 輸入框 |
| `<button>` | 按鈕 |
| `<ul>` 和 `<li>` | 列表和列表項 |

::: info 不需要記住這些
這只是幫助你理解代碼結構。在 Vibe Coding 中，你不需要自己寫這些代碼——只需要告訴 AI 你想要什麼，它會幫你寫。
:::

## 點擊按鈕沒反應？

如果你嘗試點擊"添加"按鈕，發現什麼都沒發生——這是**完全正常的**。

現在的頁面只是一個"殼"，還沒有任何功能代碼。就像一輛車的模型，看起來像車，但還不能開動。

在 4.3 節，我們會讓這些按鈕"活"起來。

## 常見問題速查

| 問題 | 可能原因 | 解決方法 |
|-----|---------|---------|
| 頁面完全空白 | 文件未保存或未正確預覽 | 保存文件，刷新瀏覽器 |
| 樣式看起來很醜 | AI 的默認樣式不符合你的審美 | 進入 4.2.3 調整 |
| 只有文字沒有樣式 | CSS 未正確引入 | 讓 AI 檢查代碼結構 |
| 頁面在手機上顯示異常 | 缺少響應式設置 | 要求 AI 添加響應式樣式 |
| 按鈕點擊無反應 | 正常！靜態頁面沒有功能 | 4.3 節會添加功能 |

- [ ] AI 已經生成了代碼
