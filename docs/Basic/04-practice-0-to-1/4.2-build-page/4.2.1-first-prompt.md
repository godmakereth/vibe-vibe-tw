# 4.2.1 用第三章框架寫第一個 Prompt

> **本步目標**：把 4.1 節的"項目啓動卡"轉化爲 AI 能理解的 Prompt

## 從項目啓動卡到 Prompt

在 4.1 節，我們整理了小李項目的「項目啓動卡」。現在要把它轉化爲 Prompt，讓 AI 幫我們生成代碼。

```
項目啓動卡                         Prompt
┌──────────────┐                ┌──────────────┐
│ 項目名稱      │                │ Situation    │
│ 目標用戶      │  ──轉化──→     │ Challenge    │
│ 核心功能      │                │ Audience     │
│ 明確不做      │                │ Format       │
│ 技術約束      │                │ Foundations  │
└──────────────┘                └──────────────┘
```

轉化的邏輯：

| 項目啓動卡內容 | 對應 Prompt 位置 | 作用 |
|--------------|-----------------|------|
| 項目名稱 + 目標用戶 | Situation（情境） | 告訴 AI 你在做什麼 |
| 核心功能列表 | Challenge（挑戰） | 告訴 AI 當前要做什麼 |
| 你是誰 | Audience（受衆） | 讓 AI 調整代碼複雜度 |
| 期望輸出格式 | Format（格式） | 讓 AI 知道給你什麼 |
| 明確不做 + 技術約束 | Foundations（基礎） | 告訴 AI 邊界在哪 |

## 三種 Prompt 版本

根據你的時間和需求，可以選擇不同詳細程度的 Prompt。

### 版本一：最簡版（趕時間用）

如果你已經熟悉了這個流程，或者只是想快速看到效果：

```
幫我做一個待辦清單網頁的靜態頁面。
包含：標題、輸入框、添加按鈕、任務列表（帶完成和刪除按鈕）。
風格簡約現代，要能在手機上正常顯示。
先不需要功能，只要頁面樣式。用 HTML + CSS，不用任何框架。
```

這個版本大約 70 字，能覆蓋最核心的信息。

### 版本二：標準版（推薦日常使用）

使用第三章的 S.C.F. 精簡框架：

```
【背景】
我正在做一個個人待辦清單網頁，幫助管理日常任務。
我是編程零基礎的初學者。

【任務】
請生成待辦清單的靜態頁面，包含以下元素：
- 頁面標題："我的待辦清單"
- 一個輸入框，用於輸入新任務
- 一個"添加"按鈕
- 任務列表區域，先用 3 條假數據展示樣式
- 每個任務有"完成"按鈕和"刪除"按鈕

【約束】
- 只需要靜態頁面，暫時不需要任何功能
- 使用純 HTML + CSS，不使用任何框架
- 現代簡約風格，清新配色
- 響應式設計，手機上也能正常顯示
- 代碼加上必要的中文註釋
```

### 版本三：完整版（第一次使用建議）

使用 S.C.A.F.F. 全要素框架，給 AI 最充分的上下文：

```
【Situation - 情境】
我是一個編程零基礎的初學者，正在學習用 AI 做一個個人待辦清單網頁。
這是我的第一個 AI 編程項目。
目標用戶是像我一樣容易忘事的職場新人，需要一個簡單的任務管理工具。

【Challenge - 挑戰】
請幫我生成待辦清單的靜態頁面，包含以下元素：
1. 頁面標題："我的待辦清單"
2. 輸入區域：
   - 一個輸入框（用於輸入新任務）
   - 一個"添加"按鈕
3. 任務列表區域：
   - 先用 3 條假數據展示樣式（如："完成項目報告"、"買牛奶"、"給媽媽打電話"）
   - 每個任務顯示任務內容
   - 每個任務右側有"完成"按鈕和"刪除"按鈕

【Audience - 受衆】
- 代碼將由我這個初學者閱讀和修改
- 請保持代碼簡單易懂
- 在關鍵位置添加中文註釋，幫助我理解代碼結構

【Format - 格式】
請生成一個完整的 HTML 文件，包含：
- HTML 結構
- 內嵌的 CSS 樣式（寫在 <style> 標籤裏）
- 文件可以直接在瀏覽器中打開查看效果

【Foundations - 基礎約束】
- 只做靜態頁面，暫時不需要 JavaScript 功能
- 不使用任何外部框架或庫（如 Bootstrap、Tailwind）
- 設計風格：現代簡約，參考蘋果或 Notion 的設計語言
- 配色清新，主色調可以用藍色或綠色系
- 響應式設計：在手機（寬度 375px）和電腦上都能正常顯示
- 頁面整體居中顯示，最大寬度 500px
```

## 爲什麼要告訴 AI 你是初學者？

你可能注意到，每個版本都提到了"我是初學者"。這不是謙虛，而是一個重要的策略：

| 不說"初學者" | 說了"初學者" |
|-------------|-------------|
| AI 可能使用高級語法和技巧 | AI 會使用基礎、直觀的寫法 |
| 代碼可能沒有註釋 | AI 會主動添加解釋性註釋 |
| 變量名可能很簡短 | AI 會使用更具描述性的命名 |
| 遇到問題你可能看不懂 | 代碼更容易閱讀和調試 |

這個小技巧能讓 AI 生成的代碼更適合你當前的水平。

## 描述方式對比：模糊 vs 具體

寫 Prompt 時，越具體越好。看看這些對比：

| 方面 | 模糊描述（避免） | 具體描述（推薦） |
|-----|----------------|-----------------|
| 顏色 | 好看的顏色 | 主色調用藍色系，如 #4F46E5 |
| 佈局 | 居中顯示 | 頁面最大寬度 500px，水平居中 |
| 風格 | 現代風格 | 參考蘋果官網的極簡風格，圓角卡片 |
| 按鈕 | 添加一些按鈕 | 每個任務右側有"完成"和"刪除"兩個按鈕 |
| 數據 | 展示一些任務 | 用 3 條假數據，如"完成項目報告"等 |

爲什麼具體描述更好？因爲 AI 面對模糊描述時只能"猜"，而具體描述讓 AI 知道你到底想要什麼。

## 檢查你的 Prompt

在發送之前，用這個清單檢查一下：

- [ ] 說明了項目背景（做什麼、給誰用）？
- [ ] 列出了要包含的頁面元素？
- [ ] 指定了技術約束（HTML + CSS，不用框架）？
- [ ] 說明了設計風格和配色偏好？
- [ ] 提到了響應式需求？
- [ ] 告訴 AI 你是初學者，需要註釋？

## 選擇哪個版本？

| 你的情況 | 建議使用 |
|---------|---------|
| 第一次跟着教程做 | **完整版**，獲得最好的結果 |
| 已經做過一次，想快速複習 | 標準版 |
| 已經很熟練，想快速驗證想法 | 最簡版 |

::: tip 建議
如果這是你第一次做 AI 編程項目，建議使用完整版。多花 2 分鐘寫 Prompt，能省下後面很多調整時間。
:::
