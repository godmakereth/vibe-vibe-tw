# 4.2 第一輪：搭建頁面框架

> **本輪目標**：用 AI 生成待辦清單的靜態頁面，先把"臉"做出來

## 經過本節學習，你將獲得

- 一個可以在瀏覽器中打開的待辦清單頁面
- 包含標題、輸入框、按鈕和任務列表的完整佈局
- 簡潔美觀的視覺設計
- 在手機和電腦上都能正常顯示的響應式頁面

## 什麼是"靜態頁面"？

靜態頁面就是隻有外觀、還沒有功能的頁面。

想象你在裝修新房：

```
毛坯房 → 刷白牆、鋪地板（先看整體效果）→ 裝電器、接水管（讓它能用）→ 入住
```

我們現在要做的，就是"刷白牆"這一步——**先把界面搭出來，看看整體效果對不對**。

這樣做有三個好處：

| 好處 | 說明 |
|------|------|
| 快速看到成果 | 5-10 分鐘就能看到頁面，建立信心 |
| 及早發現問題 | 佈局不對、風格不滿意，現在改成本最低 |
| 避免返工 | 如果做了很多功能後才發現方向錯了，改起來很痛苦 |

## 本輪的工作流程

```
第二章的思考成果 → 用第三章框架寫 Prompt → AI 生成靜態頁面 → 調整樣式 → 檢查確認
```

我們會用到 4.1 節整理的「項目啓動卡」和第三章學的 S.C.A.F.F. 框架。
