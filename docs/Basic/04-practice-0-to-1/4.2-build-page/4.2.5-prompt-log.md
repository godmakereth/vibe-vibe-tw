# 4.2.5 完整 Prompt 對話記錄

> **本節目標**：回顧從零到靜態頁面的完整對話過程，作爲你的參考模板

## 爲什麼要記錄對話？

記錄完整的對話過程有幾個好處：

1. **可復現**：如果換一個項目，你可以參照這個流程
2. **可學習**：回看每一輪用了什麼技巧，加深理解
3. **可排錯**：如果後續出問題，可以回顧哪裏可能出錯

## 完整對話記錄示例

以下是小李完成第一輪靜態頁面的完整對話過程。

### 第 1 輪：初始生成

**小李發送**：

```
【Situation - 情境】
我是一個編程零基礎的初學者，正在學習用 AI 做一個個人待辦清單網頁。
這是我的第一個 AI 編程項目。
目標用戶是像我一樣容易忘事的職場新人，需要一個簡單的任務管理工具。

【Challenge - 挑戰】
請幫我生成待辦清單的靜態頁面，包含以下元素：
1. 頁面標題："我的待辦清單"
2. 輸入區域：
   - 一個輸入框（用於輸入新任務）
   - 一個"添加"按鈕
3. 任務列表區域：
   - 先用 3 條假數據展示樣式
   - 每個任務顯示任務內容
   - 每個任務右側有"完成"按鈕和"刪除"按鈕

【Audience - 受衆】
代碼將由我這個初學者閱讀和修改，請保持簡單易懂，在關鍵位置添加中文註釋。

【Format - 格式】
請生成一個完整的 HTML 文件，包含內嵌的 CSS 樣式。

【Foundations - 基礎約束】
- 只做靜態頁面，暫時不需要 JavaScript 功能
- 不使用任何外部框架
- 現代簡約風格，配色清新
- 響應式設計，手機上也能正常顯示
```

**AI 回覆**：[生成 HTML + CSS 代碼，頁面包含基本結構]

**使用的技巧**：
- ✓ S.C.A.F.F. 完整框架
- ✓ 明確說明是初學者（降低代碼複雜度）
- ✓ 具體列出頁面元素（避免遺漏）
- ✓ 明確技術約束（不用框架，純 HTML+CSS）

**小李的檢查結果**：
- ✅ 結構完整，所有元素都在
- ⚠️ 按鈕樣式有點普通
- ⚠️ 整體配色偏暗

### 第 2 輪：調整配色和按鈕

**小李發送**：

```
整體結構很好！請做以下調整：

1. 配色問題：
   - 背景改成更淺的灰色（#f8fafc）
   - 主色調用明亮的藍色（#3b82f6）
   
2. 按鈕樣式：
   - "添加"按鈕改成圓角（8px），背景用藍色，文字白色
   - 鼠標懸停時顏色加深
   - "完成"按鈕用綠色（#10b981），"刪除"按鈕用紅色（#ef4444）

其他部分保持不變。
```

**AI 回覆**：[更新後的代碼，配色和按鈕樣式已調整]

**使用的技巧**：
- ✓ 具體的顏色值（不是"好看的顏色"）
- ✓ 明確指出要保留的部分（"其他部分保持不變"）
- ✓ 一次只調整一類問題（配色 + 按鈕）

**小李的檢查結果**：
- ✅ 配色清新多了
- ✅ 按鈕好看了
- ⚠️ 任務之間間距太小

### 第 3 輪：調整間距

**小李發送**：

```
顏色很好！請調整一下間距：

1. 任務列表的每一項之間增加 12px 的間距
2. 每個任務加一條淺灰色的底邊框（顏色用 #e5e7eb）
3. 任務右側的"完成"和"刪除"按鈕之間留 8px 間距

配色和其他樣式保持不變。
```

**AI 回覆**：[最終版本代碼，間距已調整]

**使用的技巧**：
- ✓ 具體的像素值（12px、8px）
- ✓ 具體的顏色值（#e5e7eb）
- ✓ 再次強調保留已滿意的部分

**小李的檢查結果**：
- ✅ 間距舒適
- ✅ 視覺層次清晰
- ✅ 整體滿意，可以進入下一輪

### 第 4 輪：最後確認

**小李發送**：

```
看起來很不錯了！請幫我確認一下：
1. 頁面在手機寬度（375px）下顯示是否正常？
2. 如果有問題，請修復。
```

**AI 回覆**：[確認響應式正常，或者提供修復後的代碼]

**使用的技巧**：
- ✓ 主動確認潛在問題
- ✓ 給出具體的測試條件（375px）

## 對話中使用的技巧彙總

| 輪次 | 主要技巧 | 來自章節 |
|-----|---------|---------|
| 第 1 輪 | S.C.A.F.F. 框架 | 3.2.2 |
| 第 2 輪 | 具體描述 + 保留好的部分 | 3.5.3 |
| 第 3 輪 | 漸進式調整 | 3.5.2 |
| 第 4 輪 | 主動驗證 | 3.5.3 |

## 這段對話可以改進的地方

回顧這段對話，有幾個地方可以做得更好：

### 1. 第一輪可以更具體

原版：
```
現代簡約風格，配色清新
```

改進版：
```
參考蘋果備忘錄的風格，純白背景，圓角卡片，主色調用藍色（#3b82f6）
```

**教訓**：第一輪就給出具體的風格參考和顏色，可以減少後續調整輪數。

### 2. 可以合併第 2、3 輪

如果一開始就把配色和間距問題一起說，可以省一輪對話：

```
請做以下調整：

配色：
- 背景用 #f8fafc
- 主色調用 #3b82f6

按鈕：
- 圓角 8px，懸停時顏色加深
- "完成"用綠色，"刪除"用紅色

間距：
- 任務之間 12px 間距
- 按鈕之間 8px 間距
```

**教訓**：如果問題明確，可以一次提多個調整，節省對話輪數。

### 3. 第 4 輪可以更早做

響應式測試可以在第 1 輪生成後就做，不用等到最後。

**教訓**：核心功能和響應式要儘早驗證，發現問題早修復。

## 你的對話記錄模板

你可以用這個模板記錄自己的對話過程：

```markdown
## 項目：[項目名稱]
## 日期：[日期]

### 第 1 輪
**我發送**：
[你的 Prompt]

**AI 回覆**：
[簡要描述 AI 做了什麼]

**檢查結果**：
- ✅ [好的地方]
- ⚠️ [需要改進的地方]

### 第 2 輪
...
```

記錄對話不是必須的，但如果你想回顧學習過程，或者以後遇到類似項目想參考，這會很有幫助。

## 本節小結

通過這個完整的對話記錄，你可以看到：

1. **第一輪用 S.C.A.F.F. 框架**，給 AI 充分的上下文
2. **後續輪次用具體描述**，一次調整一類問題
3. **每輪都檢查結果**，確認哪些滿意、哪些需要改進
4. **3-4 輪對話**是正常的，不要期望一次完美

現在，你可以進入第二輪，讓頁面"活"起來了。
