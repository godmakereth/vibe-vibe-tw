# 4.3.4 迭代優化的藝術

> **本節目標**：學會如何有效地追問，讓 AI 把"能用"變成"好用"

到目前爲止，你的待辦清單已經"能用"了。但可能有些地方不太滿意：

- 樣式想再調整一下
- 交互細節想優化
- 想添加一些小功能

這時候，**迭代追問**就派上用場了。

## 爲什麼 AI 第一次輸出往往不完美

這不是 AI 的問題，也不是你的問題。原因很簡單：

| 你腦海中的畫面 | AI 理解的內容 | 實際輸出 |
|--------------|--------------|---------|
| 100% 細節 | 70% 文字描述 | 60% 代碼實現 |

信息在傳遞過程中必然會有損失。所以**第一次輸出不完美是正常的**，迭代追問纔是常態。

::: info 📖 回顧第三章
這正是第三章 3.5 節"迭代對話的藝術"中講到的核心觀點。本節是那些原則的實戰演練。
:::

## 追問的三要素

有效的追問需要包含三個信息：

| 要素 | 說明 | 示例 |
|------|------|------|
| **現狀** | 現在是什麼樣 | "點擊完成後，任務還在原來的位置" |
| **期望** | 你想要什麼樣 | "我希望已完成的任務自動移到列表底部" |
| **差異** | 具體差在哪 | "現在位置不變，我要它移到底部" |

## 真實迭代對話示例

假設你對"標記完成"功能不太滿意，想做一些調整。看看這個三輪對話：

### 第一輪：發現問題

**你發現**：已完成的任務和未完成的混在一起，不好區分。

**你發送**：
```
完成功能可以用了，但我有個改進想法：
我希望已完成的任務能自動移到列表底部，
這樣未完成的任務始終顯示在上面，更容易看到。
請幫我實現這個功能。
```

**AI 可能的回應**：修改代碼，實現了自動排序。

### 第二輪：繼續調整

**你測試後發現**：排序是有了，但新添加的任務出現在了列表最後面，不方便看。

**你發送**：
```
排序功能實現了，但有個新問題：
當前情況：新添加的任務出現在列表最後面
我希望的效果：新任務應該出現在未完成任務的最上面
請調整一下，讓新任務總是顯示在最頂部（已完成的任務之前）。
```

### 第三輪：細節打磨

**你發送**：
```
排序邏輯現在正確了。
還想做一個小優化：能否添加一個"清空已完成"按鈕？
點擊後可以一次性刪除所有已完成的任務。
按鈕可以放在列表底部，只在有已完成任務時才顯示。
```

三輪對話後，你得到了一個比最初版本好用很多的待辦清單。

## 有效反饋 vs 無效反饋

| 無效反饋 | 問題在哪 | 有效反饋 |
|---------|---------|---------|
| "不對，再改改" | AI 不知道哪裏不對 | "添加後任務沒顯示，請檢查..." |
| "樣式不好看" | 太模糊，AI 無從下手 | "標題字體太小，請改成24px" |
| "感覺怪怪的" | 沒有具體信息 | "按鈕點擊後，文字變化有延遲" |
| "能不能更好一點" | 什麼叫"更好"？ | "希望按鈕有懸停效果，顏色加深" |

**核心原則**：你的描述越具體，AI 修改得越準確。

## 追問時的實用句式

收集一些好用的句式，遇到問題時可以直接套用：

### 描述問題

```
當前情況：[現在是什麼樣]
我期望的效果：[想要什麼樣]
請幫我修改。
```

### 請求添加功能

```
現有功能運行正常。
我想在此基礎上添加一個新功能：[描述功能]
具體來說：[列出細節]
```

### 調整樣式

```
功能正常，但樣式想調整：
- 當前：[現在的樣式]
- 期望：[想要的樣式]
請只修改樣式，不要改變功能邏輯。
```

### 修復 Bug

```
發現一個問題：
- 操作步驟：[1、2、3]
- 期望結果：[應該發生什麼]
- 實際結果：[實際發生了什麼]
請幫我修復。
```

## 知道什麼時候換個思路

如果同一個問題追問了 3 次還沒解決，考慮：

| 情況 | 建議 |
|------|------|
| AI 似乎理解錯了需求 | 換一種描述方式，用更具體的例子 |
| 問題太複雜 | 把需求拆成更小的步驟，一步步來 |
| AI 反覆給出類似的錯誤代碼 | 開始一個新對話，把需求重新整理後發送 |
| 超出了當前技術範圍 | 這個功能可能需要更復雜的實現，考慮暫時擱置 |

## 什麼時候開始新對話

有時候，與其在舊對話裏繼續追問，不如開始一個新對話：

| 場景 | 建議 |
|------|------|
| 對話已經很長（超過 10 輪） | AI 可能"失憶"，開新對話 |
| 需求發生了較大變化 | 把新需求整理清楚，開新對話 |
| 代碼改得面目全非 | 保存當前可用版本，開新對話從頭優化 |
| AI 似乎陷入了某種錯誤循環 | 開新對話，換一種描述方式 |

**開始新對話時的模板**：

```
我正在開發一個待辦清單網頁。

【當前狀態】
- 已實現：添加任務、刪除任務、標記完成
- 現有代碼運行正常

【本次目標】
我想優化 [具體功能]，具體需求：
1. [需求1]
2. [需求2]

【現有代碼】
[粘貼當前代碼]
```

## 本節核心要點

- ✓ 第一次輸出不完美是正常的，迭代纔是常態
- ✓ 有效追問需要三要素：現狀 + 期望 + 差異
- ✓ 描述越具體，AI 修改得越準確
- ✓ 追問 3 次還沒解決，考慮換思路或開新對話

趁熱打鐵，嘗試用本節學到的技巧，對你的待辦清單做一些優化：

- [ ] 調整一個你不滿意的樣式
- [ ] 添加一個你想要的小功能
- [ ] 如果有 Bug，嘗試用"描述問題"的句式讓 AI 修復
