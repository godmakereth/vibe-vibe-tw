# 4.3 第二輪：實現核心功能

> **本輪目標**：讓頁面能響應用戶操作——點擊按鈕真的有反應

在上一輪（4.2），你已經做出了一個"好看的空殼"：有標題、有輸入框、有按鈕、有列表區域。但如果你現在點擊"添加"按鈕，什麼都不會發生。

這就像造了一輛漂亮的汽車，但還沒裝發動機。

**本輪，我們要給它裝上"發動機"**——讓每個按鈕都能真正工作。

## 從"靜態"到"動態"

在 4.2 節，我們用 HTML 和 CSS 搭建了頁面。現在需要加入 JavaScript，讓頁面能"響應"用戶的操作。

| 概念 | 類比 | 在待辦清單中的體現 |
|------|------|-------------------|
| HTML | 房子的骨架 | 輸入框、按鈕、列表的結構 |
| CSS | 房子的裝修 | 顏色、字體、佈局 |
| JavaScript | 房子的電路和開關 | 點擊按鈕後的響應邏輯 |

你不需要"學會" JavaScript。你只需要告訴 AI 你想要什麼效果，AI 會幫你寫代碼。

## 本輪要實現的三個功能

還記得第二章定義的 P0 功能嗎？本輪我們要實現最核心的三個：

| 功能 | 用戶操作 | 預期結果 |
|------|---------|---------|
| 添加任務 | 輸入內容，點擊"添加"按鈕 | 新任務出現在列表中 |
| 刪除任務 | 點擊任務旁邊的"刪除"按鈕 | 該任務從列表中消失 |
| 標記完成 | 點擊任務的"完成"按鈕 | 任務顯示爲已完成狀態（加刪除線） |

這三個功能覆蓋了"增、刪、改"操作，是幾乎所有應用的基礎模式。

## 事件處理：讓頁面"聽懂"你的操作

在開始之前，先理解一個核心概念：**事件處理**。

想象你在餐廳點餐：

1. 你按下呼叫服務員的按鈕（**觸發事件**）
2. 服務員聽到鈴聲（**監聽事件**）
3. 服務員走過來爲你服務（**執行響應**）

JavaScript 的事件處理也是這個邏輯：

```
用戶點擊按鈕 → 程序監聽到點擊 → 執行相應的代碼
```

你不需要自己寫這些代碼。只需要告訴 AI："當用戶點擊添加按鈕時，把輸入的內容加到列表裏"，AI 就會幫你實現。

## 本輪的工作流程

```
4.3.1 添加任務 → 讓"添加"按鈕能用
       ↓
4.3.2 刪除任務 → 讓"刪除"按鈕能用
       ↓
4.3.3 標記完成 → 讓"完成"按鈕能用
       ↓
4.3.4 迭代優化 → 調整不滿意的地方
       ↓
4.3.5 階段檢查 → 確認所有功能正常
```

每完成一個功能，都要測試確認可以正常使用，再進入下一個。
