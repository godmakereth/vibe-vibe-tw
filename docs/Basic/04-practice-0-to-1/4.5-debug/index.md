# 4.5 Debug 實戰：當 AI 代碼報錯時

經過本節學習，你將掌握：
- 識別待辦清單開發中的四類常見錯誤
- 用 AI 診斷和修復代碼問題的標準流程
- 處理真實報錯的完整對話技巧
- 建立「報錯不可怕」的 Debug 心態

## 與第三章的關係

第三章 3.6 節講的是**原理層面**：AI 有哪些「不聽話」的表現、如何識別 AI 幻覺、診斷問題的思維框架。

本節講的是**實操層面**：圍繞你正在開發的待辦清單項目，展示真實的報錯信息、完整的修復對話、可直接複製的 Prompt 模板。

簡單說：3.6 教你「遇到問題該怎麼想」，4.5 教你「遇到問題該怎麼做」。

## 爲什麼報錯是正常的

在開發待辦清單的過程中，你可能已經遇到了一些報錯。別擔心，這完全正常。

| 誤解 | 事實 |
|------|------|
| "報錯說明我做錯了" | 報錯是代碼在告訴你哪裏需要調整 |
| "專業程序員不會遇到報錯" | 專業程序員每天都在 debug，只是他們知道怎麼解決 |
| "AI 生成的代碼應該沒問題" | AI 不瞭解你的具體環境，需要你幫它調整 |

好消息是：**AI 能幫你解決 90% 的常見問題**。你只需要學會「怎麼問」。
