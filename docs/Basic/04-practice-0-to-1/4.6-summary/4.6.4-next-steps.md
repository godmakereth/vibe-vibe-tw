# 4.6.4 下一步可以做什麼

> **本步目標**：規劃學習路徑，明確短期和中期目標

完成了待辦清單，你可能在想：接下來做什麼？這裏給你三個方向的選擇。

## 方向一：給待辦清單加功能（立即可做）

如果你想繼續打磨這個項目，以下是一些擴展功能的 Prompt 示例：

### 任務分類

```
【背景】
我的待辦清單已經可以添加、刪除、完成任務，並且數據會保存在 localStorage 中。

【當前任務】
請給任務添加分類功能：
1. 預設三個分類：工作、生活、學習
2. 添加任務時可以選擇分類
3. 每個分類用不同顏色標識
4. 可以按分類篩選顯示任務

【技術要求】
- 保持使用 localStorage 存儲
- 代碼風格與現有代碼一致
```

### 優先級標記

```
【背景】
我的待辦清單目前功能完整，需要添加優先級功能。

【當前任務】
請給任務添加優先級功能：
1. 三個優先級：高、中、低
2. 高優先級用紅色標識，中用黃色，低用灰色
3. 任務列表自動按優先級排序
4. 添加任務時可以選擇優先級，默認爲"中"

【技術要求】
- 優先級信息也要保存到 localStorage
- 保持現有的完成、刪除功能正常工作
```

### 搜索功能

```
【背景】
我的待辦清單任務越來越多，需要搜索功能。

【當前任務】
請添加任務搜索功能：
1. 在輸入框上方添加一個搜索框
2. 輸入關鍵詞時，即時過濾顯示匹配的任務
3. 搜索框爲空時，顯示所有任務
4. 搜索不區分大小寫

【技術要求】
- 搜索是前端過濾，不影響 localStorage 中的數據
```

這些功能你可以一個一個添加，每添加一個就測試一次，感受漸進式開發的節奏。

## 方向二：進入第五章（推薦的下一步）

第五章會解決你現在可能遇到的三個問題：

### 問題一：改崩了回不去

**場景**：你嘗試加了一個新功能，結果把原來能用的功能也搞壞了，而且你已經保存了文件，撤銷不回去了。

**第五章解決方案**：5.1 版本管理

你會學到：
- 什麼是版本管理，爲什麼需要它
- 用 GitHub Desktop 管理代碼版本
- 每次改動都能"存檔"，隨時回退

### 問題二：只能自己用

**場景**：你想把待辦清單分享給朋友，但現在它只是你電腦上的一個文件，別人打不開。

**第五章解決方案**：5.2 部署上線

你會學到：
- 用 Vercel 把網頁部署到網上
- 獲得一個真正的網址，任何人都能訪問
- 可選：綁定自己的域名

### 問題三：安全意識

**場景**：你想讓 AI 幫你做一個有登錄功能的應用，但不知道怎麼處理密碼和敏感信息。

**第五章解決方案**：5.3 安全意識

你會學到：
- 哪些東西絕對不能告訴 AI
- 如何正確處理敏感信息
- 初學者階段的安全底線

| 第五章小節 | 解決的問題 | 預計時間 |
|-----------|-----------|---------|
| 5.1 版本管理 | 改崩了回不去 | 30 分鐘 |
| 5.2 部署上線 | 只能自己用 | 20 分鐘 |
| 5.3 安全意識 | 不知道什麼不能做 | 15 分鐘 |
| 5.4 侷限性 | Vibe Coding 的邊界 | 10 分鐘 |

## 方向三：做一個新項目（鞏固技能）

用你學到的工作流，嘗試做一個新項目。以下是一些適合練手的想法：

### 個人工具類

| 項目 | 核心功能 | 複雜度 |
|------|---------|-------|
| **記事本** | 寫文字、保存、編輯 | ⭐ |
| **書籤管理器** | 保存網址、分類、搜索 | ⭐⭐ |
| **倒計時器** | 設置時間、開始、暫停、提醒 | ⭐⭐ |
| **日記本** | 按日期寫日記、查看歷史 | ⭐⭐ |

### 習慣追蹤類

| 項目 | 核心功能 | 複雜度 |
|------|---------|-------|
| **習慣打卡器** | 添加習慣、每日打卡、查看連續天數 | ⭐⭐ |
| **閱讀記錄** | 記錄書名、進度、筆記 | ⭐⭐ |
| **喝水提醒** | 記錄喝水次數、今日目標 | ⭐ |

### 給家人做的工具

| 項目 | 核心功能 | 複雜度 |
|------|---------|-------|
| **家庭相冊** | 展示照片、添加描述 | ⭐⭐ |
| **家庭食譜** | 收集菜譜、分類查看 | ⭐⭐ |
| **紀念日提醒** | 記錄重要日期、距離天數 | ⭐ |

選擇項目的建議：
1. **從自己的痛點出發**：做一個自己真的會用的東西
2. **複雜度適中**：第一個項目選 ⭐ 或 ⭐⭐，不要一上來就挑戰 ⭐⭐⭐
3. **功能要少**：第一版只做核心功能，後續再迭代
