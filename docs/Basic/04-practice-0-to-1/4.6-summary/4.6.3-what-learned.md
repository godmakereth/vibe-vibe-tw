# 4.6.3 你學到了什麼

> **本步目標**：明確你獲得的具體技能，用自評表確認學習成果

## 技能盤點：三個層次的收穫

完成第四章後，你的收穫可以分爲三個層次：

### 第一層：工作流技能

你掌握了從想法到產品的完整流程：

| 階段 | 你學會了 | 具體表現 |
|------|---------|---------|
| **需求階段** | 用結構化方式定義項目 | 能填寫項目啓動卡 |
| **表達階段** | 用 AI 能理解的方式寫 Prompt | 會用 S.C.A.F.F. 框架 |
| **開發階段** | 分階段推進項目 | 靜態→交互→存儲三輪迭代 |
| **驗證階段** | 系統性測試功能 | 會按檢查點確認成果 |
| **問題處理** | 遇到報錯不慌張 | 會用 AI 診斷和修復 |

這套工作流是可遷移的——做其他項目時，同樣適用。

### 第二層：AI 協作技能

你建立了與 AI 高效協作的能力：

| 技能 | 之前的你 | 現在的你 |
|------|---------|---------|
| **Prompt 編寫** | 不知道怎麼描述需求 | 會用結構化框架組織信息 |
| **迭代追問** | AI 沒做對就不知道怎麼辦 | 會精確反饋，逐步逼近目標 |
| **信息提供** | 不知道 AI 需要什麼上下文 | 知道背景、任務、約束的重要性 |
| **Debug 協作** | 看到報錯就慌 | 會複製報錯信息讓 AI 幫忙修復 |
| **結果驗證** | 盲目相信 AI 輸出 | 會用檢查點確認功能正確 |

這些技能的核心是：**你學會了"指揮" AI，而不是"等待" AI**。

### 第三層：心態轉變

更重要的是，你完成了一次心態上的轉變：

| 維度 | 轉變前 | 轉變後 |
|------|-------|-------|
| **身份認知** | "我不會編程" | "我能用 AI 做出東西" |
| **對報錯的態度** | 恐懼、覺得自己做錯了 | 平常心，知道可以修復 |
| **對"完美"的追求** | 想一次做到完美 | 先跑通再優化 |
| **對技術的態度** | 技術是門檻 | 技術是可以藉助的工具 |

這種心態轉變，比任何具體技能都重要。它讓你從"想學編程的人"變成了"能做東西的人"。

## 與第一章的對比：你的成長

還記得第一章的 Hello World 嗎？讓我們對比一下那時和現在：

| 維度 | 第一章 Hello World | 第四章待辦清單 |
|------|-------------------|---------------|
| **功能複雜度** | 只顯示文字 | 增刪改查 + 數據存儲 |
| **交互性** | 無交互 | 響應用戶操作 |
| **數據持久化** | 無 | localStorage |
| **開發時間** | 3 分鐘 | 約 2 小時 |
| **Prompt 數量** | 1 條 | 10-15 條 |
| **用到的思維工具** | 無 | 靈魂三問、MVP、S.C.A.F.F. 等 |
| **實用價值** | 演示用 | 真正能用 |

從 3 分鐘的 Hello World 到 2 小時的完整應用，你走過的不只是時間，更是能力的躍升。

| 項目類型 | 複雜度 | 示例 |

- 用同樣的 HTML + CSS + JavaScript 就能實現
