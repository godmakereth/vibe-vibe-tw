# 第 4 章：實戰 —— 從 0 到 1 開發個人工具

> **章節定位**：本章是 Vibe Coding 的核心實踐章節，將前三章學到的思維和技法真正落地，手把手帶你完成第一個完整的 AI 編程項目。

## 從"紙上談兵"到"真刀真槍"

在前三章，我們完成了重要的準備工作：

| 章節 | 你學到了什麼 | 比喻 |
|------|-------------|------|
| 第一章 覺醒 | 理解 Vibe Coding 是什麼 | 認識新世界 |
| 第二章 心法 | 用產品經理思維想清楚要做什麼 | 寫好劇本 |
| 第三章 技法 | 用結構化方法與 AI 高效溝通 | 學會導演的語言 |
| **第四章 實戰** | **真正動手做出一個產品** | **開拍！** |

本章，我們要把前面學的全部用起來——**做一個真正能用的東西**。

不是看別人做，不是想象着做，而是你自己親手做出來。

## 爲什麼選擇「待辦清單」作爲第一個項目

你可能會問：爲什麼不做個更酷炫的東西？比如 AI 聊天機器人、圖片生成器？

我們選擇「個人待辦清單」，有三個重要原因：

### 1. 包含完整的 CRUD 操作

| 操作 | 英文 | 在待辦清單中的體現 |
|------|------|-------------------|
| 創建 | Create | 添加新任務 |
| 讀取 | Read | 顯示任務列表 |
| 更新 | Update | 標記任務完成 |
| 刪除 | Delete | 刪除任務 |

這四個操作是幾乎所有應用的基礎。學會了這個，做其他項目就是換個皮。

### 2. 與第二章案例無縫銜接

還記得第二章的小李嗎？他想做一個待辦清單來管理自己的任務。我們在第二章幫他做了：
- 靈魂三問分析
- 用戶畫像
- MVP 功能定義
- 簡易 PRD

現在，我們要把小李的想法**真正做出來**。

### 3. 做完真的能用

這不是一個"練習項目"——做完後，你真的可以用它來管理自己的待辦事項。刷新頁面數據還在，手機上也能打開。這種"我做的東西真的能用"的成就感，是學習編程最大的動力。

## 本章學習目標

完成本章學習後，你將：

- ✅ 獨立完成一個包含增刪改查功能的完整網頁應用
- ✅ 掌握從想法到產品的完整工作流程
- ✅ 學會使用 AI IDE 進行開發
- ✅ 能夠在 AI 代碼出錯時進行調試和修復
