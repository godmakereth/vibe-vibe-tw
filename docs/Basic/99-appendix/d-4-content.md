# D.4 部署平臺對比

做好的東西要放到網上讓別人能訪問，這就是「部署」。以下是主流免費部署平臺的對比。


## 綜合對比表

| 平臺 | 免費額度 | 上手難度 | 適合場景 | 特色 |
|------|----------|----------|----------|------|
| **Vercel** | 慷慨 | 簡單 | 前端/全棧 | 最流行，和 Next.js 完美配合 |
| **Netlify** | 慷慨 | 簡單 | 靜態網站 | 功能豐富，表單功能好用 |
| **GitHub Pages** | 免費 | 簡單 | 靜態網站 | 完全免費，和 GitHub 綁定 |
| **Railway** | 有限 | 中等 | 後端服務 | 適合有後端的應用 |
| **Cloudflare Pages** | 慷慨 | 簡單 | 靜態網站 | 全球 CDN，速度快 |


## 各平臺詳解

### Vercel

**官網**：vercel.com

**簡介**：最流行的前端部署平臺，Next.js 的官方部署平臺。

**免費額度**：
- 無限靜態網站
- 每月 100GB 流量
- Serverless 函數有限制

**優點**：
- 部署超級簡單（連接 GitHub 後自動部署）
- 預覽功能好用（每個 PR 都有預覽鏈接）
- 和 Next.js 無縫配合
- 全球 CDN，訪問快

**缺點**：
- Serverless 函數有執行時間限制
- 國內訪問可能不穩定

**適合**：前端項目、Next.js 項目


### Netlify

**官網**：netlify.com

**簡介**：功能豐富的靜態網站部署平臺。

**免費額度**：
- 每月 100GB 流量
- 每月 300 分鐘構建時間
- 表單提交 100 次/月

**優點**：
- 功能豐富（表單、身份認證、函數）
- 界面友好
- 插件生態好

**缺點**：
- 免費版功能有限制
- 國內訪問可能不穩定

**適合**：靜態網站、博客、文檔網站


### GitHub Pages

**官網**：pages.github.com

**簡介**：GitHub 提供的免費靜態網站託管服務。

**免費額度**：
- 完全免費
- 每個倉庫 1GB 存儲
- 每月 100GB 流量

**優點**：
- 完全免費
- 和 GitHub 倉庫直接綁定
- 穩定可靠

**缺點**：
- 只能部署靜態網站
- 沒有 Serverless 函數
- 構建功能有限

**適合**：個人博客、項目文檔、簡單靜態網站


### Railway

**官網**：railway.app

**簡介**：適合部署有後端服務的應用。

**免費額度**：
- 每月 $5 免費額度
- 用完需付費

**優點**：
- 支持後端服務（Node.js、Python 等）
- 支持數據庫
- 部署簡單

**缺點**：
- 免費額度較少
- 用完就要付費

**適合**：需要後端的完整應用


### Cloudflare Pages

**官網**：pages.cloudflare.com

**簡介**：Cloudflare 推出的靜態網站託管服務。

**免費額度**：
- 無限請求
- 無限流量
- 每月 500 次構建

**優點**：
- 免費額度非常慷慨
- 全球 CDN，速度極快
- 和 Cloudflare 其他服務整合好

**缺點**：
- 功能相對簡單
- 學習曲線略陡

**適合**：追求訪問速度的靜態網站


## 選擇建議

| 你的情況 | 推薦平臺 |
|----------|----------|
| 第一次部署，想簡單 | Vercel |
| 用 Next.js | Vercel |
| 純靜態網站 | GitHub Pages / Netlify |
| 有後端需求 | Railway |
| 追求速度 | Cloudflare Pages |
| 國內訪問穩定 | 考慮國內雲服務（阿里雲、騰訊雲） |


## 部署流程概覽

```mermaid
flowchart LR
    A[本地代碼] --> B[推送到 GitHub]
    B --> C[連接部署平臺]
    C --> D[自動構建]
    D --> E[上線！]
    E --> F[獲得網址]
```

基本流程：
1. 把代碼推送到 GitHub
2. 在部署平臺連接 GitHub 倉庫
3. 平臺自動檢測、構建、部署
4. 獲得一個可訪問的網址

::: tip 新手建議
第一次部署？選 **Vercel**。操作簡單，文檔齊全，社區活躍。遇到問題容易找到答案。
:::
