# C.1 AI 編程相關

遇到不認識的詞？來這裏查。每個術語都用最通俗的話解釋。


## Vibe Coding

**一句話解釋**：不寫代碼，用自然語言告訴 AI 你想要什麼，讓 AI 幫你生成代碼。

**類比**：就像你告訴裝修師傅「我想要一個溫馨的客廳」，而不是自己去搬磚砌牆。

**例子**：
- 你說：「做一個顯示今日名言的網頁，藍色背景，白色大字」
- AI：輸出完整的 HTML/CSS/JS 代碼


## Spec Coding

**一句話解釋**：先寫好詳細的規範文檔，再讓 AI 按規範生成代碼。

**類比**：就像蓋房子前先畫好圖紙，每個房間多大、門窗在哪都寫清楚。

**和 Vibe Coding 的區別**：
| | Vibe Coding | Spec Coding |
|---|---|---|
| 文檔 | 不需要 | 需要詳細文檔 |
| 適合 | 簡單項目、快速驗證 | 複雜項目、團隊協作 |
| 特點 | 快、靈活 | 穩、可維護 |


## Prompt

**一句話解釋**：你給 AI 的指令或問題。

**類比**：就像你給外賣小哥的備註：「少鹽少油，多放辣」。Prompt 寫得越清楚，AI 做出來的東西越符合你的預期。

**好 Prompt 的特點**：
- 說清楚要什麼（目標）
- 說清楚給誰用（用戶）
- 說清楚不要什麼（限制）


## Context（上下文）

**一句話解釋**：AI 理解你需求所需要的背景信息。

**類比**：就像醫生看病需要知道你的病史。你告訴 AI 越多背景信息，它就越能給出準確的答案。

**Context 包括什麼**：
- 你之前說過的話
- 你提供的代碼
- 你描述的項目背景
- 你使用的技術棧


## Context Window（上下文窗口）

**一句話解釋**：AI 一次能「記住」的信息量上限。

**類比**：就像人的短期記憶容量。聊得太長，AI 會「忘記」最開始說的內容。

**實際影響**：
- 對話太長時，可能需要重新說明背景
- 一次給太多代碼，AI 可能處理不過來
- 不同的 AI 工具，窗口大小不同


## Hallucination（幻覺）

**一句話解釋**：AI 一本正經地胡說八道，編造不存在的信息。

**類比**：就像一個太想幫忙的朋友，不知道答案也要硬編一個。

**常見表現**：
- 編造不存在的庫或函數名
- 給出「看起來對但實際不能運行」的代碼
- 引用不存在的文檔或鏈接

**如何應對**：
- 對 AI 的輸出保持懷疑，重要信息要驗證
- 讓 AI 解釋它的代碼，檢查邏輯是否說得通
- 直接運行代碼看看能不能跑


## Agent（智能體）

**一句話解釋**：能自主完成多步驟任務的 AI 系統。

**類比**：普通 AI 像計算器（你問一句答一句），Agent 像助理（你給個任務，它自己規劃步驟完成）。

**例子**：
- 你說：「幫我做一個待辦清單應用」
- Agent：自己分析需求 → 設計結構 → 寫代碼 → 測試 → 修復問題


## MCP（Model Context Protocol）

**一句話解釋**：讓 AI 能連接外部工具和數據的標準協議。

**類比**：就像 USB 接口標準。有了統一標準，AI 就能方便地「插上」各種工具使用。

**實際用途**：
- 讓 AI 能讀寫本地文件
- 讓 AI 能訪問數據庫
- 讓 AI 能調用 API


## Token

**一句話解釋**：AI 處理文本的基本單位，大約等於一個單詞或幾個漢字。

**類比**：就像話費的「分鐘」。你和 AI 聊天消耗的是 Token，用完了要充值。

**關於費用**：
- 大多數工具按 Token 收費
- 輸入和輸出都消耗 Token
- 漢字通常比英文消耗更多 Token

**省 Token 的小技巧**：
- 描述儘量簡潔但不模糊
- 不要重複已經說過的信息
- 一次對話解決一個問題
