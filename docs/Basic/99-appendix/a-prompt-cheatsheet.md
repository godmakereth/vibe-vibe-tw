# A. Prompt 速查清單

寫 Prompt 時的快速參考。詳細講解見[第三章：技法](/Basic/03-technique/)。


## S.C.A.F.F. 框架速查

| 要素 | 含義 | 一句話提醒 |
|------|------|-----------|
| **S** - Situation | 背景情境 | 你在做什麼項目？用什麼技術？ |
| **C** - Challenge | 具體挑戰 | 現在卡在哪裏？要解決什麼問題？ |
| **A** - Ask | 明確請求 | 你希望 AI 做什麼？輸出什麼？ |
| **F** - Format | 輸出格式 | 要代碼？要解釋？要列表？ |
| **F** - Filter | 限制條件 | 什麼不要做？有什麼約束？ |


## 常見場景要點

### 創建項目
- 說清楚：做什麼、給誰用、核心功能（3個以內）
- 先從最簡單的版本開始

### 修改代碼
- 指明位置：哪個文件、哪個函數
- 說清楚：現在是什麼樣、想改成什麼樣
- 加一句：「只改這裏，其他不動」

### 修復報錯
- 複製完整錯誤信息（比「報錯了」有用100倍）
- 說明：做什麼操作時出的錯

### 解釋代碼
- 直接貼代碼，問「這段代碼在做什麼」
- 可以追問：「爲什麼要這樣寫」

### 優化代碼
- 說明優化目標：可讀性？性能？減少重複？
- 讓 AI 解釋改了什麼


## 提升效果的小技巧

| 技巧 | 適用場景 |
|------|----------|
| 給 1-2 個示例 | AI 總是理解錯你的意圖時 |
| 讓 AI「想一想再回答」 | 複雜邏輯、多步驟任務 |
| 讓 AI 自我檢查 | 生成的代碼不太放心時 |
| 分步驟提問 | 一次問太多 AI 容易亂 |


## 記住這一條

> **Prompt 的核心是「說清楚」，不是「套模板」。**
>
> 學會框架後，用自己的話表達就好。
