# C.2 開發相關

這些是軟件開發中常用的術語。不需要深入理解，知道大概意思就行。


## MVP

**全稱**：Minimum Viable Product（最小可行產品）

**一句話解釋**：能用的最簡版本，只包含最核心的功能。

**類比**：先做一個能騎的自行車，而不是一上來就想造特斯拉。

**爲什麼重要**：
- 快速驗證想法是否可行
- 及早發現問題
- 避免在沒人要的功能上浪費時間


## PRD

**全稱**：Product Requirement Document（產品需求文檔）

**一句話解釋**：描述產品「做什麼」的文檔。

**通常包括**：
- 產品目標
- 用戶是誰
- 核心功能
- 不包含什麼

**對 AI 編程的價值**：
有了 PRD，AI 能更準確地理解你的需求，生成的代碼更符合預期。


## Debug

**一句話解釋**：找出並修復代碼中的錯誤。

**類比**：就像找出燈泡不亮的原因——是燈泡壞了？還是開關沒開？還是停電了？

**常見 Debug 方法**：
- 看錯誤信息
- 加 `console.log` 打印變量值
- 一行一行檢查代碼
- 問 AI


## Deploy（部署）

**一句話解釋**：把你做的東西放到網上，讓別人能訪問。

**類比**：就像把你寫的書從手稿變成上架銷售的實體書。

**常用部署平臺**：
- Vercel
- Netlify
- GitHub Pages


## Git

**一句話解釋**：代碼版本管理工具，可以記錄每次修改，隨時回退。

**類比**：就像遊戲的存檔功能。做錯了可以讀檔回到之前的狀態。

**核心概念**：
| 概念 | 解釋 |
|------|------|
| Repository（倉庫） | 存放項目的地方 |
| Commit（提交） | 保存一次修改記錄 |
| Branch（分支） | 代碼的平行版本 |
| Push（推送） | 把本地代碼上傳到雲端 |
| Pull（拉取） | 把雲端代碼下載到本地 |


## GitHub

**一句話解釋**：最大的代碼託管平臺，基於 Git。

**類比**：就像「代碼界的百度網盤」，你的代碼存在雲端，隨時隨地可以訪問。

**常見用途**：
- 存儲和備份代碼
- 和別人協作開發
- 展示你的項目
- 找開源項目學習


## API

**全稱**：Application Programming Interface（應用程序接口）

**一句話解釋**：程序之間交流的「暗號」。

**類比**：就像餐廳的菜單。你按照菜單點菜（發請求），廚房就給你做（返回結果）。你不需要知道廚房怎麼做的。

**例子**：
- 天氣 API：你發請求「北京天氣」，它返回「晴，25°C」
- 地圖 API：你發請求「從 A 到 B」，它返回路線


## 框架

**一句話解釋**：別人寫好的代碼骨架，你在上面填東西就行。

**類比**：就像搭積木。框架給你提供基礎結構，你只需要往裏面放內容。

**常見框架**：
| 框架 | 用途 |
|------|------|
| React / Vue | 做網頁界面 |
| Next.js | 做完整網站 |
| Express | 做後端服務 |


## 前端 vs 後端

| | 前端 | 後端 |
|---|---|---|
| 是什麼 | 用戶看得見的部分 | 用戶看不見的部分 |
| 類比 | 餐廳的裝修和菜單 | 廚房和倉庫 |
| 負責 | 界面、交互、動畫 | 數據處理、存儲、邏輯 |
| 技術 | HTML/CSS/JS | Node.js/Python/數據庫 |


## 全棧

**一句話解釋**：前端後端都會。

**類比**：既能設計餐廳裝修，也能在廚房做菜。

**在 AI 時代**：
藉助 AI，普通人也能做出「全棧」應用——因爲 AI 可以幫你處理你不擅長的部分。
