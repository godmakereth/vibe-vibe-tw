# 5.3.7 本節要點總結

恭喜你完成了安全章節的學習。現在你已經掌握了 AI 編程時代最重要的安全知識。

## 三大禁令

記住這三條底線，可以避免 90% 的安全問題：

| 禁令 | 說明 |
|------|------|
| ❌ 不告訴 AI 密碼/密鑰 | 即使是"幫我檢查格式對不對"也不行 |
| ❌ 不把密鑰寫在代碼裏 | 使用 .env 文件和環境變量 |
| ❌ 不上傳 .env 文件 | 確保 .gitignore 配置正確 |

## 兩個好習慣

1. **安裝新庫前先驗證**：去 npmjs.com 或 pypi.org 確認包存在且可信
2. **提交前快速檢查**：搜索 `password`、`secret`、`key` 等關鍵詞

## 一個核心原則

::: tip 核心原則
**AI 生成的代碼，最終責任在你。**

保持審查習慣，不盲目信任 AI 的輸出。
:::
