# 5.3.0 2025年安全現狀：觸目驚心的數字

> 這些不是危言聳聽，而是 2024-2025 年權威機構發佈的真實數據。

經過本節學習，你將瞭解：
- AI 生成代碼的安全風險有多普遍
- 密鑰泄露問題有多嚴重
- 爲什麼現在就要重視安全

## AI 生成代碼的安全風險

來源：Veracode 2025 GenAI Code Security Report

Veracode 在 2025 年測試了超過 100 個大語言模型，分析它們生成的 Java、Python、JavaScript、C# 代碼。結果令人擔憂：

| 數據 | 含義 |
|------|------|
| **45%** | AI 生成的代碼未通過安全測試 |
| **72%** | Java 代碼的安全失敗率（所有語言中最高） |
| **86%** | AI 未能防禦跨站腳本攻擊的比例 |
| **88%** | AI 未能防禦日誌注入攻擊的比例 |

更值得注意的是：**更新、更大的模型並沒有產生更安全的代碼**。安全性能在過去幾年幾乎沒有改善。

## 密鑰泄露危機

來源：GitHub 2025、GitGuardian 2025、Verizon DBIR 2025

| 數據 | 來源 | 含義 |
|------|------|------|
| **3900 萬** | GitHub 2025 | 2024 年在 GitHub 上檢測到的泄露密鑰數量 |
| **2380 萬** | GitGuardian 2025 | 2024 年公開倉庫新增的泄露憑證（同比增長 25%） |
| **70%** | GitGuardian 2025 | 2022 年泄露的密鑰至今仍然有效 |
| **35%** | GitGuardian 2025 | 私有倉庫包含明文密鑰的比例 |
| **94 天** | Verizon DBIR 2025 | 泄露密鑰的平均修復時間 |

這意味着：攻擊者平均有 **3 個月的時間窗口** 來利用泄露的密鑰。

## 頂級公司也難逃一劫

| 數據 | 含義 |
|------|------|
| **65%** | Forbes AI 50 榜單中的頂級 AI 公司曾在 GitHub 上泄露過憑證 |
| **$488 萬** | 涉及憑證泄露的數據泄露事件平均成本（IBM 2024） |

如果連這些技術最先進的公司都會犯錯，普通開發者更需要警惕。

## 新興威脅：包幻覺攻擊

來源：德克薩斯大學、俄克拉荷馬大學、弗吉尼亞理工聯合研究（2025）

| 數據 | 含義 |
|------|------|
| **19.7%** | AI 推薦的軟件包實際上不存在 |
| **21.7%** | 開源模型的包幻覺率 |
| **5.2%** | GPT 系列模型的包幻覺率（相對較低） |
| **58%** | 幻覺包名會重複出現，可被攻擊者預測 |

攻擊者已經開始利用這個漏洞：註冊 AI 常"幻覺"出來的包名，植入惡意代碼，等待開發者上鉤。

## 這些數字意味着什麼

::: warning 核心信息
這些數字不是爲了嚇你，而是告訴你一個事實：

**AI 編程的安全風險是真實存在的，不是"以後再說"的事情。**

但好消息是：只要瞭解規則，大部分風險都可以輕鬆避免。
:::
