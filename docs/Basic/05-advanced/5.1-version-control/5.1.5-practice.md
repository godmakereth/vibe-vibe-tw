# 5.1.5 GitHub Desktop 實操：10 分鐘上手

本節帶你一步步完成 GitHub Desktop 的安裝和基本操作。

## Step 1：下載安裝

訪問 GitHub Desktop 官網下載安裝包：

**官網地址**：[desktop.github.com](https://desktop.github.com)

- 根據你的操作系統（Windows/Mac）下載對應版本
- 下載完成後，雙擊安裝包，按提示完成安裝

**你應該看到**：安裝完成後，GitHub Desktop 的圖標出現在你的應用列表中。

::: warning 國內用戶提示
GitHub 在國內訪問可能較慢。如果下載速度很慢，可以嘗試：
1. 換個時間段下載（早晨通常較快）
2. 使用手機熱點
3. 搜索"GitHub Desktop 國內鏡像"

如果實在無法下載，可以暫時跳過這一節，先用 5.1.2 和 5.1.3 的方法。
:::

## Step 2：註冊/登錄 GitHub 賬號

打開 GitHub Desktop 後，它會提示你登錄。

**如果你沒有 GitHub 賬號**：
1. 點擊"Create your free account"或訪問 [github.com](https://github.com)
2. 填寫用戶名、郵箱、密碼
3. 完成郵箱驗證

**如果你已有賬號**：
1. 點擊"Sign in to GitHub.com"
2. 在彈出的瀏覽器頁面中登錄
3. 授權 GitHub Desktop 訪問你的賬號

**你應該看到**：登錄成功後，界面顯示"Let's get started!"或類似的歡迎頁面。

## Step 3：把待辦清單變成倉庫

現在我們要把第四章做的待辦清單項目變成一個 Git 倉庫。

### 方式一：從現有文件夾創建

1. 點擊 **"Create a New Repository on your Hard Drive"** 或 **"Add an Existing Repository"**

2. 如果選擇"Create New"：
   - **Name**：輸入倉庫名，如 `my-todo-list`
   - **Local Path**：選擇你的項目文件夾的**上級目錄**
   - 勾選 **"Initialize this repository with a README"**
   - 點擊 **"Create Repository"**

3. 把你的待辦清單文件複製到新創建的倉庫文件夾中

### 方式二：直接添加現有項目（推薦）

1. 點擊 **File** → **Add Local Repository**
2. 選擇你的待辦清單項目文件夾
3. 如果提示"這不是一個 Git 倉庫"，點擊 **"create a repository"**
4. 確認創建

**你應該看到**：左側顯示你的項目名稱，中間區域顯示文件變化。

## Step 4：第一次 Commit（存檔）

現在你的項目已經是一個 Git 倉庫了。來創建第一個存檔點。

1. 在 GitHub Desktop 主界面，左側會顯示所有變化的文件
2. 如果是新倉庫，所有文件都會顯示爲"新增"

3. 在左下角找到 **Summary** 輸入框
   - 輸入一句描述，比如："第四章待辦清單完成版"
   - Description 可以留空

4. 點擊藍色的 **"Commit to main"** 按鈕

**你應該看到**：
- 左側的文件列表清空了（因爲所有變化都已保存）
- 界面提示"No local changes"

恭喜！你創建了第一個存檔點。

## Step 5：查看歷史（History）

來看看剛纔創建的存檔。

1. 點擊頂部的 **"History"** 標籤（或在左側找到）
2. 你會看到一個時間線，每個節點就是一個 Commit

**你應該看到**：
- 剛纔的 Commit 出現在列表中
- 顯示你寫的描述（如"第四章待辦清單完成版"）
- 右側顯示這次 Commit 包含了哪些文件

## Step 6：模擬改崩然後回退

來測試一下"讀檔"功能。

### 6.1 先做一個改動

1. 用你的 AI IDE 或文本編輯器打開待辦清單的 HTML 文件
2. 隨便改點東西（比如把標題改成"測試回退"）
3. 保存文件

### 6.2 創建一個新的存檔

1. 回到 GitHub Desktop
2. 你會看到剛纔的改動顯示在 Changes 列表中
3. 寫一個描述："測試改動"
4. 點擊 **"Commit to main"**

### 6.3 回退到之前的版本

1. 點擊 **"History"** 標籤
2. 找到"第四章待辦清單完成版"那個 Commit
3. 右鍵點擊它
4. 選擇 **"Revert Changes in Commit"**（或 **"Revert this Commit"**）
5. 確認操作

**你應該看到**：
- 創建了一個新的 Commit，描述類似"Revert 測試改動"
- 打開 HTML 文件，標題恢復了原樣

## Step 7：推送到雲端（Push）

最後一步，把你的代碼上傳到 GitHub 網站。

1. 點擊頂部的 **"Publish repository"** 或 **"Push origin"** 按鈕
2. 在彈出的對話框中：
   - **Name**：確認倉庫名稱
   - **Description**：可以寫一句描述（可選）
   - **Keep this code private**：建議勾選（保持私有）
3. 點擊 **"Publish Repository"**

**你應該看到**：
- 按鈕變成"Fetch origin"
- 沒有報錯信息

### 驗證成功

1. 打開瀏覽器，訪問 [github.com](https://github.com)
2. 登錄你的賬號
3. 在你的個人頁面，應該能看到剛纔創建的倉庫
4. 點擊進去，能看到你的代碼文件

恭喜！你的代碼現在有了雲端備份。即使電腦壞了，也能從 GitHub 找回來。

## 常見問題

| 問題 | 可能原因 | 解決方法 |
|------|---------|---------|
| Push 失敗 | 網絡問題 | 檢查網絡連接，稍後重試 |
| 找不到項目文件夾 | 路徑問題 | 確認文件夾位置，重新添加 |
| 登錄失敗 | 賬號問題 | 在瀏覽器中先登錄 github.com 試試 |
| 操作很慢 | 網絡問題（國內常見） | 耐心等待，或換個時間 |
| 提示"not a git repository" | 還沒初始化 | 按 Step 3 創建倉庫 |
