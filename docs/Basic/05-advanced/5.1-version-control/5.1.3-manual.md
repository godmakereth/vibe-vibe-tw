# 5.1.3 手動版本管理：簡單但有效

最簡單的版本管理方法：**定期複製文件夾**。

這個方法不需要學習任何新工具，不需要聯網，人人都會。雖然"土"，但非常有效。

## 基本操作

1. 找到你的項目文件夾（比如 `todo-list`）
2. 複製整個文件夾
3. 給副本起個有意義的名字
4. 繼續在原文件夾裏開發

就這麼簡單。

## 推薦的命名規範

一個好的命名能讓你快速找到需要的版本。

### 格式建議

```
項目名-v版本號-功能描述/
```

或者加上日期：

```
項目名-v版本號-功能描述-日期/
```

### 待辦清單項目示例

```
todo-list/                          ← 當前開發的版本
todo-list-v1-基礎功能完成/            ← 第一個能用的版本
todo-list-v2-添加刪除功能/            ← 添加了刪除功能
todo-list-v3-數據保存/               ← 添加了 localStorage
todo-list-v4-深色模式嘗試/            ← 準備大改之前的備份
```

### 如果你更喜歡用日期

```
todo-list/
todo-list-20250107-基礎完成/
todo-list-20250108-刪除功能/
todo-list-20250108-數據保存/
```

## 什麼時候應該保存一個版本

不需要每改一行代碼就備份。以下幾個時機最重要：

| 時機 | 爲什麼 | 示例 |
|------|-------|------|
| **完成一個功能後** | 這個版本是確定能用的 | 添加任務功能做好了 |
| **準備做重大改動前** | 萬一改崩了可以回退 | 準備加深色模式 |
| **今天工作結束時** | 保護今天的成果 | 下班前備份一下 |
| **準備讓 AI 做大改動前** | AI 可能改錯地方 | 讓 AI 重構代碼 |

## 實操：給你的待辦清單創建第一個備份

現在就動手，給你的待辦清單項目創建一個備份。

### 步驟

1. **找到項目文件夾**
   - 打開你在第四章創建的項目文件夾
   - 通常在 `文檔/vibe-coding-projects/todo-list/` 這樣的位置

2. **複製文件夾**
   - Windows：右鍵 → 複製，然後右鍵 → 粘貼
   - Mac：選中文件夾，按 Command+C，然後 Command+V

3. **重命名副本**
   - 把副本命名爲：`todo-list-v1-第四章完成版`

4. **驗證**
   - 確認你現在有兩個文件夾
   - 原文件夾繼續開發
   - 副本作爲備份保存

### 成功驗證

```
vibe-coding-projects/
├── todo-list/                      ← 繼續在這裏開發
└── todo-list-v1-第四章完成版/        ← 備份，不要動它
```

## 這個方案的特點

**優點**：

- ✅ 簡單直觀，人人都會
- ✅ 不需要學習新工具
- ✅ 不需要聯網
- ✅ 備份完整，想恢復直接複製回去

**侷限性**：

- ❌ 佔用磁盤空間（項目大的話會很佔地方）
- ❌ 難以對比兩個版本的差異
- ❌ 容易忘記備份
- ❌ 無法雲端同步

## 與 AI IDE 歷史功能配合使用

推薦的組合方式：

| 場景 | 使用方式 |
|------|---------|
| 日常小改動 | 依賴 AI IDE 自帶歷史 |
| 完成重要功能 | 手動複製一個備份 |
| 準備大改之前 | 手動複製一個備份 |
| 項目完成階段性目標 | 手動複製一個備份 |

這樣既不會備份太頻繁，又能保護重要的版本。

::: tip 核心觀點
手動備份是最後的保險。即使你學會了更高級的工具，偶爾手動備份一下也是好習慣。
:::
