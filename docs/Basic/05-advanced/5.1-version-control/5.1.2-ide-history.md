# 5.1.2 零門檻方案：AI IDE 自帶歷史功能

好消息：大多數 AI 編程工具已經內置了版本歷史功能。

你不需要安裝任何新軟件，不需要學習新概念，只需要知道在哪裏找到它。

## AI 代碼編輯器（需安裝類）

如果你使用的是 Cursor、Windsurf、Trae 等需要安裝的 AI IDE，它們都有歷史恢復功能。

### Cursor

Cursor 基於 VS Code，繼承了它的 Local History 和 Timeline 功能。

**如何找到歷史**：
1. 在左側 Explorer 面板底部，找到 **Timeline** 區域
2. 點擊展開，可以看到當前文件的所有歷史版本
3. 每次保存都會自動記錄一個版本

**如何回退**：
1. 在 Timeline 中找到想要恢復的時間點
2. 右鍵點擊該版本
3. 選擇"Restore Contents"恢復文件內容

**另一種方式**：
- 右鍵點擊文件 → 選擇 **Local History** → **Find Local History**
- 可以看到更詳細的歷史記錄

::: warning 注意事項
- Timeline 僅保存在本地，換電腦就沒了
- 歷史記錄有數量和時間限制
- 建議重要節點配合手動備份
:::

### Windsurf

Windsurf 有獨特的 **Cascade Checkpoints** 功能（2025 年 12 月更新）。

**如何找到歷史**：
1. 打開 Cascade 面板（通常在右側）
2. 找到 **Checkpoints** 或 **Named Checkpoints** 選項
3. 可以查看之前保存的檢查點

**如何回退**：
1. 選擇想要恢復的檢查點
2. 點擊 **Revert** 按鈕
3. 代碼會回到該檢查點的狀態

**特色功能**：
- 可以給檢查點命名，方便識別
- 支持在不同檢查點之間快速切換

### Trae

Trae（包括國際版和國內版）提供 **Diff View** 功能來查看代碼變更。

**如何找到歷史**：
1. 使用 **Diff View** 工具查看代碼變更歷史
2. 可以看到每次 AI 修改的內容對比

**如何回退**：
1. 在 Diff View 中查看具體改動
2. 手動撤銷不需要的改動
3. 或使用 IDE 的 Undo 功能

::: tip 建議
Trae 的歷史功能相對基礎，建議配合手動備份使用（見 5.1.3）。
:::

### Kiro

Kiro 在 2025 年 11 月推出了 **Checkpointing** 功能。

**如何找到歷史**：
1. 每次 Kiro 修改代碼時，會自動創建檢查點
2. 在界面中可以看到檢查點列表

**如何回退**：
1. 找到想要恢復的檢查點
2. 一鍵點擊即可回退

**注意事項**：
- Checkpointing 只恢復當前會話中 Kiro 做的改動
- 不會保存整個代碼庫的狀態
- 適合快速撤銷最近的 AI 修改

## 網頁版 AI 編程（零安裝類）

網頁版工具的歷史功能通常更完善，因爲數據保存在雲端。

### 豆包 AI 編程

國內用戶首選，訪問流暢。

**如何找到歷史**：
1. 在項目界面中查找版本歷史入口
2. 可以查看之前的項目狀態

**如何回退**：
1. 選擇需要恢復的版本
2. 確認恢復操作

**優勢**：
- 國內訪問速度快
- 中文界面友好
- 完全免費

### Bolt.new

Bolt.new 有完善的 **Version History** 功能（2025 年 11 月更新）。

**如何找到歷史**：
1. 在側邊欄找到 **版本歷史** 或 **History** 選項
2. 可以看到項目的所有歷史版本
3. 版本按時間順序排列

**如何回退**：
1. 找到想要恢復的版本
2. 點擊 **Restore** 按鈕
3. 項目會恢復到該版本狀態

**特色功能**：
- **Bookmark**：可以給重要版本添加書籤，方便快速找到
- 支持查看每個版本的具體改動
- 版本歷史保存在雲端，換設備也能訪問

### v0.dev

v0.dev 在 2025 年 2 月更新了版本管理功能。

**如何找到歷史**：
1. 在編輯器內可以看到版本列表
2. 每次生成都會創建新版本

**如何回退**：
1. 點擊想要恢復的版本
2. 選擇切換或恢復
3. 可以在不同版本之間快速切換

**特點**：
- UI 組件質量高
- 版本切換流暢

### Lovable

Lovable 在 2025 年 3 月推出了 **Versioning 2.0**。

**如何找到歷史**：
1. 打開 **History** 面板
2. 編輯記錄按日期組織，類似 Google Docs
3. 界面直觀，容易找到想要的版本

**如何回退**：
1. 找到想要恢復的編輯
2. 點擊恢復按鈕
3. 系統會創建一個新的編輯卡片（不會覆蓋當前版本）

**特色功能**：
- **Bookmarks**：給重要版本添加書籤
- **智能恢復**：恢復時創建新版本，方便追蹤
- **Dev Mode**：可以直接在 Lovable 編輯器中查看和編輯代碼

### Replit

Replit 的歷史功能非常完善（2025 年 5 月更新）。

**如何找到歷史**：
1. 打開任意文件
2. 點擊編輯器右下角的 **History** 按鈕
3. 可以看到該文件的所有歷史版本

**如何回退**：
1. 在歷史面板中選擇版本
2. 可以預覽該版本的內容
3. 確認後恢復

**特色功能**：
- **File History**：單個文件的歷史
- **App History**：整個應用的歷史（包括數據庫狀態）
- **Checkpoints**：可以手動創建檢查點

## 實操任務：用你的待辦清單測試

現在，用你在第四章做的待辦清單項目來測試歷史功能。

### 測試步驟

1. **打開你的待辦清單項目**

2. **做一個小改動**
   - 比如修改標題文字
   - 或者改變按鈕顏色

3. **保存文件**

4. **找到歷史功能**
   - 根據你使用的工具，按照上面的指引找到歷史入口

5. **查看歷史版本**
   - 你應該能看到改動之前的版本

6. **嘗試恢復**
   - 選擇之前的版本
   - 確認恢復操作
   - 檢查代碼是否恢復

### 成功驗證

- [ ] 找到了歷史功能入口
- [ ] 能看到之前的版本列表
- [ ] 成功恢復到了之前的版本
- [ ] 代碼確實回到了改動之前的狀態

如果以上都完成了，恭喜你——你已經學會了最基本的"存檔讀檔"。

## 這個方案的侷限性

雖然 AI IDE 自帶的歷史功能很方便，但它有一些侷限：

| 侷限 | 說明 |
|------|------|
| **僅本地保存**（安裝類工具） | 換電腦、重裝系統，歷史就沒了 |
| **歷史有限** | 通常只保留最近的 N 個版本或 N 天內的版本 |
| **無法雲端同步**（安裝類工具） | 不能在多臺設備間同步 |
| **不夠精細** | 難以對比兩個版本的具體差異 |

對於個人小項目，這些侷限通常不是問題。但如果你想要更可靠的保護，可以結合下一節的手動備份方法。

::: tip 核心觀點
先用好工具自帶的功能，夠用就行。不需要一開始就追求完美的解決方案。
:::
