# 5.1 後悔藥：代碼改崩了怎麼辦

還記得你在第四章做的待辦清單嗎？那是你的第一個真正能用的作品。

現在，我們要給它裝上"時光機"。

## 爲什麼需要版本管理

你辛苦做了 2 小時的項目，想加個新功能，結果改着改着整個頁面白屏了。更慘的是：你不記得改了哪裏，想改回去也改不回去。

這種事情，幾乎每個學習者都會遇到。

**版本管理就是給你的代碼裝上"時光機"**——隨時可以回到過去的某個狀態，就像遊戲裏的存檔點。

## 三級方案：從零門檻到進階

根據你的需求和時間，我們提供三級方案：

| 級別 | 方案 | 適合誰 | 學習時間 |
|------|------|--------|---------|
| **零門檻** | AI IDE 自帶歷史 | 所有人 | 5 分鐘 |
| **簡單** | 手動複製文件夾 | 想要額外保險的人 | 3 分鐘 |
| **進階** | GitHub Desktop | 想要雲端備份、爲進階版打基礎 | 15 分鐘 |

建議：**至少掌握零門檻方案**。如果時間充裕，GitHub Desktop 會讓你受益更多。

## 學完本節，你將掌握

- 使用 AI IDE 自帶的歷史功能回退代碼
- 用手動方式保存項目的重要版本
- 使用 GitHub Desktop 進行版本管理（可選）
- 把項目上傳到 GitHub 雲端備份（可選）

::: tip 核心觀點
版本管理不是高級技能，是基本生存技能。就像寫文檔要隨時保存一樣，寫代碼也需要隨時"存檔"。
:::
