# 5.1.4 進階方案：GitHub Desktop 入門

如果你想要更專業的版本管理，**GitHub Desktop** 是一個很好的起點。

它是一個圖形界面工具，讓你不用敲命令就能使用 Git——程序員最常用的版本管理工具。

## 先理解三個概念

在開始之前，我們先搞清楚三個經常被混淆的概念：

| 概念 | 是什麼 | 比喻 |
|------|-------|------|
| **Git** | 一個版本管理工具 | 時光機的核心引擎 |
| **GitHub** | 一個在線代碼託管平臺 | 雲端的代碼倉庫 |
| **GitHub Desktop** | Git 的圖形界面客戶端 | 時光機的遙控器 |

簡單來說：
- Git 是技術
- GitHub 是網站
- GitHub Desktop 是讓你用鼠標操作 Git 的軟件

你不需要理解 Git 的所有功能。只需要知道：**GitHub Desktop 能幫你存檔、回退、雲端備份**，就夠了。

## 爲什麼選擇 GitHub Desktop

| 優勢 | 說明 |
|------|------|
| **圖形界面** | 點擊操作，不用記命令 |
| **免費** | 完全免費使用 |
| **雲端備份** | 代碼保存在 GitHub，換電腦也能找回 |
| **業界標準** | Git 是程序員的標配技能 |
| **爲進階版打基礎** | 進階版會學習更多 Git 技巧 |

## 核心概念（用生活類比）

GitHub Desktop 裏有一些專業術語，但其實很好理解：

| Git 術語 | 生活類比 | 一句話解釋 |
|---------|---------|-----------|
| **Repository（倉庫）** | 帶版本追蹤的文件夾 | 你的項目存放地，每次改動都會被記錄 |
| **Commit（提交）** | 存檔點 | 保存當前狀態，配上一句描述 |
| **Push（推送）** | 上傳到雲端 | 把本地的存檔同步到 GitHub 網站 |
| **Pull（拉取）** | 從雲端下載 | 把 GitHub 上的最新版本下載到本地 |
| **History（歷史）** | 存檔記錄 | 所有存檔點的列表 |
| **Revert（回退）** | 讀檔 | 把代碼恢復到某個存檔點 |

## 基本工作流程

使用 GitHub Desktop 的日常流程很簡單：

```
改代碼 → 保存文件 → Commit（存檔）→ Push（上傳雲端）
                         ↓
                可以隨時查看歷史
                可以隨時回退到任何存檔點
```

每次 Commit 就像在遊戲裏存了一個檔。你可以有很多個存檔，隨時回到任何一個。

## 學習成本

| 操作 | 難度 | 學習時間 |
|------|------|---------|
| 安裝軟件 | 簡單 | 2 分鐘 |
| 創建倉庫 | 簡單 | 2 分鐘 |
| 提交代碼（Commit） | 簡單 | 1 分鐘 |
| 查看歷史 | 簡單 | 1 分鐘 |
| 回退版本 | 簡單 | 1 分鐘 |
| 推送到雲端（Push） | 簡單 | 1 分鐘 |

總共只需要 10-15 分鐘，你就能掌握基本操作。

## 與進階版的銜接

本節介紹的 GitHub Desktop 是**圖形界面入門工具**，目的是讓你先體驗版本管理的好處。

::: info 進階版預告
在**進階版**中，你將學習：
- **Git 命令行**：更靈活、更強大的操作方式
- **分支管理**：同時開發多個功能，互不干擾
- **Pull Request**：團隊協作的標準工作流
- **衝突解決**：多人修改同一文件時怎麼處理

這些都是專業開發者的必備技能。但對於基礎版來說，GitHub Desktop 已經足夠。
:::

## 是否需要學這個

**如果你滿足以下任一條件，建議學**：
- 想要雲端備份，換電腦也能找回代碼
- 想爲後續學習打基礎
- 想體驗專業開發者的工作方式
- 有 15 分鐘的時間

**如果你暫時沒需求，可以跳過**：
- AI IDE 自帶歷史 + 手動備份已經夠用
- 以後需要時再學也來得及
