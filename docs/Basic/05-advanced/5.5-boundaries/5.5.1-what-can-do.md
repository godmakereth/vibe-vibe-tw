# 5.5.1 Vibe Coding 能做什麼

經過本節學習，你將瞭解：
- Vibe Coding 真正擅長的六大場景
- 2025 年 AI 編程的真實數據
- 爲什麼你走在正確的路上

## 先看數據：AI 編程已是主流

在討論 Vibe Coding 能做什麼之前，先看一組 2025 年的真實數據：

| 數據 | 來源 |
|------|------|
| **41%** 的代碼現在由 AI 生成或輔助 | Stack Overflow 2025 開發者調查 |
| **84%** 的開發者正在使用或計劃使用 AI 編程工具 | Index.dev 2025 報告 |
| **82%** 的開發者每週使用 AI 編程工具 | Second Talent 2025 統計 |
| **25%** 的新代碼由 AI 輔助完成 | Google 2025 官方數據 |

這些數字說明：**你學習 Vibe Coding，不是在追趕潮流，而是在擁抱已經到來的現實。**

## Vibe Coding 擅長的六大場景

### 場景一：個人工具

**爲什麼擅長**：需求由你自己定義，不需要考慮複雜的用戶權限、多人協作、海量數據。AI 能完整理解你的意圖。

| 項目類型 | 複雜度 | 成功率 |
|---------|--------|--------|
| 待辦清單 | 低 | 很高 |
| 習慣打卡器 | 低 | 很高 |
| 個人記賬本 | 中 | 高 |
| 讀書筆記管理 | 中 | 高 |
| 個人密碼管理（本地版） | 中 | 高 |

**真實案例**：你在第四章做的待辦清單，就是典型的個人工具。從零到能用，可能只花了一兩個小時。

### 場景二：原型驗證

**爲什麼擅長**：原型的目標是"快速驗證想法是否可行"，不需要完美，不需要能承載百萬用戶。Vibe Coding 的速度優勢在這裏發揮得淋漓盡致。

根據 Index.dev 2025 的報告，使用 AI 編程工具的開發者：
- 完成原型的速度提升 **30%-75%**
- GitHub Copilot 用戶每週完成的項目數量增加 **126%**

**典型場景**：
- 創業者驗證一個產品想法
- 學生爲課程項目做 Demo
- 向老闆或客戶演示一個概念

### 場景三：效率腳本

**爲什麼擅長**：腳本通常是"輸入 → 處理 → 輸出"的線性流程，邏輯清晰，沒有複雜的狀態管理。這正是 AI 最擅長的模式。

| 腳本類型 | 示例 |
|---------|------|
| 文件批處理 | 批量重命名、批量壓縮圖片、整理下載文件夾 |
| 數據轉換 | Excel 轉 JSON、CSV 合併、格式標準化 |
| 自動化任務 | 定時備份、自動發郵件提醒、爬取公開數據 |

**一個真實場景**：假設你每週要把 10 個 Excel 表格合併成一個彙總表。手動做需要 30 分鐘，用 Vibe Coding 寫個腳本可能只需要 10 分鐘，而且以後每週只需要運行一下，幾秒鐘搞定。

### 場景四：數據分析與可視化

**爲什麼擅長**：數據分析有明確的輸入（數據）和輸出（圖表/結論），中間的處理步驟可以清晰描述。AI 能很好地理解"我想看 XX 數據的 YY 趨勢"這類需求。

| 分析類型 | 適合程度 |
|---------|---------|
| 銷售數據趨勢分析 | 很適合 |
| 問卷調查結果可視化 | 很適合 |
| 個人消費分析 | 很適合 |
| 簡單的統計報告 | 很適合 |

### 場景五：靜態網站

**爲什麼擅長**：靜態網站沒有複雜的後端邏輯，主要是"展示內容"。HTML + CSS + 少量 JavaScript，是 AI 最熟練的領域之一。

| 網站類型 | 複雜度 | 成功率 |
|---------|--------|--------|
| 個人簡歷頁面 | 低 | 很高 |
| 作品集展示 | 低 | 很高 |
| 活動宣傳頁 | 低 | 很高 |
| 個人博客（靜態） | 中 | 高 |
| 小型企業官網 | 中 | 高 |

### 場景六：學習編程概念

**爲什麼擅長**：AI 可以用你能理解的語言解釋代碼，幫你建立直覺。你不需要先學會語法才能開始做東西，而是在做的過程中逐漸理解。

這正是 Vibe Coding 的核心價值：**降低入門門檻，讓"做中學"成爲可能。**

## 一個共同特徵：問題邊界清晰

如果你仔細看上面六個場景，會發現它們有一個共同特徵：

> **問題的邊界是清晰的。**

- 個人工具：你自己用，需求你說了算
- 原型驗證：只需要"能演示"，不需要"能上線"
- 效率腳本：輸入輸出明確
- 數據分析：數據是固定的，分析目標是明確的
- 靜態網站：內容是確定的，交互很簡單
- 學習概念：目標是"理解"，不是"生產"

當問題邊界清晰時，AI 能很好地理解你的意圖，生成的代碼大概率能直接用。

## 本節要點

- 2025 年，41% 的代碼由 AI 生成或輔助，AI 編程已是主流
- Vibe Coding 擅長六大場景：個人工具、原型驗證、效率腳本、數據分析、靜態網站、學習概念
- 這些場景的共同特徵是"問題邊界清晰"
- 在這些領域，大膽嘗試，快速出成果
