## 核心概念速查表

這張表彙總了第五章最重要的概念，可以收藏備用：

| 概念 | 一句話解釋 | 爲什麼重要 |
|------|-----------|-----------|
| **版本管理** | 給代碼裝上"時光機"，可以隨時回到過去 | 改崩了不怕，大膽實驗 |
| **Commit** | 一次"存檔"，記錄代碼在某個時刻的狀態 | 每個存檔都是一個安全點 |
| **部署** | 把本地代碼放到服務器上，讓別人能訪問 | 從"自己玩"到"分享給世界" |
| **域名** | 網站的"門牌號"，如 example.com | 讓別人能找到你的作品 |
| **.env 文件** | 存放密鑰等敏感信息的配置文件 | 保護你的隱私和賬戶安全 |
| **.gitignore** | 告訴 Git 哪些文件不要上傳 | 防止敏感信息泄露 |
| **Slopsquatting** | AI 推薦不存在的庫，被惡意者搶注 | 2024-2025 年新型安全威脅 |
| **迭代** | 根據反饋不斷改進產品 | 好產品是改出來的，不是一次做出來的 |
| **README** | 項目的說明文檔，介紹項目是什麼、怎麼用 | 讓別人（和未來的自己）看懂項目 |

## 常見誤區提醒

基於前面章節的學習，這裏提醒幾個容易踩的坑：

| 誤區 | 正確做法 |
|------|---------|
| "我要等代碼完美了再 Commit" | 半成品也可以 Commit，隨時可以改 |
| "部署好麻煩，先本地用着吧" | 部署只需要幾分鐘，現在就做 |
| "AI 生成的代碼應該沒問題" | 永遠審查，特別是涉及安全的部分 |
| "基礎版太簡單，我要直接學進階" | 基礎版的方法論是進階的根基 |
| "我做的東西太簡單，不好意思分享" | 能解決問題的就是好東西，大膽分享 |
