# 5.6.4 進入進階版的準備

經過本節閱讀，你將獲得：
- 基礎版畢業自測清單
- 判斷是否需要進階的 5 個信號
- 進階版詳細模塊預告
- 基礎版到進階版的技能升級路徑

## 自測：你準備好了嗎？

完成以下自測，每項打勾得 1 分：

### 核心能力（必備項）

- [ ] 我完成了至少一個完整的 Vibe Coding 項目
- [ ] 我能用 S.C.A.F.F. 框架寫出清晰的 Prompt
- [ ] 我理解版本管理的基本概念（Commit、Push）
- [ ] 我成功部署過項目到互聯網
- [ ] 我知道三大安全禁令

### 思維能力（加分項）

- [ ] 我能用靈魂三問定義一個項目的需求
- [ ] 我能區分"必須有"和"可以有"的功能
- [ ] 當 AI 給出錯誤答案時，我知道如何調整 Prompt
- [ ] 我養成了提交前檢查敏感信息的習慣
- [ ] 我清楚 Vibe Coding 的能力邊界

### 評分標準

| 得分 | 建議 |
|------|------|
| 8-10 分 | 你已經完全準備好了，可以直接進入進階版 |
| 5-7 分 | 建議再做 1-2 個基礎版項目鞏固，然後進入進階版 |
| 0-4 分 | 建議回顧第二到第五章，重點補足薄弱環節 |

## 需要進階的 5 個信號

如果你遇到以下情況，說明是時候學習進階版了：

### 信號 1：你想要用戶登錄功能

**場景**：你做了一個記賬本，想讓家人各自登錄，看到自己的數據。

**爲什麼基礎版做不到**：用戶登錄需要後端驗證身份，需要數據庫存儲賬號信息。localStorage 只能存在單個瀏覽器裏。

**進階版解決方案**：NextAuth 用戶認證 + 數據庫存儲

### 信號 2：你想要數據雲端同步

**場景**：你在公司電腦上記錄的待辦，回家想在自己電腦上繼續看。

**爲什麼基礎版做不到**：localStorage 是瀏覽器本地存儲，不同設備、不同瀏覽器之間無法共享。

**進階版解決方案**：後端 API + 數據庫 + 用戶認證

### 信號 3：你想做多人協作功能

**場景**：你想做一個家庭共享的購物清單，家人可以同時編輯。

**爲什麼基礎版做不到**：多人同時操作需要即時同步，需要處理數據衝突。

**進階版解決方案**：WebSocket 即時通信 + 衝突處理策略

### 信號 4：你對"代碼爲什麼能工作"產生好奇

**場景**：AI 給你的代碼能用，但你想知道爲什麼這樣寫。

**爲什麼基礎版沒講**：基礎版的目標是"能做出東西"，不深究原理。

**進階版解決方案**：Bootcamp 模塊系統講解計算機基礎、網絡原理、編程概念

### 信號 5：你想參與團隊項目

**場景**：你想和朋友一起開發一個項目，或者參與開源項目。

**爲什麼基礎版不夠**：多人協作需要分支管理、代碼審查、CI/CD 流程。

**進階版解決方案**：Git 進階工作流 + 團隊協作最佳實踐

## 不需要進階的場景

同樣重要的是，認清什麼時候**不需要**進階：

| 場景 | 基礎版已經夠用 |
|------|---------------|
| 做給自己用的個人工具 | localStorage 能滿足需求 |
| 靜態展示網站（個人主頁、簡歷） | 不需要後端 |
| 簡單的計算/轉換工具 | 純前端就能實現 |
| 學習和練習 AI 編程 | 基礎版方法論可複用 |
| 快速驗證一個想法 | MVP 不需要完整後端 |

**基礎版已經夠用的情況下，不必急於進階。** 用基礎版多做幾個項目，把方法論內化，比匆忙學習新技術更重要。

## 進階版模塊詳解

如果你決定進入進階版，以下是你將學習的內容：

### Bootcamp：計算機基礎

| 你將學到 | 解決什麼問題 |
|---------|-------------|
| 文件系統和路徑 | 理解項目結構，不再"找不到文件" |
| 命令行基礎 | 不怕黑窗口，能執行基本操作 |
| 網絡基礎（HTTP、DNS） | 理解網頁是如何加載的 |
| 前端三件套原理 | 知道 HTML/CSS/JS 各自的角色 |

### 技術棧：Next.js 全棧開發

| 技術 | 作用 | 爲什麼選它 |
|------|------|-----------|
| Next.js | 前後端一體化框架 | 上手快，生態好，部署方便 |
| TypeScript | 類型安全的 JavaScript | 減少錯誤，AI 生成代碼更可靠 |
| Prisma | 數據庫操作工具 | 不用寫 SQL，用代碼操作數據庫 |
| Tailwind CSS | 樣式框架 | 快速構建美觀界面 |

### 核心能力提升

| 基礎版技能 | 進階後升級爲 |
|-----------|-------------|
| localStorage 存儲 | 數據庫持久化（PostgreSQL/MySQL） |
| 靜態頁面部署 | 全棧應用部署（包含後端） |
| 個人使用工具 | 多用戶協作平臺 |
| 手動測試 | 自動化測試 + CI/CD |
| GitHub Desktop | Git 命令行 + 分支策略 |
| 基礎安全意識 | 完整的 Web 安全防護 |

### 進階版學習路徑預覽

```
進階版學習之旅
│
├── Bootcamp（2-3 天）
│   └── 補齊計算機基礎，爲後續學習打基礎
│
├── 第 1-2 章：概念與工具（3-4 天）
│   └── Next.js 項目搭建 + AI 協作工作流
│
├── 第 3-4 章：前後端開發（1-2 周）
│   └── 頁面開發 + API 設計 + 數據庫操作
│
├── 第 5-6 章：產品與安全（3-4 天）
│   └── PRD 文檔 + 用戶認證 + 安全防護
│
├── 第 7-9 章：工程實踐（1 周）
│   └── API 規範 + Git 協作 + 測試
│
├── 第 10-11 章：部署運維（3-4 天）
│   └── Docker + CI/CD + 監控
│
└── 第 12 章：高級專題（選學）
    └── SEO + 即時通信 + 文件處理
```

**預計總學習時間**：4-6 周（每天 2-3 小時）

## 從基礎版到進階版的平滑過渡

進階版會複用基礎版的核心方法論：

| 基礎版方法 | 進階版如何延續 |
|-----------|---------------|
| JTBD / MVP 思維 | 用於設計更復雜的功能 |
| S.C.A.F.F. 框架 | 繼續用於與 AI 溝通 |
| 迭代開發 | 用於全棧項目開發 |
| 安全三禁令 | 擴展爲完整的安全體系 |
| 版本管理 | 升級爲專業的 Git 工作流 |

**你在基礎版學到的一切都不會浪費。** 進階版是在這個基礎上的延伸和深化。

## 本節要點

- 用自測清單評估自己的準備程度
- 5 個信號告訴你是否需要進階
- 基礎版夠用時不必急於進階
- 進階版會複用基礎版的核心方法論
- 預計 4-6 周完成進階版學習

::: tip 最後的話
進階是一個選擇，不是必須。

如果基礎版已經能滿足你的需求，繼續用它創造價值。

如果你想突破邊界，進階版在等着你。
:::
