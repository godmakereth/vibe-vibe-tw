# 5.2.1 爲什麼要"部署上線"

在第四章，你親手做出了一個待辦清單。它能添加任務、刪除任務、標記完成，數據還能保存下來。這已經是一個真正可用的工具了。

但有一個問題：它只能在你的電腦上運行。

想讓朋友看看你做的東西？要麼把電腦遞給他，要麼錄個屏發過去。想在手機上用？做不到。想在公司電腦上用？也不行。

**部署**，就是把你的項目放到互聯網上，讓它擁有一個真正的網址。

## 部署是什麼

用一個類比來理解：

- 你在本地開發，就像在自己家裏做飯，只有家裏人能喫到
- 部署上線，就像開了一家餐廳，任何人都可以來品嚐

技術上說，部署就是把你的代碼放到一臺 24 小時運行的服務器上。但你不需要自己買服務器，現在有很多免費的平臺幫你做這件事。

## 不同人羣的使用場景

| 你是誰 | 部署後能做什麼 |
|--------|--------------|
| 學生 | 把課程作業部署上線，提交一個鏈接給老師，比截圖專業多了 |
| 職場人 | 做一個效率工具，在公司和家裏都能用，手機上也能用 |
| 給家人做工具 | 爸媽用手機掃個碼就能打開，不用教他們裝軟件 |
| 學習者 | 積累作品集，每個項目都有真實網址，求職時拿得出手 |
| 創業者 | 快速驗證想法，讓潛在用戶試用，收集真實反饋 |

## 部署後你將獲得

- 一個真正的網址（如 `my-todo.zeabur.app`）
- 任何設備都能訪問，包括手機、平板
- 可以分享給任何人，發個鏈接就行
- 24 小時在線，你關機了別人也能訪問
- 代碼更新後自動同步，不需要手動操作

## 部署有多簡單

你可能覺得"部署"聽起來很技術、很複雜。實際上，現在的部署平臺已經把流程簡化到了極致。

你需要做的只有三步：

1. 把代碼上傳到 GitHub（如果你跟着 5.1 做了，這一步已經完成）
2. 在部署平臺上登錄，選擇你的項目
3. 點擊部署，等待 1-3 分鐘

沒有服務器要配置，沒有複雜的命令要記，就是點幾下鼠標的事。

::: tip 你比想象中離成功更近
如果你已經完成了第四章的待辦清單，並且按照 5.1 把代碼上傳到了 GitHub，那麼你距離"讓全世界都能訪問你的作品"只差 5 分鐘。
:::
