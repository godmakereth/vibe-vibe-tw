# 5.2.9 本節檢查清單

完成本節學習後，逐項確認你已經掌握了以下內容。

## 知識檢查

- [ ] 理解了"部署"的含義：把代碼放到服務器上，讓所有人都能訪問
- [ ] 知道 Zeabur 和 Vercel 的區別，以及爲什麼大陸用戶建議用 Zeabur
- [ ] 瞭解了代碼更新後自動部署的工作原理

## 操作檢查

- [ ] 註冊了部署平臺賬號（Zeabur 或 Vercel）
- [ ] 成功部署了第四章的待辦清單項目
- [ ] 獲得了一個可公開訪問的網址
- [ ] 在瀏覽器中驗證了網站功能正常
- [ ] **在手機上驗證了可以正常訪問**（這一條很重要）

## 進階檢查（可選）

- [ ] 嘗試修改代碼，提交更新，驗證自動部署生效
- [ ] 把網址分享給朋友或家人，讓他們試用
- [ ] 瞭解了部署失敗時如何排查問題

## 本節核心要點

經過本節學習，你應該掌握了：

| 能力 | 具體表現 |
|------|---------|
| 部署網站 | 能獨立將本地項目部署到互聯網上 |
| 平臺選擇 | 知道根據自己的情況選擇合適的部署平臺 |
| 問題排查 | 遇到部署問題時知道如何查看日誌和求助 AI |
| 持續更新 | 理解 Push 即部署的工作流程 |

## 你現在擁有了什麼

完成本節後，你已經：

- 擁有了一個真正的網址（如 `my-todo.zeabur.app`）
- 可以讓任何人通過這個鏈接訪問你的作品
- 具備了將任何項目發佈到互聯網的能力
- 你的作品 24 小時在線，不需要開着電腦

這是一個了不起的里程碑。

從第四章到現在，你完成了：
- 從零開始做出一個待辦清單
- 把代碼上傳到 GitHub 保管
- 把網站發佈到互聯網上

你已經走完了從"想法"到"產品"的完整流程。
