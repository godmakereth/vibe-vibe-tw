# 5.2.5 部署後的實用操作

部署成功只是開始。你會持續改進你的項目，需要知道如何更新線上版本、如何查看部署狀態、以及出問題時如何回滾。

## 如何更新網站

這是最常用的操作。當你在本地修改了代碼，想讓線上版本也更新，你需要做的是：

```
本地修改代碼 → git add → git commit → git push → 自動部署
```

具體步驟：

1. 在本地修改你的代碼（比如改了一個按鈕的顏色）
2. 使用 GitHub Desktop 提交更改
3. 點擊"Push origin"推送到 GitHub
4. 等待 1-2 分鐘
5. 刷新你的網站，看到更新

**你不需要**重新登錄部署平臺，也不需要點擊任何部署按鈕。Zeabur 和 Vercel 都會自動檢測你的代碼更新，並自動重新部署。

這就是"持續部署"（Continuous Deployment）的魔力。

## 如何查看部署歷史

每次代碼更新後的部署都會留下記錄。你可以在部署平臺的控制檯中查看：

### 在 Zeabur 中

1. 登錄 Zeabur 控制檯
2. 點擊你的項目
3. 點擊你的服務
4. 找到"部署記錄"或"Deployments"選項卡

你可以看到：
- 每次部署的時間
- 部署是成功還是失敗
- 每次部署對應的 Git 提交

### 在 Vercel 中

1. 登錄 Vercel Dashboard
2. 點擊你的項目
3. 點擊"Deployments"選項卡

每次 Git Push 都會產生一條部署記錄，甚至每個分支都有獨立的預覽鏈接。

## 如何回滾到之前的版本

如果你更新後發現網站出了問題，可以快速回滾到之前正常的版本。

### 在 Zeabur 中

1. 找到部署歷史
2. 找到之前正常的部署記錄
3. 點擊"重新部署"或"Redeploy"

### 在 Vercel 中

1. 在 Deployments 頁面找到之前的部署
2. 點擊右側的三個點菜單
3. 選擇"Promote to Production"

回滾是即時的，通常幾秒鐘就能生效。

## 如何查看部署日誌

當部署失敗時，日誌是排查問題的關鍵。

在部署記錄中，點擊具體的某次部署，可以看到：
- 構建日誌：項目是如何被構建的
- 運行日誌：項目運行時的輸出

如果看到紅色的錯誤信息，可以複製給 AI 幫你分析。

## 自定義域名（進階預告）

現在你使用的是平臺提供的免費域名：
- Zeabur：`your-project.zeabur.app`
- Vercel：`your-project.vercel.app`

如果你想要自己的域名（如 `my-todo.com`），需要：
1. 購買域名（阿里雲、騰訊雲、Cloudflare 等）
2. 在部署平臺中綁定域名
3. 配置 DNS 記錄

這些操作稍微複雜一些，屬於進階版的內容。對於學習和個人使用來說，免費域名完全夠用。
