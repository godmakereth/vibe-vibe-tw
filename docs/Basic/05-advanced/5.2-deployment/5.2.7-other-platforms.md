# 5.2.7 其他部署選擇

除了 Zeabur 和 Vercel，還有一些其他的部署平臺可以選擇。這裏做簡要介紹，供參考。

## Netlify：拖拽部署

Netlify 是另一個流行的靜態網站託管平臺，它最大的特點是支持"拖拽部署"。

### 使用方法

1. 打開 [Netlify Drop](https://app.netlify.com/drop)
2. 把你的項目文件夾直接拖進瀏覽器頁面
3. 等待幾秒鐘，部署完成
4. 獲得一個 `*.netlify.app` 的網址

> **[配圖建議]**
> 畫面內容：Netlify Drop 頁面，展示拖拽區域
> Prompt：web interface showing drag and drop zone, file folder being dragged, clean modern design, light background, 8k --ar 16:9

### 優缺點

| 優點 | 缺點 |
|------|------|
| 操作極簡，拖拽即可 | 更新需要重新拖拽 |
| 不需要 GitHub 賬號 | 大陸訪問有問題 |
| 秒級部署 | 不適合頻繁更新的項目 |

**適合**：臨時演示、一次性項目、不想折騰 Git 的情況。

### 持續部署方式

如果想要代碼更新後自動部署，可以：

1. 註冊 Netlify 賬號
2. 連接 GitHub 倉庫
3. 配置構建設置

這樣就和 Zeabur、Vercel 一樣，Push 後自動部署了。

## GitHub Pages：完全免費

GitHub Pages 是 GitHub 提供的靜態網站託管服務，完全免費，無需任何額外賬號。

### 基本原理

GitHub Pages 會把你倉庫中的 HTML 文件直接發佈成網站。

### 使用方法概要

1. 確保你的倉庫是 public（公開）
2. 確保有一個 `index.html` 文件
3. 進入倉庫的 Settings → Pages
4. 選擇要部署的分支（通常是 main）
5. 等待幾分鐘，獲得 `username.github.io/repo-name` 的網址

### 優缺點

| 優點 | 缺點 |
|------|------|
| 完全免費 | 僅支持靜態網站 |
| 與 GitHub 深度集成 | 大陸訪問不穩定 |
| 支持自定義域名 | 配置略複雜 |

**適合**：個人博客、文檔站點、開源項目主頁。

::: tip 進階版預告
GitHub Pages 的詳細配置和 GitHub Actions 自動部署將在進階版中講解。
:::

## Railway：免費額度較少

Railway 是一個全棧應用部署平臺，支持前端、後端、數據庫一站式部署。

**特點**：
- 支持多種語言和框架
- 提供數據庫服務
- 免費額度每月 $5

**官網**：[railway.app](https://railway.app/)

**適合**：需要後端服務的項目。對於我們的純前端待辦清單，Railway 有點"殺雞用牛刀"。

## Render：全棧部署

Render 類似於 Railway，是一個全棧部署平臺。

**特點**：
- 免費層支持靜態網站
- 支持後端服務和數據庫
- 自動從 Git 部署

**官網**：[render.com](https://render.com/)

## Cloudflare Pages：邊緣部署

Cloudflare Pages 是 Cloudflare 提供的靜態網站託管服務。

**特點**：
- 完全免費
- 全球邊緣網絡加速
- 與 Cloudflare 生態深度集成

**限制**：
- 大陸訪問需要額外配置
- 配置比 Zeabur 稍複雜

**官網**：[pages.cloudflare.com](https://pages.cloudflare.com/)

## 如何選擇

對於基礎版的學習，**不需要糾結**於這些選擇。建議：

- 大陸用戶：用 Zeabur
- 海外用戶：用 Vercel
- 臨時演示：用 Netlify Drop

等你完成基礎版學習，有更多需求時，再探索其他平臺。
