# 5.2.4 Vercel 部署實戰：海外用戶/有域名用戶

Vercel 是全球最流行的前端部署平臺之一，尤其適合 Next.js、React 等現代前端框架。它的自動檢測和部署流程非常智能，幾乎不需要任何配置。

::: warning 中國大陸用戶請注意
Vercel 的免費域名 `*.vercel.app` 在中國大陸無法正常訪問。如果你在大陸且沒有自定義域名，建議使用 Zeabur（參見 5.2.3）。

本節適合以下讀者：
- 你在海外
- 你有自己的域名，準備用 Cloudflare 代理
- 你想了解 Vercel 的使用方法，爲將來做準備
:::

## Step by Step 操作

### Step 1：訪問 Vercel 並登錄

打開瀏覽器，訪問 [vercel.com](https://vercel.com)。

點擊右上角的"Sign Up"或"Log In"，選擇"Continue with GitHub"。

> **[配圖建議]**
> 畫面內容：Vercel 首頁，突出 GitHub 登錄選項
> Prompt：modern landing page illustration, GitHub login button highlighted, dark theme with purple accents, clean design, professional, 8k --ar 16:9

**你應該看到**：GitHub 授權頁面。點擊"Authorize Vercel"。

### Step 2：進入控制檯

登錄後，你會看到 Vercel 的 Dashboard。

如果是新用戶，頁面會引導你創建第一個項目。

### Step 3：導入項目

點擊"Add New..."按鈕，選擇"Project"。

或者直接點擊"Import Project"。

**你應該看到**：一個讓你選擇 Git 倉庫的頁面。

### Step 4：選擇倉庫

在"Import Git Repository"區域，你會看到你的 GitHub 倉庫列表。

如果看不到倉庫，點擊"Adjust GitHub App Permissions"去授權。

找到你的待辦清單項目，點擊"Import"。

> **[配圖建議]**
> 畫面內容：倉庫列表頁面，突出 Import 按鈕
> Prompt：dashboard interface showing repository list, import button highlighted, modern UI design, dark theme, clean layout, 8k --ar 16:9

### Step 5：配置項目

Vercel 會自動檢測你的項目類型。

對於我們的待辦清單（純 HTML/CSS/JS），通常不需要任何配置。

檢查以下設置：
- **Framework Preset**：如果是純 HTML，選擇"Other"
- **Root Directory**：保持默認（`.`）
- **Build Command**：留空（純靜態項目不需要構建）
- **Output Directory**：留空或填 `.`

**你應該看到**：配置頁面，大部分選項已自動填好。

### Step 6：部署

確認配置無誤後，點擊"Deploy"按鈕。

Vercel 會開始構建和部署你的項目。這通常需要 30 秒到 2 分鐘。

**你應該看到**：
- 即時的構建日誌
- 狀態從"Building"變成"Ready"
- 出現慶祝動畫和預覽圖

### Step 7：獲得網址

部署成功後，Vercel 會給你一個網址，格式類似：

```
https://your-project-name.vercel.app
```

點擊這個鏈接，查看你的網站。

::: warning 再次提醒
如果你在中國大陸，這個鏈接可能無法訪問。解決方案見 [5.2.6 大陸訪問解決方案](./5.2.6-china-solution.md)。
:::

## Vercel 的特色功能

Vercel 有一些很實用的功能，瞭解一下：

| 功能 | 說明 |
|------|------|
| Preview Deployments | 每次 Push 都會生成一個預覽鏈接 |
| 自動 HTTPS | 所有網站自動啓用 HTTPS |
| 邊緣網絡 | 全球 CDN 加速 |
| 回滾 | 一鍵回滾到之前的版本 |
