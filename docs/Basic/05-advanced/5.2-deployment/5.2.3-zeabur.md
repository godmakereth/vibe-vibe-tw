# 5.2.3 Zeabur 部署實戰：大陸用戶首選

Zeabur 是一個由國人團隊開發的部署平臺，它的免費域名在國內可以正常訪問，這是選擇它的最重要原因。

## 經過本節操作，你將獲得

- 一個可以在國內正常訪問的網址
- 自動部署能力：代碼更新後自動同步
- 完整的部署流程經驗

## Zeabur 的優勢

| 特點 | 說明 |
|------|------|
| 大陸可訪問 | 免費域名 `*.zeabur.app` 沒有被牆 |
| 中文支持 | 界面支持中文，客服也是中文 |
| 自動檢測 | 自動識別你的項目類型，無需配置 |
| 免費額度 | 每月 $5 免費額度，小項目完全夠用 |
| 支付寶付款 | 如果需要升級，可以用支付寶 |

## 準備工作

在開始之前，確認你已經完成：

- [ ] 第四章的待辦清單項目已完成
- [ ] 按照 5.1 的步驟，項目已上傳到 GitHub
- [ ] 有一個 GitHub 賬號

如果還沒上傳到 GitHub，請先回到 [5.1 版本管理](../5.1-version-control/) 完成這一步。

## Step by Step 操作

### Step 1：訪問 Zeabur 並登錄

打開瀏覽器，訪問 [zeabur.com](https://zeabur.com)。

點擊右上角的"登錄"按鈕，選擇"使用 GitHub 登錄"。

> **[配圖建議]**
> 畫面內容：Zeabur 首頁，箭頭指向右上角的登錄按鈕
> Prompt：clean screenshot style illustration of a modern web dashboard homepage, login button highlighted, minimal design, light background, soft shadows, 8k --ar 16:9

**你應該看到**：GitHub 授權頁面，詢問是否允許 Zeabur 訪問你的賬號。

點擊"Authorize"（授權）。

### Step 2：創建項目

登錄後，你會看到 Zeabur 的控制檯。

點擊"創建項目"（Create Project）按鈕。

> **[配圖建議]**
> 畫面內容：Zeabur 控制檯，突出顯示"創建項目"按鈕
> Prompt：minimal dashboard interface illustration, create project button prominently displayed, clean layout, pastel colors, professional look, 8k --ar 16:9

**你應該看到**：彈出一個對話框，讓你選擇服務器區域。

### Step 3：選擇服務器區域

在彈出的對話框中，選擇服務器區域。

**推薦選擇**：
- 如果你在中國大陸：選擇 **Asia**（亞洲）或 **Hong Kong**（香港）
- 如果你在其他地區：選擇離你最近的區域

選擇後點擊"創建"。

**你應該看到**：一個空的項目面板，顯示"添加服務"按鈕。

### Step 4：添加服務

在項目面板中，點擊"添加服務"（Add Service）。

在彈出的菜單中，選擇"部署你的代碼"（Deploy your code）或"Git"。

> **[配圖建議]**
> 畫面內容：服務選擇菜單，突出"部署代碼"選項
> Prompt：dropdown menu UI illustration, deploy code option highlighted, modern interface design, clean white background, subtle shadows, 8k --ar 16:9

### Step 5：授權並選擇倉庫

如果是第一次使用，Zeabur 會請求訪問你的 GitHub 倉庫。

點擊"配置 GitHub"，然後選擇"授權所有倉庫"或僅授權你的待辦清單倉庫。

授權完成後，你會看到你的 GitHub 倉庫列表。

**找到並選擇**你在第四章創建的待辦清單倉庫（比如 `my-todo-list`）。

**你應該看到**：Zeabur 開始檢測你的項目類型。

### Step 6：等待構建

選擇倉庫後，Zeabur 會自動開始構建你的項目。

這個過程通常需要 1-3 分鐘。你可以看到構建日誌即時滾動。

**你應該看到**：
- 狀態從"Building"變成"Deploying"
- 最終變成綠色的"Running"

::: tip 構建需要時間
第一次構建可能需要 2-3 分鐘，耐心等待。如果超過 5 分鐘還沒完成，可能是項目有問題。
:::

### Step 7：生成域名

構建成功後，你的服務會顯示爲"Running"狀態。

但此時還沒有公開網址。你需要爲它生成一個域名。

1. 點擊你的服務（待辦清單）
2. 在服務詳情頁，找到"網絡"（Networking）選項卡
3. 點擊"生成域名"（Generate Domain）

> **[配圖建議]**
> 畫面內容：服務詳情頁的網絡選項卡，突出"生成域名"按鈕
> Prompt：web dashboard networking settings, generate domain button highlighted, clean interface, light theme, professional design, 8k --ar 16:9

**你應該看到**：系統自動生成一個類似 `your-project-abc123.zeabur.app` 的域名。

### Step 8：訪問你的網站

點擊生成的域名鏈接，或者複製鏈接在新標籤頁打開。

**你應該看到**：你的待辦清單頁面，和在本地看到的一模一樣。

恭喜！你的作品現在已經在互聯網上了。

## 成功驗證清單

完成部署後，逐項驗證：

- [ ] 服務狀態顯示綠色的"Running"
- [ ] 點擊域名鏈接能打開頁面
- [ ] 頁面顯示正常，樣式沒有丟失
- [ ] 可以正常添加、刪除、完成任務
- [ ] **用手機瀏覽器打開鏈接，也能正常訪問**

最後一項尤其重要——這證明了你的網站真的可以從任何設備訪問。

## 常見問題

| 問題 | 可能原因 | 解決方法 |
|------|---------|---------|
| 構建失敗 | 代碼有語法錯誤 | 先在本地確認項目能正常運行 |
| 頁面空白 | 入口文件路徑不對 | 確保 `index.html` 在項目根目錄 |
| 樣式丟失 | CSS 路徑使用了絕對路徑 | 改用相對路徑，如 `./style.css` |
| 功能不工作 | JavaScript 報錯 | 打開瀏覽器控制檯查看錯誤信息 |
