# 5.2.8 部署常見問題

部署過程中遇到問題是正常的。這裏整理了最常見的問題和解決方案。

## 問題速查表

| 問題 | 可能原因 | 解決方法 |
|------|---------|---------|
| 部署失敗 | 代碼有語法錯誤 | 先在本地確認能正常運行 |
| 頁面空白 | 入口文件路徑不對 | 確保 `index.html` 在項目根目錄 |
| 樣式丟失 | CSS 路徑使用了絕對路徑 | 改用相對路徑（如 `./style.css`） |
| 圖片不顯示 | 圖片路徑錯誤或文件名大小寫 | 檢查路徑和文件名大小寫 |
| 功能不工作 | JavaScript 報錯 | 打開瀏覽器控制檯（F12）查看錯誤 |
| 大陸無法訪問 | DNS 污染 | 換用 Zeabur 或帽子云 |
| 加載很慢 | 資源文件太大 | 壓縮圖片，檢查是否有不必要的大文件 |
| 更新後沒變化 | 瀏覽器緩存 | 強制刷新（Ctrl+F5）或清除緩存 |

## 問題詳解

### 問題 1：部署失敗，構建報錯

**現象**：點擊部署後，狀態一直顯示"Building"然後變成"Failed"。

**排查步驟**：

1. 查看構建日誌，找到紅色的錯誤信息
2. 檢查本地項目是否能正常運行
3. 確認所有必要的文件都已提交到 GitHub

**常見原因**：
- 代碼有語法錯誤
- 缺少必要的配置文件
- 依賴沒有正確安裝

**AI 求助模板**：

```
我在 [Zeabur/Vercel] 上部署項目時，構建失敗了。

構建日誌顯示這個錯誤：
[粘貼紅色的錯誤信息]

我的項目是一個 [純 HTML/CSS/JS 靜態網站 / React 項目 / ...]。
本地運行是 [正常的 / 也有問題]。

請幫我分析原因並給出解決方案。
```

### 問題 2：頁面空白

**現象**：部署成功了，但打開網址是空白頁面。

**排查步驟**：

1. 打開瀏覽器開發者工具（F12）
2. 查看 Console（控制檯）是否有紅色錯誤
3. 查看 Network（網絡）是否有 404 錯誤

**常見原因**：
- `index.html` 不在根目錄
- JavaScript 文件路徑錯誤導致腳本加載失敗
- 使用了絕對路徑，線上環境找不到文件

**解決方法**：

```html
<!-- 錯誤：絕對路徑 -->
<script src="/js/app.js"></script>

<!-- 正確：相對路徑 -->
<script src="./js/app.js"></script>
```

### 問題 3：樣式丟失

**現象**：頁面能顯示，但樣式完全亂了，或者沒有樣式。

**常見原因**：
- CSS 文件路徑錯誤
- CSS 文件沒有被提交到 GitHub
- 使用了絕對路徑

**解決方法**：

```html
<!-- 錯誤 -->
<link rel="stylesheet" href="/style.css">

<!-- 正確 -->
<link rel="stylesheet" href="./style.css">
```

### 問題 4：功能不工作

**現象**：頁面顯示正常，但按鈕點擊沒反應，功能不工作。

**排查步驟**：

1. 打開瀏覽器控制檯（F12 → Console）
2. 查看是否有紅色錯誤信息
3. 嘗試觸發功能，看是否有新的錯誤出現

**AI 求助模板**：

```
我的待辦清單部署後，[添加任務 / 刪除任務 / ...] 功能不工作了。

瀏覽器控制檯顯示這個錯誤：
[粘貼錯誤信息]

本地運行時功能是正常的。
請幫我分析原因。
```

### 問題 5：更新後網站沒變化

**現象**：代碼已經推送到 GitHub，但網站還是舊版本。

**可能原因**：
1. 部署還沒完成（去控制檯查看狀態）
2. 瀏覽器緩存

**解決方法**：
- 等待部署完成（通常 1-2 分鐘）
- 強制刷新頁面：Windows 按 `Ctrl+F5`，Mac 按 `Cmd+Shift+R`
- 打開隱私模式/無痕模式訪問

## 萬能解決方案

當你遇到任何部署問題時，都可以用這個模板向 AI 求助：

```
我在部署網站時遇到了問題。

【環境信息】
- 部署平臺：[Zeabur / Vercel / Netlify / ...]
- 項目類型：[純 HTML/CSS/JS / React / Vue / ...]
- 本地運行：[正常 / 有問題]

【問題描述】
[描述你遇到的具體問題]

【錯誤信息】
[粘貼完整的錯誤信息，包括構建日誌或瀏覽器控制檯的錯誤]

【已嘗試的方法】
[列出你已經嘗試過的解決方法]

請幫我分析原因並給出解決方案。
```

::: tip 記住
遇到問題不要慌。部署問題通常都有明確的錯誤信息，把錯誤信息發給 AI，大多數問題都能快速解決。
:::
