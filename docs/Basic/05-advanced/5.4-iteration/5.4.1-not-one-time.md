# 5.4.1 你的項目不是一次性的

經過本節學習，你將理解：
- 爲什麼"完成"只是開始
- 真實產品是如何一步步進化的
- 如何用迭代思維規劃下一步

## 你可能有的困惑

"第四章做完的待辦清單，已經能用了，還需要改嗎？"

這個想法很正常。我們從小寫作業，交了就完事了。但做工具不一樣——**好用的工具，都是用出來的，不是一次做出來的**。

## 看看那些你每天用的產品

微信剛上線時是什麼樣子？

| 時間 | 版本 | 功能 |
|------|------|------|
| 2011年1月 | 1.0 | 只能發消息、發圖片 |
| 2012年4月 | 4.0 | 加入朋友圈 |
| 2013年8月 | 5.0 | 加入微信支付 |
| 2017年1月 | 小程序上線 | 不用下載App就能用各種服務 |

微信用了6年，從一個簡單的聊天工具變成了一個超級平臺。

淘寶最早就是一個簡陋的網頁，連圖片都模糊。你現在用的"猜你喜歡"、"直播購物"，都是後來一點點加上去的。

**沒有哪個產品是一步到位的。** 你的待辦清單也不需要。

## 一次性作業 vs 持續進化

| 一次性作業心態 | 持續進化心態 |
|---------------|-------------|
| "做完就不管了" | "做完是起點" |
| "要一次做到完美" | "先能用，再好用" |
| "功能越多越好" | "解決一個問題再說下一個" |
| "有bug說明我做錯了" | "有bug說明我發現了改進點" |

第二種心態，是創造者的心態。

## 迭代循環：做 → 用 → 改 → 做

```
    ┌─────────────┐
    │   做一版    │
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │   用一用    │
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │  發現問題   │
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │   改進它    │
    └──────┬──────┘
           │
           └──────────────┐
                          │
           ┌──────────────┘
           │
           ▼
      (回到"做一版")
```

每轉一圈，你的工具就好用一點。

這不是因爲你第一次做得不好，而是**有些問題只有用了纔會發現**。

## 回顧靈魂三問：你的改進方向

還記得第二章的"靈魂三問"嗎？

- **用戶是誰？** → 誰在用你的待辦清單？你自己？家人？室友？
- **痛點在哪？** → 用了一段時間後，什麼地方讓你不爽？
- **爲什麼選你？** → 和手機自帶的備忘錄比，你的有什麼特別之處？

這三個問題，在迭代階段同樣有用。

當你不知道下一步該改什麼時，問自己：
- 我用的時候，哪一步最麻煩？
- 我希望它能做什麼，但目前做不到？

這些答案，就是你的迭代方向。

## 本節要點

- 完成第一版只是開始，不是終點
- 好的工具都是在使用中慢慢進化出來的
- 每次改進只需要解決一個問題
- 第二章的靈魂三問，可以幫你找到改進方向

::: tip 一個好的心態
不要想着"一次做到完美"。

做出來 → 用起來 → 發現問題 → 改進它。

這就是創造者的日常。
:::
