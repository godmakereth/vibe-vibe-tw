# 5.4.2 收集反饋：讓使用者告訴你

經過本節學習，你將掌握：
- 爲什麼反饋比自己悶頭想更有用
- 三種零門檻的反饋收集方法
- 一個簡單的反饋記錄模板
- 如何讓 AI 幫你分析反饋

## 爲什麼需要別人的反饋

你做的工具，你自己覺得好用，不代表別人覺得好用。

根據 ProdPad 2025 年的調研，**91% 的用戶認爲產品創新應該來自客戶的聲音**。換句話說，閉門造車做出來的東西，大概率不是用戶真正想要的。

你可能會想："我的待辦清單就是給自己用的，不需要別人的意見吧？"

其實不然。就算是給自己用，讓朋友試一下也能發現問題：
- 你覺得"顯而易見"的操作，別人可能根本不知道怎麼用
- 你習慣的界面佈局，換個人可能覺得很彆扭
- 你從來不用的功能，別人可能每天都想用

**找幾個人試用，是發現問題最快的方法。**

## 方法一：真人測試法

最簡單的方法：把你的待辦清單發給 3-5 個人試用。

可以是：
- 家人（特別是不太懂技術的）
- 室友或同事
- 朋友

不需要多，3個人就夠了。

### 怎麼發給他們

如果你已經完成了 5.2 節的部署，直接把網址發給他們就行：

> "我做了一個待辦清單工具，你能幫我試用一下嗎？大概花2分鐘。網址是 xxx.zeabur.app"

如果還沒部署，可以讓他們到你電腦前試用。

## 方法二：觀察法

觀察他們怎麼用，比直接問更有用。

當別人在試用時，注意觀察：

| 觀察點 | 可能說明的問題 |
|--------|--------------|
| 停頓了很久，不知道點哪裏 | 界面不夠直觀 |
| 問你"這個按鈕是幹嘛的" | 按鈕文字不夠清楚 |
| 反覆點同一個地方 | 交互反饋不明顯 |
| 皺眉頭、嘆氣 | 某個操作讓他們感到挫敗 |
| "哦，原來是這樣" | 操作邏輯不符合直覺 |

**不要在旁邊指導**。讓他們自己摸索，你只是觀察。這樣才能看到真實的使用情況。

## 方法三：直接問

觀察完之後，可以問他們三個問題：

```
1. 第一次打開時，你知道該怎麼用嗎？
2. 有什麼地方讓你感到困惑或不方便？
3. 如果可以加一個功能，你希望是什麼？
```

這三個問題覆蓋了：
- 易用性（能不能快速上手）
- 痛點（哪裏不好用）
- 需求（希望有什麼功能）

## 記錄反饋的模板

收集到的反饋，用一個簡單的表格記錄下來：

| 反饋來源 | 問題/建議 | 優先級 | 狀態 |
|---------|----------|-------|------|
| 室友 | 想要任務截止日期 | P1 | 待開發 |
| 媽媽 | 按鈕太小，不好點 | P0 | 已修復 |
| 自己 | 想要分類標籤 | P2 | 計劃中 |
| 同事 | 能不能有深色模式 | P2 | 計劃中 |

### 優先級怎麼定

| 優先級 | 含義 | 舉例 |
|--------|------|------|
| **P0** | 必須馬上修 | 功能壞了、按鈕點不動、界面錯亂 |
| **P1** | 儘快做 | 影響日常使用的問題，多人反饋的需求 |
| **P2** | 有空再說 | 錦上添花的功能，個別人的特殊需求 |

一個簡單的判斷標準：
- **影響所有人** → P0 或 P1
- **隻影響部分人** → P1 或 P2
- **沒人提但你自己想做** → 先放一放

## 讓 AI 幫你分析反饋

如果你收集了不少反饋，可以讓 AI 幫你整理和分析。

```
我做了一個待辦清單工具，收集到了一些用戶反饋，請幫我分析：

反饋列表：
1. 室友說想要任務截止日期提醒
2. 媽媽說按鈕太小，在手機上不好點
3. 同事說希望有深色模式
4. 我自己覺得任務多了之後不好找，想要分類功能
5. 室友還說任務完成後的動畫挺好看的（正面反饋）

請幫我：
1. 按優先級排序這些反饋
2. 分析哪些是共性問題，哪些是個人偏好
3. 建議我先做哪個改進
```

AI 會幫你梳理出一個清晰的行動計劃。

## 常見誤區

### 只聽表揚，忽略批評

"挺好的"、"還不錯"這種反饋雖然聽着舒服，但對改進沒有幫助。

**真正有價值的反饋往往是批評**：
- "我不知道怎麼刪除任務"
- "這個顏色看着有點刺眼"
- "能不能加個搜索功能"

這些纔是改進的方向。

### 試圖滿足所有人

不同的人會提出不同甚至矛盾的需求。你不需要滿足所有人。

回到第二章的靈魂三問：**你的工具到底是給誰用的？** 優先滿足核心用戶的需求。

## 本節檢查清單

- [ ] 找到了 3-5 個人試用你的待辦清單
- [ ] 觀察了他們的使用過程
- [ ] 問了三個關鍵問題
- [ ] 用表格記錄了收集到的反饋
- [ ] 給反饋標註了優先級

::: tip 反饋是禮物
每一條反饋，不管是批評還是建議，都是幫你改進的線索。

別人願意花時間告訴你問題，說明他們在乎你的作品。
:::
