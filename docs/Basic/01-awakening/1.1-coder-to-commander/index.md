# 1.1 編程已死，編程永生：從 Coder 到 Commander

> "There's a new kind of coding I call 'vibe coding', where you fully give in to the vibes, embrace exponentials, and forget that the code even exists."
> 
> —— Andrej Karpathy, 2025年2月


## 本節你將瞭解

如果你從未寫過代碼，卻聽說"編程很難"、"需要學好幾年"，那這一節會讓你大喫一驚。

2025年，編程世界發生了一場靜悄悄的革命。你不需要記住任何語法，不需要理解複雜的算法，甚至不需要看懂代碼——**你只需要把想要的東西說清楚**。

在這一節，你會看到：

- **世界正在發生什麼**：一條推文如何引發編程方式的鉅變
- **角色如何轉變**：從"敲代碼的人"變成"指揮AI的人"
- **一個生動的類比**：用開車來理解這場變革
- **這對你意味着什麼**：爲什麼說"人人都能成爲Builder"


## 爲什麼要讀這一節？

你可能會想："我又不想當程序員，爲什麼要了解編程的變化？"

答案很簡單：**這場變化讓編程不再只屬於程序員了**。

根據2025年的調研，**63%的Vibe Coding用戶是非開發者**——設計師、產品經理、創業者、甚至英語專業畢業生。

就像智能手機讓每個人都能拍出好照片一樣，AI 編程工具正在讓每個人都能把想法變成現實。

不管你是想做一個小工具解決工作中的煩惱，還是想驗證一個創業點子，或者只是想給孩子做一個有趣的小遊戲——現在，你都可以自己動手了。

準備好了嗎？讓我們從一條改變世界的推文說起。
