## 工作流程的變化

**傳統編程：**

1. 學習編程語言：花幾個月學 JavaScript、Python 或其他語言
2. 理解語法規則：記住各種括號、分號、縮進規則
3. 一行一行寫代碼：把每個功能都"翻譯"成計算機能懂的指令
4. 不停地調試：程序報錯了？自己一行行找問題
5. 查文檔、問社區：遇到不會的，花幾小時搜索解決方案

整個過程像在學一門外語——你得先學會語法，才能表達自己想說的話。

**AI 時代：**

1. 告訴 AI 你想要什麼："幫我做一個待辦清單，可以添加任務、標記完成、刪除任務"
2. 看 AI 生成代碼：幾秒鐘後，代碼就出現了
3. 試試看效果：運行一下，看看是不是你想要的
4. 告訴 AI 怎麼改："把完成的任務變成灰色，加一個刪除線"
5. 繼續迭代：直到滿意爲止

整個過程更像在和一個懂技術的助手對話——你說想法，它來實現。

## 核心技能的遷移

| 過去的核心技能 | 現在的核心技能 |
|--------------|--------------|
| 記住語法規則 | 把需求說清楚 |
| 手寫代碼 | 判斷代碼對不對 |
| 調試找bug | 描述問題讓AI修 |
| 閱讀技術文檔 | 知道什麼該問、什麼該查 |

用開車的類比：

| 你需要知道的 | 你不需要知道的 |
|-------------|---------------|
| 我要去哪（需求） | 發動機怎麼工作（底層原理） |
| 走哪條路更快（大方向） | 變速箱的原理（語法細節） |
| 到沒到、對不對（驗收） | 如何修車（手寫每行代碼） |
| 路況不對要調整（迭代） | 汽車電路圖（複雜架構） |

::: tip 一句話總結
從"寫代碼"變成"表達清楚 + 判斷對錯"。
:::

## 什麼沒變

雖然"怎麼做"變了，但有些東西永遠不會變：

- **你需要知道自己想要什麼**：AI再聰明，也不能替你想清楚要做什麼
- **你需要判斷結果好不好**：AI可能會出錯，你得能看出來
- **你需要有解決問題的思路**：遇到問題時，知道怎麼一步步排查

::: warning AI 目前還不是"完全自動駕駛"
就像今天的自動駕駛汽車，AI編程也需要你保持關注、隨時接手、負最終責任。Vibe Coding 更像是"L3 級自動駕駛"——大部分時候AI來做，但你得盯着，關鍵時刻你得上。
:::

AI 接管了"執行"，但"思考"和"決策"還是你的事。

## Context Engineering：進一步的進化

2025年下半年，行業開始從 Vibe Coding 進化到 **Context Engineering**（上下文工程）。

Vibe Coding 強調"跟着感覺走"，快速出結果。Context Engineering 則強調：給AI提供足夠的上下文，讓它理解你的項目背景、代碼規範、業務邏輯。

換句話說：想讓AI幫得更好，你需要學會"提供好的上下文"。這也是本教程後面會重點教你的內容。

## 這對你意味着什麼

如果你會用滴滴打車，你就已經具備了 Vibe Coding 最核心的能力：

- 說清楚你要去哪（表達需求）
- 判斷司機有沒有走錯路（驗收結果）
- 必要時給司機指路（引導調整）

這取決於個人優勢。如果你擅長表達、溝通，擅長想清楚需求，對細節敏感、能發現問題——這些都是你的優勢。

如果你之前總覺得"編程太難了"，很可能是因爲你不擅長記語法、寫代碼——但那不是你真正的短板。**真正重要的能力，你可能早就有了。**
