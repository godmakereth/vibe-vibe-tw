# 1.5.4 常見問題 FAQ

做第一個項目的時候，你可能會遇到一些困惑。

別擔心，這些問題大家都會遇到。


## Q1："我的結果和教程不一樣怎麼辦？"

**這太正常了。**

AI 每次生成的內容都會有些不同。就像你問兩個設計師"做一個好看的頁面"，他們給你的方案肯定也不一樣。

::: tip 關鍵是
只要你的頁面：
- 能正常顯示
- 有名言和作者
- 看起來還不錯

就算成功了！
:::

如果你想要和教程一模一樣的效果，可以在 Prompt 裏描述得更具體，比如：
- 指定具體的顏色（"背景用 #f0f4f8"）
- 指定具體的字體大小（"名言用 32px 的字體"）
- 指定具體的佈局（"名言在頁面垂直居中"）


## Q2："代碼看不懂正常嗎？"

**完全正常。**

起步階段，你不需要看懂每一行代碼。

就像你用微信發消息，不需要懂微信是怎麼寫出來的。

Vibe Coding 的核心理念是：**讓 AI 處理代碼語法，你專注於需求和結果**。

::: info 好消息
隨着使用次數增多，你會自然地開始認識一些常見的代碼結構。

如果你好奇某段代碼是什麼意思，可以問 AI：

```
請解釋一下這段代碼是做什麼的，用最簡單的話說
```

AI 會用你能懂的語言給你解釋。這不是必須的，但會讓你和 AI 的協作更順暢。
:::


## Q3："可以用中文嗎？"

**當然可以。**

現在的 AI 對中文的理解已經非常好了。

用中文寫 Prompt，AI 完全能懂。

::: tip 什麼時候用英文？
有些技術名詞用英文會更準確，比如：
- "按鈕" vs "button"
- "懸停效果" vs "hover effect"

但大部分情況下，用中文完全沒問題。

如果你英文好，用英文也可以——效果差不多。
:::


## Q4："AI 生成的東西報錯了怎麼辦？"

**把錯誤信息複製給 AI。**

這是最簡單有效的方法：

```
頁面報錯了，錯誤信息是：
[粘貼錯誤信息]

請幫我修復
```

AI 會分析錯誤原因，然後給你修復後的代碼。

::: warning 如果反覆修不好
1. **簡化你的需求**：把複雜功能拆成更小的步驟，一次只做一件事
2. **換一種描述方式**：同樣的需求，用不同的話說，AI 可能更容易理解
3. **從最簡版本重新開始**：刪掉複雜功能，先讓基礎版跑起來，再逐步加功能

問題通常出在"描述不夠清楚"，而不是工具不行。
:::


## Q5："這個網頁怎麼發給別人看？"

現在你做的網頁只在工具的預覽窗口裏。想讓別人訪問，需要"部署"——這個我們在後面的章節會詳細講。

**現在不用着急，先把東西做出來再說。**


## Q6："我想保存這個項目怎麼辦？"

不同類型的工具保存方式不同：

| 工具類型 | 保存方式 |
|----------|----------|
| 網頁版工具（如 Bolt.new、豆包） | 註冊賬號後自動保存，也可以導出代碼 |
| AI 代碼編輯器（如 Cursor、Trae） | 本地文件，隨時保存 |

如果你只是想保留代碼，最簡單的方法是：
1. 複製 AI 生成的所有代碼
2. 粘貼到一個文本文件裏
3. 保存到電腦上


## Q7："我可以用這個做商業項目嗎？"

可以，但建議分階段：

1. **先用 Vibe Coding 驗證想法**——快速做出原型，看看是否可行
2. **如果要正式上線**——涉及用戶數據、支付等敏感功能時，需要學習進階篇的安全（第6章）和測試（第9章）內容

很多成功的產品，都是從一個粗糙的原型開始的。本教程的進階篇會教你如何把原型變成正式產品。


## Q8："不同工具界面不一樣怎麼辦？"

**不用擔心，核心操作都一樣。**

無論你用 Bolt.new、豆包、Cursor 還是其他工具，基本流程都是：

1. 找到輸入框
2. 用中文描述你想要什麼
3. 等 AI 生成
4. 看預覽效果
5. 不滿意就繼續說

界面按鈕的位置可能不同，但"用自然語言描述需求"這個核心不會變。

如果找不到某個功能，直接問 AI："怎麼在 [工具名] 裏保存項目？"


## 還有其他問題？

如果你遇到了上面沒提到的問題，試試這個方法：

**直接問 AI。**

```
我在用 [工具名] 做一個網頁，遇到了這個問題：
[描述你的問題]

請問怎麼解決？
```

AI 能幫你思考大部分問題——但記得驗證它給的答案是否靠譜。

這也是 Vibe Coding 的精髓：**遇到問題，先問 AI，再自己驗證**。
