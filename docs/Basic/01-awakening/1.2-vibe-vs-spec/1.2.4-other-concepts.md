# 1.2.4 其他你需要知道的概念

在使用 AI 編程的過程中，你會經常遇到一些術語。這一節，我們用最簡單的方式把它們講清楚。

不需要死記硬背，用到的時候回來查就行。


## Prompt（提示詞）

### 一句話解釋

**Prompt 就是你對 AI 說的話。**

你輸入什麼，AI 就根據你的輸入生成回應。這個輸入就叫 Prompt。

### 類比

Prompt 就像你在餐廳點菜時對服務員說的話：

- "來一碗牛肉麪" → 簡單 Prompt
- "來一碗牛肉麪，不要香菜，少放辣，面要硬一點" → 詳細 Prompt

說得越清楚，得到的結果越符合你的期望。

### 好 Prompt vs 差 Prompt

| 差 Prompt | 好 Prompt |
|-----------|-----------|
| "幫我做個網站" | "幫我做一個個人博客首頁，要有標題、簡介、三篇文章列表，風格簡潔現代" |
| "這個代碼有問題" | "這段代碼運行時報錯 'undefined is not a function'，錯誤出現在第 15 行，幫我看看怎麼修" |

### 你需要記住的

寫 Prompt 就像寫微信消息——**說清楚你想要什麼，對方纔能幫到你**。


## Context（上下文）

### 一句話解釋

**Context 是 AI 在回答你時能"看到"的所有信息。**

包括：
- 你當前說的話
- 你之前說的話
- 你給 AI 看的文件、代碼
- AI 自己之前的回覆

### 類比

想象你和朋友聊天：

- 如果你們剛認識，你說"那個事怎麼樣了"，對方會一臉懵
- 如果你們聊了很久，對方知道"那個事"指什麼，就能回答

AI 也一樣。它能"記住"多少，取決於 Context 有多大。

### Context Window（上下文窗口）

每個 AI 模型能"記住"的信息量是有限的，這個限制叫 **Context Window**。

| 模型 | 大約能記住多少 |
|------|---------------|
| 早期模型 | 幾千字 |
| GPT-4 | 約 2.5 萬字 |
| Claude 3 | 約 15 萬字 |
| 更新的模型 | 可能更多 |

### 爲什麼重要

如果你的對話太長，或者給 AI 看的文件太大，它可能會"忘記"早期的內容。

更重要的是：**提供好的 Context 是讓 AI 更好地幫你的關鍵**。

AI 不會讀心術。你給它的背景信息越清晰、越相關，它生成的代碼就越符合你的預期。這也是爲什麼 Spec Coding 要先寫文檔——文檔本身就是給 AI 的 Context。

**實用技巧**：
- 重要的信息放在最前面或最後面（AI 對這兩個位置記憶更好）
- 對話太長時，開一個新對話，把關鍵信息重新說一遍
- 在項目中維護一份"項目背景"文檔，每次對話時可以快速提供給 AI


## Hallucination（幻覺）

### 一句話解釋

**幻覺 = AI 一本正經地胡說八道。**

AI 有時候會編造看起來很合理、但實際上是錯的信息。

### 例子

你問 AI："XXX 這個函數怎麼用？"

AI 可能會給你一個詳細的說明，但這個函數根本不存在——它是 AI "編"出來的。

### 爲什麼會這樣

AI 不是"查資料"，而是"根據模式生成文字"。它不知道自己說的是真是假，它只知道"這樣說看起來很合理"。

### 如何應對

1. **驗證**：AI 說的東西，尤其是具體的函數名、API、數據，要自己覈實
2. **懷疑**：如果 AI 說得太肯定、太詳細，反而要多留個心眼
3. **讓它引用來源**：問 AI "你說的這個信息來自哪裏？"（雖然它可能繼續編，但會降低胡說的概率）

### 你需要記住的

**AI 是助手，不是權威。它說的話要驗證，不能全信。**


## Agent（智能體）

### 一句話解釋

**Agent = 能自己規劃、自己行動的 AI。**

普通的 AI 是"你問一句，它答一句"。

Agent 是"你給它一個目標，它自己想辦法完成"。

### 類比

| 類型 | 類比 |
|------|------|
| 普通 AI | 問路人"去火車站怎麼走"，他告訴你路線 |
| Agent | 告訴司機"帶我去火車站"，他自己規劃路線、開車、到達 |

### 在 AI 編程中的應用

- **普通 AI**：你說"幫我寫一個登錄功能"，它給你代碼
- **Agent**：你說"幫我做一個完整的用戶系統"，它自己分析需求、創建文件、寫代碼、測試、修 bug...

### 常見的 Agent 工具

- Cursor 的 Composer（多文件協作模式）
- Replit Agent
- Claude 的 Artifacts
- GitHub Copilot Workspace

### 你需要記住的

Agent 更強大，但也更不可控。對於新手，建議先用普通 AI，等熟悉了再嘗試 Agent。


## MCP（Model Context Protocol）

### 一句話解釋

**MCP = 讓 AI 能連接外部工具和數據的標準協議。**

### 類比

想象 AI 是一個很聰明的人，但它被關在一個房間裏，只能通過窗口和你對話。

MCP 就像給這個房間開了幾扇門：
- 一扇門通向數據庫
- 一扇門通向文件系統
- 一扇門通向互聯網
- ...

有了這些門，AI 能做的事情就多多了。

### 爲什麼重要

沒有 MCP 時，你需要手動把信息複製給 AI。

有了 MCP，AI 可以直接讀取你的文件、查詢數據庫、調用 API。

### 你需要知道的

MCP 是 2024-2025 年興起的新標準。如果你用 Cursor、Claude Desktop 等工具，可能會看到 MCP 相關的設置。

**現階段**：知道有這個東西就行，不需要深入瞭解。等你用到的時候再學。


## Token（令牌）與計費

### 一句話解釋

**Token 是 AI 處理文字的基本單位。AI 服務按 Token 數量收費。**

### Token 是什麼

AI 不是一個字一個字讀你的文字，而是把文字切成小塊來處理。這些小塊叫 Token。

大致換算（英文）：
- 1 個 Token ≈ 4 個字符
- 100 個 Token ≈ 75 個單詞

中文的話，1 個漢字通常是 1-2 個 Token。

### 爲什麼重要

1. **影響能處理的信息量**：Context Window 通常用 Token 數來衡量
2. **影響費用**：AI 服務按輸入 Token + 輸出 Token 收費

### 計費邏輯（以 API 調用爲例）

```
費用 = (輸入 Token 數 × 輸入單價) + (輸出 Token 數 × 輸出單價)
```

不同模型價格差異很大：

| 模型 | 大約價格（每百萬 Token） |
|------|-------------------------|
| GPT-4o | $2.5 - $10 |
| Claude 3.5 Sonnet | $3 - $15 |
| 更便宜的模型 | $0.1 - $1 |

### 如何省 Token

1. **Prompt 寫得精煉**：廢話少說
2. **不要重複粘貼大段代碼**：只粘相關部分
3. **用更便宜的模型做簡單任務**

### 你需要記住的

- 如果用免費工具（如 ChatGPT 免費版），不用操心 Token
- 如果用付費 API，注意控制用量，尤其是在調試時
- Cursor 等工具通常有用量限制，超了要加錢


## 概念速查表

| 概念 | 一句話解釋 | 生活類比 |
|------|-----------|----------|
| Prompt | 你對 AI 說的話 | 點菜時說的話 |
| Context | AI 能看到的所有信息 | 聊天記錄 |
| Context Window | AI 能記住的信息量上限 | 記憶力 |
| Hallucination | AI 一本正經地胡說 | 隨口編的"事實" |
| Agent | 能自己規劃行動的 AI | 自動駕駛 vs 導航 |
| MCP | 讓 AI 連接外部工具的協議 | USB 接口 |
| Token | AI 處理文字的單位 | 計費單位 |
