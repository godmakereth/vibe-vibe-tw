# 1.4.2 所有工具的共同能力

上一節我們看了五大類工具。工具雖多，但它們都有一些"共同的核心能力"。

理解了這些，你換任何工具都能快速上手。就像學會了開車，換什麼品牌的車都能開。


## 五大核心能力

![五大核心能力](/images/5_ablities.png)

### 能力 1：自然語言生成代碼

**你說中文，它寫代碼。**

這是所有 AI 編程工具最核心的能力。你不需要記住任何編程語法，只要能把想法說清楚就行。

```
你輸入：「幫我做一個按鈕，點擊後顯示"你好"」

AI 輸出：
<button onclick="alert('你好')">點我</button>
```

> **關鍵心法**：描述越具體，結果越好。不是"做個按鈕"，而是"做一個藍色的圓角按鈕，點擊後彈出'你好世界'"。


### 能力 2：理解上下文

**AI 能"看到"你的項目，給出更精準的建議。**

普通聊天機器人每次對話都是"失憶"的。但專業的 AI 編程工具能記住：
- 你之前說了什麼
- 你的項目裏有哪些文件
- 代碼之間是怎麼關聯的

這就像請了一個助手，他不僅聽你說話，還看過你的所有文檔。

```
場景：你的項目裏已經有一個 user.js 文件

你說：「在首頁顯示用戶名」

普通 AI：可能給你一個不兼容的方案
專業工具：會讀取 user.js，用你已有的變量和函數
```

::: tip 爲什麼專業工具比 ChatGPT 好用？
普通聊天 AI（如網頁版 ChatGPT）每次對話都是"失憶"的，你需要反覆粘貼代碼。

而 Cursor、Windsurf 這類工具能直接"看到"你的整個項目，理解文件之間的關係。這就是爲什麼我們推薦使用專業的 AI 編程工具，而不是用通用聊天 AI。
:::


### 能力 3：解釋和修改代碼

**看不懂的代碼，問它就行。**

遇到不理解的代碼？選中它，問 AI：
- "這段代碼是做什麼的？"
- "爲什麼要這樣寫？"
- "能不能用更簡單的方式？"

AI 會用你能理解的語言解釋，還能幫你改寫。

```
你選中一段代碼，問：「這是什麼意思？」

AI 回答：
「這段代碼的作用是：
1. 獲取用戶輸入的文字
2. 檢查是否爲空
3. 如果不爲空，就保存到本地存儲

簡單說，就是"保存用戶填寫的內容"」
```


### 能力 4：多輪對話迭代

**不滿意？繼續聊，直到滿意爲止。**

AI 編程不是"一次成功"，而是"對話式改進"。

```
第 1 輪：「幫我做一個計時器」
→ AI 生成了基礎版本

第 2 輪：「把數字變大一點」
→ AI 調整了字體大小

第 3 輪：「加一個暫停按鈕」
→ AI 添加了暫停功能

第 4 輪：「顏色換成藍色」
→ AI 修改了配色
```

每一輪對話都是在上一輪的基礎上改進。這就是 Vibe Coding 的核心節奏：**說 → 看 → 改 → 說 → 看 → 改...**


### 能力 5：錯誤診斷和修復

**報錯了？把錯誤信息丟給 AI，它幫你修。**

這可能是最實用的能力。當你的代碼出問題時：

1. 複製錯誤信息
2. 發給 AI
3. AI 告訴你哪裏錯了、怎麼改

```
你說：「報錯了：Cannot read property 'length' of undefined」

AI 回答：
「這個錯誤的意思是：你在讀取一個不存在的東西的長度。

可能的原因：
1. 變量還沒有賦值就使用了
2. 數據還沒加載完就訪問了

修復方法：在使用前先檢查變量是否存在。

修改後的代碼：
if (myArray && myArray.length > 0) {
  // 你的代碼
}」
```


## 這些能力意味着什麼？

理解了這五大能力，你會發現：

| 過去（不會編程） | 現在（有 AI 幫忙） |
|-----------------|-------------------|
| 看不懂代碼 | 讓 AI 解釋給你聽 |
| 不會寫代碼 | 用中文描述，AI 來寫 |
| 報錯不知所措 | 複製錯誤，AI 幫你修 |
| 改動怕改崩 | 告訴 AI 你想改什麼 |
| 功能不會加 | 對話式一步步加 |

**你需要學的不是"編程語言"，而是"如何清楚地表達你想要什麼"。**


## 本節小結

所有 AI 編程工具都有五大共同能力：

1. **自然語言生成代碼** — 你說中文，它寫代碼
2. **理解上下文** — 它知道你的項目長什麼樣
3. **解釋和修改代碼** — 看不懂就問，想改就說
4. **多輪對話迭代** — 不滿意就繼續聊
5. **錯誤診斷和修復** — 報錯了丟給它
