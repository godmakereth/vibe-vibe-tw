# 1.4.4 本教程的工具策略

你可能在想：「這個教程到底用什麼工具？我要不要跟着裝一樣的？」

這一節，我們來聊聊本教程的"工具哲學"。


## 我們的策略：不綁定任何工具

**本教程不會要求你必須用某個特定工具。**

爲什麼？三個原因：

### 原因 1：工具變化太快

2024 年最火的工具，2025 年可能就被新工具超越了。

- Cursor 曾經一家獨大，現在 Windsurf 在追趕
- Bolt.new 去年纔出現，今年已經是零基礎首選
- 明年又會有新的工具出來

如果我們綁定某個工具，這份教程很快就會過時。

### 原因 2：方法論比工具重要

我們教的是**思維方式**，不是**按鈕在哪**。

當你學會了：
- 如何把想法拆解成 AI 能理解的描述
- 如何迭代式地改進
- 如何 Debug（修復問題）

這些能力在任何工具上都能用。工具只是載體，方法論纔是核心。

### 原因 3：每個人情況不同

- 有人能裝軟件，有人只能用網頁
- 有人在國內，有人在國外
- 有人願意付費，有人想先用免費的

強制大家用同一個工具，反而會給很多人造成障礙。


## 本教程的使用方式

### 我們會怎麼寫示例

在教程中，當我們需要演示操作時：

1. **給出通用的 Prompt**（提示詞），而不是具體的按鈕操作
2. **解釋背後的思路**，而不是死記硬背的步驟
3. **偶爾用某個工具截圖**，但會說明"其他工具操作類似"

```
✅ 我們會這樣寫：
「把這段描述發給 AI：幫我做一個計時器，能開始、暫停、重置」

❌ 我們不會這樣寫：
「打開 Cursor，點擊左上角的 Chat，輸入...」
```

### 你應該怎麼跟着學

1. **選一個你能用的工具**（參考上一節的推薦）
2. **把教程裏的 Prompt 複製過去**
3. **看效果，對比教程的預期結果**
4. **如果不一樣，問 AI 幫你調整**

不同工具的界面不同，但核心流程是一樣的：**描述需求 → 看結果 → 迭代改進**。


## 本節小結

- **本教程不綁定特定工具**，教的是通用的方法論
- **工具選擇**：參考 [1.4.3](./1.4.3-logic-interaction.md) 選類別，[1.4.1](./1.4.1-env-setup.md) 選具體工具
- **方法論比工具重要**：學會思路，換工具只是換界面
