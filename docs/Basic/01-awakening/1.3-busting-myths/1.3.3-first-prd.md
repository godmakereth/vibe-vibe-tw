# 1.3.3 迷信三："需要懂很多技術名詞"

> "什麼 API、框架、後端、數據庫……聽都聽不懂，怎麼學？"

如果你聽到這些詞就頭大，我理解你。

但我要告訴你一個好消息：**你不需要先學會這些詞，再開始做東西。**


## 傳統學習 vs AI 時代學習

**傳統方式：**
```
學概念 → 學語法 → 學框架 → 做項目
（可能需要半年到一年）
```

**AI 時代的方式：**
```
說出你想要的 → 做出來 → 遇到問題再學
（可能只需要一下午）
```

**順序完全反過來了。**

你不需要先搞懂什麼是"前端"、"後端"、"數據庫"，才能做一個網站。

你只需要說："幫我做一個能記錄待辦事項的網頁"。

AI 會幫你處理所有技術細節。


## 那些名詞到底是什麼？

如果你好奇，我用最簡單的話解釋幾個常見的：

| 技術名詞 | 大白話解釋 |
|---------|-----------|
| 前端 | 用戶能看到的部分（頁面、按鈕、顏色） |
| 後端 | 用戶看不到的部分（數據處理、存儲） |
| API | 兩個程序之間的"傳話筒" |
| 數據庫 | 存數據的地方，像個超級 Excel |
| 框架 | 別人寫好的代碼模板，讓你不用從零開始 |
| 部署 | 把你的網站放到網上，讓別人能訪問 |

**知道這些就夠了。** 用到的時候再深入瞭解。


## 邊做邊學的正確姿勢

與其先啃概念，不如：

### Step 1：先動手做
不管懂不懂，先讓 AI 幫你生成代碼，看看效果。

### Step 2：遇到問題，問 AI
```
AI 說要配置一個 API，這是什麼意思？
能用更簡單的話解釋一下嗎？
```

### Step 3：記住"夠用就好"
不需要完全理解每個概念。知道它是幹什麼的，能用就行。

### Step 4：逐漸積累
做的項目多了，這些詞自然就熟悉了。


## 真實案例

**張老師，45 歲，小學語文老師**

她想做一個"學生作業提交"的小網頁。

一開始，她完全不懂什麼是"前端"、"後端"。

她就直接告訴 AI：

```
我想做一個網頁，學生可以輸入自己的名字，
然後上傳作業文件，我能看到誰提交了。
```

AI 幫她生成了代碼。她看了效果，發現可以用。

**三個月後**，她已經知道：
- "前端"就是學生看到的提交頁面
- "後端"就是處理上傳文件的部分
- "數據庫"就是存儲提交記錄的地方

這些概念，她是**做完項目之後**才真正理解的。

她說：

> "如果讓我先學這些概念，我可能學兩週就放棄了。但先做出來，再回頭看，一下子就懂了。"


## 一個小提醒

雖然不需要先學概念，但有一個能力很重要：

**願意搜索和提問。**

遇到不懂的詞，你可以：
- 直接問 AI："XX 是什麼意思？"
- 用搜索引擎查一下
- 在本教程的附錄術語表裏找

**好奇心比知識儲備更重要。**


## 小結

| 迷信 | 真相 |
|-----|------|
| 要先學會技術名詞 | 邊做邊學，遇到再查 |
| 不懂概念就做不了 | AI 會幫你處理技術細節 |
| 學編程要從基礎學起 | 先做出東西，再補基礎 |

**不要讓"不懂"成爲"不做"的藉口。**


::: tip 給自己的話
學習新東西，最有效的方式不是"先學後用"，而是"邊用邊學"。

就像學游泳。你可以在岸上看一年的教程，也可以直接跳下水撲騰。

哪種方式學得更快？你懂的。

AI 就是你的游泳圈。先跳下去，別怕。
