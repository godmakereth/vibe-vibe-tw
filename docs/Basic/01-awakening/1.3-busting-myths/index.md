# 1.3 破除迷信：不懂英文、數學不好也能寫代碼嗎？

在開始動手之前，我想先幫你清除一些心理障礙。

這些年，我遇到過太多人，一聽到"編程"兩個字就搖頭：

> "我數學不好，肯定學不會。"
> "我英語太差了，代碼都是英文的。"
> "要懂那麼多技術名詞，太難了。"

如果你也有這些想法，別擔心。

**這些都是迷信。**

在 AI 編程時代，這些"門檻"已經被大大降低，甚至消失了。


## 本章你將學到

| 小節 | 迷信 | 真相 |
|------|------|------|
| 1.3.1 | "編程需要數學很好" | 日常編程 95% 用不到高深數學 |
| 1.3.2 | "要先學好英文" | 中文 Prompt 完全可行 |
| 1.3.3 | "需要懂很多技術名詞" | 邊做邊學，AI 會解釋 |
| 1.3.4 | "Vibe Coding 只是玩具" | 有邊界，但能做很多事 |
| 1.3.5 | "AI 會搶走程序員的工作" | 協作而非替代 |
| 1.3.6 | "AI 生成的代碼總是完美的" | AI 會犯錯，需要驗證 |


## 爲什麼要先破除迷信？

因爲**心理障礙比技術障礙更難跨越**。

很多人不是學不會，而是還沒開始就把自己嚇退了。

我見過數學只考 60 分的人，用 AI 做出了漂亮的網站。
我見過從沒寫過代碼的 HR，用 AI 做了一個面試排班工具。
我見過英語四級沒過的創業者，用 AI 做出了產品原型。

他們的共同點是：**先試了，再說。**

這一章，我要幫你卸下心理包袱。

看完這章，你會發現：原來這些"門檻"，根本不是門檻。

但如果你對自己有任何懷疑，請認真讀完。**信心，是學習最好的燃料。**
