# 1.3.2 迷信二："要先學好英文"

> "代碼都是英文的，我英語不好，肯定看不懂。"

這個擔憂，放在五年前可能有點道理。

但現在？**完全不用擔心。**


## 2025 年，AI 懂中文

現在的 AI 編程工具，幾乎都能理解中文。

你完全可以這樣和 AI 對話：

```
幫我做一個網頁，顯示當前的北京時間，
背景用淺藍色，字體要大一點。
```

AI 會直接生成代碼。你不需要寫一個英文單詞。


## 中文 Prompt vs 英文 Prompt

很多人問：用英文是不是效果更好？

**對於大多數場景，差別不大。**

| 場景 | 中文表現 | 建議 |
|------|---------|------|
| 日常開發需求 | ✅ 完全夠用 | 用中文 |
| 描述界面樣式 | ✅ 完全夠用 | 用中文 |
| 解釋代碼報錯 | ✅ 完全夠用 | 用中文 |
| 涉及專業術語 | ⚠️ 偶爾需要英文 | 混用即可 |

舉個例子，這兩個 Prompt 效果幾乎一樣：

**中文版：**
```
做一個按鈕，點擊後顯示"你好世界"
```

**英文版：**
```
Create a button that shows "Hello World" when clicked
```


## 實用建議：混用策略

最佳實踐是**中英混用**：

- 描述需求用中文（說得更清楚）
- 遇到專業術語保留英文（減少歧義）

比如：

```
幫我做一個 login 頁面，包含用戶名和密碼輸入框，
用 flexbox 居中顯示，樣式參考 Material Design。
```

這樣的混用方式，AI 完全能理解。


## 那代碼裏的英文呢？

代碼本身確實是英文的。但你需要"精通"它嗎？

**不需要。**

你需要做的是：

1. **告訴 AI 你想要什麼**（用中文）
2. **驗證效果對不對**（運行看結果）
3. **不對就告訴 AI 哪裏不對**（用中文描述問題）

起步階段，你不需要看懂每一行代碼。但隨着使用次數增多，你會自然地開始認識一些常見的英文單詞和代碼結構——這不是必須的，但會讓你和 AI 協作更順暢。

好消息是：代碼裏的英文單詞就那麼幾十個常用的。用着用着就熟了。


## 真實案例

**李阿姨，58 歲，英語只會說 "Hello"**

她女兒教她用 AI 做了一個"菜譜收藏"網頁。

整個過程：

1. 李阿姨說："我想把我的拿手菜做成一個網頁"
2. 女兒幫她打成文字發給 AI
3. AI 生成了網頁
4. 李阿姨說："字太小了"，女兒再發給 AI
5. 搞定

李阿姨全程沒寫一個英文單詞。

她說：

> "我以爲要學英語才能做網站，結果跟說話一樣。"


## 小結

| 迷信 | 真相 |
|-----|------|
| 編程必須用英文 | 中文 Prompt 完全可行 |
| 代碼是英文的，看不懂 | Vibe Coding 不需要讀懂代碼 |
| 英文好才能學好編程 | 表達清楚比英語好重要 |

**語言不是障礙。AI 是你的翻譯官。**


::: tip 給自己的話
如果你英語不好，沒關係。

先用中文把想法說清楚，這是最重要的。

等你做出幾個項目，自然會慢慢認識一些英文單詞。

那時候再學，有目標、有動力，效率反而更高。
:::
