# 1.3.4 迷信四："Vibe Coding 只是玩具"

> "用 AI 寫代碼？那不就是玩玩嗎，能做正經東西嗎？"

這個問題問得好。

誠實的回答是：**有邊界，但能做的事情比你想象的多得多。**


## Vibe Coding 能做什麼

先說能做的。這個清單可能會讓你驚訝：

### 個人工具類
- ✅ 待辦清單、記賬本、習慣打卡
- ✅ 個人博客、作品集網站
- ✅ 簡歷頁面、個人主頁
- ✅ 學習筆記整理工具

### 工作效率類
- ✅ 數據可視化看板
- ✅ 表單收集工具
- ✅ 簡單的自動化腳本
- ✅ 內部使用的小工具

### 創意項目類
- ✅ 互動網頁、小遊戲
- ✅ 活動邀請函、電子賀卡
- ✅ 產品原型、概念驗證
- ✅ 數據分析報告

### 學習項目類
- ✅ 課程作業、畢業設計原型
- ✅ 技術概念驗證
- ✅ 個人技能展示

**這些，Vibe Coding 都能搞定。**


## Vibe Coding 不適合什麼

但也有一些場景，純 Vibe Coding 確實不太夠：

| 場景 | 爲什麼不適合 |
|------|-------------|
| 涉及資金交易 | 安全性要求高，需要專業審計 |
| 處理敏感數據 | 隱私合規要求，不能靠"差不多" |
| 高併發系統 | 性能優化需要專業知識 |
| 長期維護的產品 | 代碼質量和可維護性很重要 |
| 對外發布的商業產品 | 需要更嚴謹的開發流程 |

**這些場景需要專業開發者介入**，或者你需要進階學習更多知識。


## 正確的定位

把 Vibe Coding 想象成**瑞士軍刀**：

- 它不是專業的廚師刀，切不了最複雜的菜
- 但它能幫你解決 80% 的日常需求
- 而且隨身攜帶，隨時能用

**Vibe Coding 的價值在於：**

1. **快速驗證想法**：幾小時做出原型，而不是幾周
2. **降低入門門檻**：不需要先學半年再動手
3. **自己解決小問題**：不用等技術部門排期
4. **學習的起點**：從做出東西開始，再深入學習


## 真實數據

來看看一些有意思的數據：

**Y Combinator 2025 年的統計：**
> 25% 的創業公司，代碼的 95% 是 AI 生成的

這些不是玩具項目，而是拿到投資、有用戶的真實產品。

**GitHub 2024 年的調查：**
> 使用 AI 編程工具的開發者，編碼速度提升 55%

連專業開發者都在用 AI 輔助。

**關鍵不在於"用不用 AI"，而在於"用得好不好"。**


## 從"玩具"到"工具"的進階路徑

如果你擔心 Vibe Coding 只是玩玩：

### 階段 1：純 Vibe
- 用自然語言描述需求
- AI 生成代碼
- 看效果、調整、迭代

**你能做出**：個人工具、簡單網頁

### 階段 2：Vibe + 代碼閱讀
- 開始看懂 AI 生成的代碼
- 理解大致的結構和邏輯

**你能做出**：更復雜的應用、開始修改細節

### 階段 3：Vibe + 手動修改
- 能自己改一些小地方
- 知道什麼時候 AI 出錯了

**你能做出**：質量更高的項目、能解決更多問題

### 階段 4：AI 輔助開發
- AI 是你的助手，不是唯一依賴
- 有獨立的技術判斷能力

**你能做出**：商業級產品、專業項目

**本教程會帶你從階段 1 走到階段 2。** 進階版會帶你走得更遠。


## 小結

| 迷信 | 真相 |
|-----|------|
| Vibe Coding 只能做玩具 | 能做很多實用的東西 |
| AI 生成的代碼不靠譜 | 用對了場景，完全夠用 |
| 專業開發不需要 AI | 連專業開發者都在用 |

**Vibe Coding 是工具，不是玩具。**

**關鍵在於你用它來做什麼。**


::: warning 重要提醒
Vibe Coding 有邊界，這很正常。

任何工具都有適用範圍。

知道它能做什麼、不能做什麼，比盲目吹捧或貶低都重要。
