# 第 2 章：心法 —— 像產品經理一樣思考

> **章節導語**
> 
> 你即將學到的，不是"如何給AI寫更好的提示詞"——那是第三章的事。
> 
> 這一章要解決的是一個更根本的問題：**在你開口告訴AI"幫我做xxx"之前，你應該想清楚什麼？**
> 
> 90%的Vibe Coding失敗，不是因爲AI不夠聰明，而是因爲你自己都沒想清楚要做什麼。
> 
> 讀完這一章，你會獲得一套產品經理級別的思維工具。從此以後，無論你想做什麼——一個小工具、一個數據分析、一個自動化腳本，還是一個產品原型——你都會知道如何思考。

> **預計閱讀時間**：完整閱讀約 2 小時 | 快速通關約 30 分鐘



## 經過本章學習，你將掌握

- **七個思維模型**：JTBD、逆向思維、減法思維、故事思維、靈魂三問、場景應用、問題發現
- **一套開發前檢查流程**：在動手之前確保想清楚了
- **從"有想法"到"準備好開發"的完整思考鏈條**

## 本章適用於你想做的任何事情

在開始之前，我想先說清楚一件事：

**這一章不只是關於"做產品"或"創業"的。**

無論你想用Vibe Coding做什麼，這些思維方式都適用：

| 你想做的事情 | 本章對你的幫助 |
|-------------|---------------|
| **解決個人痛點的小工具** | 幫你想清楚"我到底要解決什麼問題" |
| **數據分析和可視化** | 幫你明確"我要回答什麼問題，給誰看" |
| **自動化腳本** | 幫你識別"哪些重複勞動值得自動化" |
| **產品原型驗證** | 幫你設計"最小可驗證版本" |
| **學習項目練手** | 幫你選擇"適合自己水平的項目" |
| **給家人做個小網頁** | 幫你理解"他們真正需要什麼" |

**核心思想只有一個：先想清楚，再動手。**

這聽起來很簡單，但大多數人（包括我自己）都會跳過這一步，直接告訴AI"幫我做一個xxx"。

結果就是：AI做出來的東西，和你想要的總是差那麼一點。

不是AI的問題，是你沒想清楚。
