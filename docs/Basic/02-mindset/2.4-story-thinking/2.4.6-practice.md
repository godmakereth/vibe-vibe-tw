# 2.4.6 實戰練習：爲你的項目應用故事思維

現在，讓我們把本節學到的故事思維工具應用到你自己的項目中。

準備好你想做的那個項目想法，跟着下面的練習一步步完成。


## 練習一：寫一個用戶故事

用「開端→衝突→解決→結局」的結構，爲你的項目寫一個用戶故事。

### 模板

```markdown
## 用戶故事：[給你的用戶起個名字]

### 開端：用戶的日常
[描述用戶是誰，他的日常生活/工作是什麼樣的]
（1-2 句話）

### 衝突：用戶的困境
[描述用戶遇到了什麼問題，爲什麼這讓他困擾]
（2-3 句話）

### 解決：用戶發現你的產品
[描述用戶如何使用你的產品，解決了什麼問題]
（1-2 句話）

### 結局：用戶獲得的價值
[描述問題解決後，用戶的生活/工作有什麼改變，他有什麼感受]
（1-2 句話）
```

### 示例

```markdown
## 用戶故事：小王

### 開端：用戶的日常
小王是一個喜歡讀書的產品經理，每天通勤時會在手機上看電子書。

### 衝突：用戶的困境
他看書時經常劃線標記好句子，但這些劃線散落在不同的書裏。
想回顧時要一本本翻，很麻煩，慢慢就不劃了。
他覺得很多好內容「看過就忘」，很可惜。

### 解決：用戶發現你的產品
他發現了一個工具，能自動收集所有書的劃線，按書籍整理好。

### 結局：用戶獲得的價值
現在他通勤時經常翻翻之前的劃線，覺得「讀過的書真正變成了自己的」。
```


## 練習二：畫一個三維用戶畫像

用三維構建法，爲你的目標用戶畫一個立體的畫像。

### 模板

```markdown
## 用戶畫像：[名字]

### 第一維：表面屬性
- 年齡/職業/生活狀態：
- 技術熟練度：
- 一句話描述這個人：

### 第二維：行爲習慣
- 典型使用場景（時間、地點、狀態）：
- 現在怎麼解決這個問題：
- 試過什麼方法，爲什麼沒用：

### 第三維：深層動機
- 表面上他說需要什麼：
- 實際上他害怕什麼：
- 真正渴望什麼感受：
```

### 小技巧：連問 5 次「爲什麼」

填寫第三維時，對你寫的每個答案問「爲什麼」：

| 問題 | 我的回答 |
|-----|---------|
| 用戶需要什麼？ | [你的回答] |
| 爲什麼需要這個？ | [繼續追問] |
| 爲什麼這很重要？ | [繼續追問] |
| 如果沒有會怎樣？ | [繼續追問] |
| 他最擔心的是什麼？ | [這通常就是深層動機] |


## 練習三：畫一個簡化版用戶旅程

爲你的用戶畫一個簡化版的旅程地圖，關注四個關鍵階段。

### 模板

```markdown
## 簡化版用戶旅程

### 階段 1：發現
- 用戶如何知道這個產品/工具：
- 用戶的第一反應：
- 這個階段的關鍵挑戰：

### 階段 2：首次使用
- 用戶第一步會做什麼：
- 可能遇到的困惑：
- 期望給用戶什麼感受：

### 階段 3：核心流程（日常使用）
- 用戶最常重複的操作是什麼：
- 這個流程目前順暢嗎（1-5 分）：
- 如何讓這個流程更好：

### 階段 4：長期價值
- 用戶爲什麼會持續使用：
- 什麼會讓用戶推薦給別人：
```


## 自檢清單

完成上面的練習後，用這個清單檢查你的故事是否足夠立體：

### 故事完整性檢查

- [ ] 我能清楚說出用戶是誰（有名字、有形象）
- [ ] 我能描述用戶的日常（典型的一天/一週）
- [ ] 我能說出用戶現在用什麼方法解決問題
- [ ] 我知道用戶爲什麼對現有方法不滿意
- [ ] 我理解用戶的深層動機（恐懼/渴望/期待）

### 設計指導性檢查

- [ ] 基於這個故事，我能判斷「這個功能該不該做」
- [ ] 基於這個故事，我能判斷「哪個功能更重要」
- [ ] 這個故事能幫助 AI 理解我的需求
- [ ] 讀這個故事，別人也能「看到」這個用戶

### 常見問題排查

| 問題 | 症狀 | 解決方法 |
|-----|------|---------|
| 用戶太抽象 | "年輕人""職場人士" | 具體到一個人，給他起個名字 |
| 只有表面屬性 | 只寫了年齡職業 | 補充行爲習慣和深層動機 |
| 痛點不夠深 | "想要更方便" | 連問 5 次「爲什麼」 |
| 故事是自己的 | 把自己當用戶 | 問「真正的用戶會這樣想嗎」 |


## 進階：把練習成果變成 Prompt

完成練習後，你可以用這個模板把成果整合成一個故事化 Prompt：

```markdown
我想做 [你的項目]，先介紹一下背景：

## 我是誰
[從練習二的第一維提取]

## 我的現狀
[從練習二的第二維提取]

## 我的痛點
[從練習二的第三維提取]

## 我的期望
[從練習一的「結局」提取]

## 用戶旅程
[從練習三提取，可以簡化爲幾個關鍵點]

請幫我 [設計/實現/規劃] 這個項目。
```


## 本節要點

三個練習幫你把故事思維落地到自己的項目：

1. **用戶故事**：用四幕結構描述用戶從困境到解決的過程
2. **三維畫像**：從表面屬性、行爲習慣、深層動機三個維度理解用戶
3. **用戶旅程**：關注發現、首次使用、核心流程、長期價值四個階段

完成這些練習後，你就擁有了一份可以直接用於 Prompt 寫作的「用戶故事素材庫」。
