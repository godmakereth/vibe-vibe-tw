# 2.4.5 用故事思維寫 Prompt：讓 AI 真正理解你

## 同樣的需求，不同的結果

來看兩個 Prompt 的對比：

### 無故事版本

```
幫我做一個待辦清單工具，需要這些功能：
1. 添加任務
2. 完成任務
3. 查看任務列表
```

### 有故事版本

```
我想做一個待辦清單工具，先介紹一下背景：

我是一個互聯網公司的運營，每天大概有 10 件以上的事要處理——工作任務、要回的消息、要買的東西、要處理的雜事。

我試過用便籤紙，但經常丟；試過幾個待辦 App，但都太複雜了，光是錄入任務就累得不想用。我的核心痛點是：怕忘事，腦子裏總裝着「還有什麼沒做完」的焦慮。

我希望有一個工具能：
- 打開就能添加，不用選分類、填日期、設優先級
- 一眼看到今天要做什麼
- 做完一件就劃掉，獲得那種「搞定」的爽感

請幫我設計這個工具的界面和核心功能。
```

第二個 Prompt 比第一個長很多，但 AI 的輸出會有本質區別。


## 區別在哪裏

| 方面 | 無故事版本 | 有故事版本 |
|-----|-----------|-----------|
| **AI 知道用戶是誰** | ❌ 不知道 | ✅ 忙碌的職場人 |
| **AI 知道使用場景** | ❌ 不知道 | ✅ 每天處理 10+ 事項 |
| **AI 知道約束條件** | ❌ 不知道 | ✅ 用過複雜 App，需要極簡 |
| **AI 知道核心痛點** | ❌ 不知道 | ✅ 怕忘事、怕焦慮 |
| **AI 能推斷優先級** | ❌ 只能猜 | ✅ 簡單 > 功能多 |

有了故事背景，AI 可以做出更合理的設計決策：

- 界面會更簡潔，而不是堆滿功能
- 添加任務會設計成一步完成，而不是多個表單
- 完成任務時會加入滿足感的反饋


## 故事化 Prompt 的四個要素

一個好的故事化 Prompt 應該包含四個要素：

| 要素 | 回答的問題 | 作用 |
|-----|-----------|------|
| **身份** | 我是誰？ | 讓 AI 理解用戶特徵 |
| **現狀** | 我現在怎麼做？遇到什麼問題？ | 讓 AI 理解背景和約束 |
| **痛點** | 我最困擾的是什麼？ | 讓 AI 明確核心問題 |
| **期望** | 我希望達到什麼效果？ | 讓 AI 知道成功標準 |


## 通用故事化 Prompt 模板

可以直接複製使用的模板：

```markdown
我想做 [項目類型]，先介紹一下背景：

## 我是誰
[描述用戶身份、基本情況]

## 我的現狀
[描述現在是怎麼做的、用什麼工具、有什麼問題]

## 我的痛點
[描述最困擾的問題、爲什麼困擾]

## 我的期望
[描述希望達到的效果，包括功能和感受]

請幫我 [具體請求：設計/實現/分析]。
```


## 多場景 Prompt 示例

### 場景一：做產品

```markdown
我想做一個習慣打卡工具，先介紹一下背景：

## 我是誰
我是一個想養成早起習慣的上班族，之前堅持過很多次都失敗了。

## 我的現狀
- 用過幾個習慣 App，界面花哨、功能太多
- 每天要點好幾下才能完成打卡，煩得不想用
- 最終都是用了幾天就放棄了

## 我的痛點
打卡太麻煩，讓我覺得「打卡」本身成了負擔，而不是幫助。

## 我的期望
- 打開就能打卡，最好就一個按鈕
- 能看到連續打卡天數，給我堅持的動力
- 界面極簡，沒有多餘的東西分散注意力

請幫我設計這個工具的界面和交互流程。
```

### 場景二：數據分析

```markdown
我需要做一份銷售數據分析，先介紹一下背景：

## 我是誰
我是一個電商公司的運營，需要向老闆彙報 Q3 的銷售情況。

## 我的現狀
- 手裏有三個月的訂單數據（Excel 格式）
- 老闆每週一早上只有 15 分鐘看報告
- 上次做的報告太長，老闆看了兩頁就放下了

## 我的痛點
不知道老闆最關心什麼，報告總是抓不住重點。

## 我的期望
- 一份老闆能在 5 分鐘內看完的報告
- 第一頁就能看到最重要的結論
- 有明確的行動建議，不只是數據展示

請幫我規劃這份報告的結構和核心圖表。
```

### 場景三：自動化腳本

```markdown
我想寫一個自動化腳本，先介紹一下背景：

## 我是誰
我是公司的財務，每週五要彙總 5 個部門的週報。

## 我的現狀
- 每週五下午花 2 小時做這個事情
- 流程是：打開 5 個 Excel → 複製數據 → 粘貼到彙總表 → 調整格式
- 有時候會漏數據或者格式出錯，要反覆檢查

## 我的痛點
這個工作重複又無聊，但不能出錯，所以特別累心。

## 我的期望
- 點一下就能自動彙總
- 輸出的格式和我手動做的一樣
- 我只需要快速檢查一下結果就行

請幫我設計這個腳本的邏輯，用 Python 實現。
```

### 場景四：給家人做工具

```markdown
我想給我 60 歲的媽媽做一個喫藥提醒，先介紹一下背景：

## 我是誰（用戶是我媽媽）
- 62 歲，退休教師，獨居
- 有高血壓，每天早晚各喫一次藥
- 會用微信、看短視頻，其他 App 用得少

## 她的現狀
- 靠記憶喫藥，經常忘記自己喫過沒有
- 有時候重複喫，有時候忘喫
- 不願意麻煩我，但我其實很擔心

## 她的痛點
記性不好，但又不想承認自己記性不好。

## 她的期望
- 字要大，她眼神不太好
- 操作要少，最好就一個按鈕
- 能讓我遠程看到她今天喫藥沒有

請幫我設計一個她能用的喫藥提醒頁面。
```


## AI 輸出對比

以待辦清單爲例，看看兩種 Prompt 的 AI 輸出差異：

### 無故事版本的典型輸出

```
這是一個基礎的待辦清單設計：

1. 頂部有一個輸入框，用於添加任務
2. 下方是任務列表，每個任務有：
   - 複選框（標記完成）
   - 任務文本
   - 刪除按鈕
3. 可以考慮添加以下功能：
   - 任務分類
   - 截止日期
   - 優先級標籤
   ...
```

### 有故事版本的典型輸出

```
基於你的需求，我建議一個極簡設計：

界面只有兩個元素：
1. 一個大輸入框，點擊就能開始輸入，回車直接添加
2. 任務列表，點擊任務就標記完成（有劃線動畫和輕微震動反饋）

設計原則：
- 沒有分類、日期、優先級——減少錄入負擔
- 完成動畫要有滿足感——一個簡單的「√」動畫
- 整體色調簡潔，不花哨

特別說明：
- 考慮到你說「錄入比做事還累」，所以刪掉了所有非必要字段
- 考慮到你說「搞定的爽感」，完成時會有小動畫
...
```

區別很明顯：有故事的版本給出了更貼合需求的設計，而且能解釋「爲什麼這樣設計」。


## 本節要點

故事化 Prompt 不是爲了「寫得長」，而是爲了「讓 AI 真正理解你」：

- 四個要素：身份、現狀、痛點、期望
- 好的 Prompt 讓 AI 能做出推斷，而不只是執行指令
- 模板可以直接複製使用，根據場景填充內容
