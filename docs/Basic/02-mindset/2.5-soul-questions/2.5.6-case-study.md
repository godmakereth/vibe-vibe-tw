# 2.5.6 實戰案例：四種場景的靈魂三問

靈魂三問不只是用來"做產品"的。下面用四個不同場景的案例，展示這套框架的實際應用。


## 案例一：待辦清單工具（產品場景）

**原始想法**："我想做一個待辦清單"

### 靈魂三問分析

| 問題 | 分析過程 | 最終答案 |
|-----|---------|---------|
| 用戶是誰？ | "想管理任務的人"太寬泛 → 聚焦到"事情多、怕忘事的職場人" → 給個名字 | 小李，28歲運營，每天有10+件事要處理 |
| 痛點在哪？ | "想管理任務"是表面 → 追問爲什麼 → 發現是"總擔心忘了什麼重要的事"的焦慮 | 害怕遺漏重要事項，想要"一切盡在掌握"的安心感 |
| 爲什麼選你？ | 調研發現現有App功能太多太複雜 → 差異點：極簡到只有三個功能 | 最簡單的待辦工具，打開就能用，3秒添加任務 |

### 自檢結果

- 用戶是誰：🟢 具體到"28歲運營小李"
- 痛點在哪：🟢 有情緒（焦慮）、高頻（每天）
- 爲什麼選你：🟢 有差異（極簡 vs 複雜）

**結論：可以開始**


## 案例二：Q3銷售分析報告（數據分析場景）

**原始想法**："老闆讓我做一個Q3銷售分析"

### 靈魂三問分析

| 問題 | 分析過程 | 最終答案 |
|-----|---------|---------|
| 用戶是誰？ | 不是"公司"，是"看這個報告的人" → 是銷售總監王總 | 王總，45歲銷售總監，每週一早上有30分鐘看報表 |
| 痛點在哪？ | 不是"想看數據"，是"需要快速做出決策" → 追問發現是"怕做錯決策、浪費資源" | 擔心看不清趨勢、做出錯誤判斷，需要"一眼看出該關注什麼" |
| 爲什麼選你？ | 這是內部任務，沒有"競品" → 問題變成"爲什麼是這種呈現方式" | 30分鐘內能看完、能直接得出結論、不需要再追問細節 |

### 自檢結果

- 用戶是誰：🟢 具體到"王總，週一早上看報表"
- 痛點在哪：🟢 有情緒（擔心決策錯誤）、有場景
- 爲什麼選你：🟡 "爲什麼這樣呈現"還需要細化

**結論：基本可以開始，呈現方式需要再想想**

### 延伸思考

數據分析場景的"爲什麼選你"，核心是**呈現方式的選擇**：

- 爲什麼用柱狀圖而不是表格？因爲對比更直觀
- 爲什麼只放5個關鍵指標？因爲王總只有30分鐘
- 爲什麼先放結論後放數據？因爲決策者要先知道"該怎麼辦"


## 案例三：Excel自動彙總腳本（自動化場景）

**原始想法**："每週彙總Excel太煩了，想寫個腳本"

### 靈魂三問分析

| 問題 | 分析過程 | 最終答案 |
|-----|---------|---------|
| 用戶是誰？ | 就是我自己 → 但要具體化 | 每週五下午需要彙總5個部門Excel的我 |
| 痛點在哪？ | "太煩"不夠具體 → 追問發現是"每週浪費2小時做重複勞動，還容易出錯" | 每週浪費2小時、容易複製粘貼出錯、做完很煩躁 |
| 爲什麼選你？ | 問"爲什麼值得自己做" → 計算投入產出 | 投入3小時開發，每週省2小時，3週迴本，一年省100+小時 |

### 自檢結果

- 用戶是誰：🟢 具體到"週五下午彙總Excel的我"
- 痛點在哪：🟢 有頻率（每週）、有時間成本（2小時）、有情緒（煩躁）
- 爲什麼選你：🟢 投入產出算清楚了（3小時換100小時）

**結論：可以開始**

### 投入產出計算

| 項目 | 數值 |
|-----|------|
| 每次手動操作時間 | 2小時 |
| 每年發生次數 | 52次 |
| 每年總耗時 | 104小時 |
| 預計開發時間 | 3小時 |
| 回本週期 | 1.5周 |
| 一年淨節省 | 101小時 |


## 案例四：給媽媽做喫藥提醒（給家人場景）

**原始想法**："想給媽媽做一個喫藥提醒的小工具"

### 靈魂三問分析

| 問題 | 分析過程 | 最終答案 |
|-----|---------|---------|
| 用戶是誰？ | 不是"老年人"，是"我媽媽" → 要具體描述她的情況 | 62歲的媽媽，會發微信語音，其他App用得少，眼神不太好 |
| 痛點在哪？ | "忘喫藥"是表面 → 追問發現深層是"怕給兒女添麻煩"和"擔心身體出問題" | 忘喫藥會擔心病情加重，但又不想總麻煩兒女提醒 |
| 爲什麼選你？ | 調研發現現有健康App界面太複雜 → 媽媽用不來 | 專爲媽媽設計：大字、大按鈕、只有"該喫藥了"一個提醒 |

### 自檢結果

- 用戶是誰：🟢 具體到"62歲的媽媽，會用微信，眼神不好"
- 痛點在哪：🟢 有情緒（擔心）、有頻率（每天要喫藥）
- 爲什麼選你：🟢 有差異（專爲媽媽設計 vs 通用健康App）

**結論：可以開始**

### 設計啓示

當用戶是家人時，"爲什麼選你"的核心往往是**適配性**：

- 現有App對這個人來說太複雜 → 你可以做一個極簡版
- 現有App沒有考慮這個羣體的特殊需求 → 你可以專門適配
- 家人不好意思付費 → 你可以免費給TA用


## 四個案例的對比

| 場景 | 用戶 | 痛點特徵 | "爲什麼選你"的核心 |
|-----|------|---------|-------------------|
| 產品 | 某類人羣中的具體個體 | 功能+情感需求 | 差異化定位 |
| 數據分析 | 看報告的決策者 | 信息不清晰、決策困難 | 呈現方式的選擇 |
| 自動化 | 自己 | 重複勞動、浪費時間 | 投入產出是否划算 |
| 給家人做 | 具體的家人 | 現有工具不適用 | 專門適配TA的需求 |
