# 2.5 靈魂三問：開發前必須回答的問題

經過本節學習，你將掌握：
- 在動手開發前用3分鐘完成自檢的方法
- 判斷自己是否"準備好開始"的標準
- 一個可以反覆使用的快速決策框架


## 什麼是靈魂三問

在創業圈和產品圈，有三個問題被反覆提及。Y Combinator 的合夥人在面試創業者時會問，紅杉資本的投資人在評估項目時會問，優秀的產品經理在立項前也會問自己。

這三個問題是：

1. **用戶是誰？**（Who）
2. **痛點在哪？**（What Problem）
3. **爲什麼選你？**（Why You）

之所以叫"靈魂三問"，是因爲這三個問題直指本質。回答得清楚，後面的事情就順；回答得模糊，做出來的東西大概率沒人用。


## 與前幾節的關係

你可能會問：前面不是講了 JTBD、用戶畫像、減法思維嗎？這裏爲什麼還要再講？

答案是：**靈魂三問是前面所有工具的"濃縮精華"。**

| 前面學過的工具 | 對應的靈魂三問 |
|--------------|---------------|
| 三維用戶畫像（2.4節） | 用戶是誰？ |
| JTBD 任務分析（2.1節） | 痛點在哪？ |
| 減法思維、差異化（2.3節） | 爲什麼選你？ |

如果說前面的工具是"詳細版手冊"，靈魂三問就是"口袋卡片"。你不需要每次都做完整的用戶畫像分析，但每次動手前，都應該能用一兩句話回答這三個問題。


## 什麼時候用

**在打開 AI 工具、寫第一行 Prompt 之前。**

具體來說：

- 當你有了一個想法，準備開始做的時候
- 當你已經做了一半，發現方向不對的時候
- 當你向別人解釋"我在做什麼"的時候

這三個問題用 3 分鐘就能問完，但能省下 3 小時的無效開發。


## 回答不出來怎麼辦

如果你發現自己無法用一句話清晰回答任何一個問題，說明你還沒準備好。

這時候有兩個選擇：

1. **回到前面的章節**，用更系統的工具（用戶畫像、JTBD）深入分析
2. **先做個最小實驗**，通過動手來驗證和澄清想法

兩種方式都可以。重要的是意識到：**模糊的想法做出模糊的產品，清晰的想法才能做出有用的東西。**
