# 2.5.4 問題三：爲什麼選你？

這是三個問題中最難回答的一個。

知道了用戶是誰、痛點在哪，還要回答：**憑什麼用戶要用你做的東西，而不是別的方案？**


## 這個問題爲什麼難

市場上可能已經有：
- 100 個待辦清單 App
- 50 個習慣打卡工具
- 20 個數據可視化軟件
- 無數個 Excel 模板

用戶爲什麼要用你做的，而不是這些已有的產品？

如果回答不出來，說明你做的東西沒有存在的理由。


## 不同場景下"爲什麼選你"的含義

這個問題在不同場景下，問法略有不同：

| 場景 | 問題變體 | 要回答的核心 |
|-----|---------|-------------|
| 做產品 | 爲什麼用戶選你而非競品？ | 你的獨特價值是什麼 |
| 做個人工具 | 爲什麼自己做而非用現成的？ | 現有工具哪裏不滿足你 |
| 寫自動化腳本 | 爲什麼值得花時間做？ | 投入產出是否划算 |
| 給家人做 | 爲什麼不直接用市面上的App？ | 現有App對這個人不適用在哪 |


## 差異化的四個來源

如果要爲用戶提供一個選擇你的理由，可以從這四個方向找：

| 差異化來源 | 說明 | 示例 |
|-----------|------|------|
| **更聚焦** | 只解決一個具體問題，不貪多 | 只管"今天要做什麼"，不做長期規劃 |
| **更簡單** | 砍掉80%功能，把20%做到極致 | 只有添加、完成、查看，沒有分類標籤優先級 |
| **更便宜** | 免費或低成本提供類似價值 | 免費、無廣告、不需要註冊 |
| **新場景** | 在別人沒覆蓋的場景下解決問題 | 專爲60歲以上老人設計，大字、大按鈕、無干擾 |

注意：**差異化不是"更多"**。

"我的App有更多功能" ≠ 差異化

"我的App專門解決XX人的XX問題" = 差異化


## 做個人工具時的"爲什麼選你"

當你給自己做工具時，"爲什麼選你"變成了"爲什麼值得自己做"。

需要回答這個問題：**現有工具哪裏不行，以至於我必須自己做一個？**

| 現有方案 | 問題在哪 | 自己做的理由 |
|---------|---------|-------------|
| 用手機備忘錄記待辦 | 沒有提醒，經常忘 | 需要一個到點提醒的簡單工具 |
| 用現成的記賬App | 分類太細，記起來太麻煩 | 只需要一個"今天花了多少"的極簡記錄 |
| 手動合併Excel | 每週花2小時，太浪費時間 | 寫個腳本一鍵完成，每週省2小時 |

如果現有工具已經夠用，就沒必要自己做。承認這一點不丟人。


## 投入產出計算

對於自動化腳本和個人工具，可以做一個簡單的投入產出計算：

```
節省的時間 = 每次節省的分鐘數 × 每月發生的次數 × 使用的月數
開發成本 = 用 AI 做出來需要的小時數

如果 節省的時間 > 開發成本 × 3，就值得做
```

**示例：**
- 每週手動合併Excel需要2小時
- 預計用1年，共52周
- 節省時間 = 2小時 × 52 = 104小時
- 用AI寫腳本預計需要3小時
- 104 > 3 × 3 = 9，值得做


## 30秒電梯演講模板

如果你能用30秒向別人解釋清楚"爲什麼選你"，說明你想明白了。

試着用這個模板：

```
對於 [目標用戶]，
他們在 [什麼場景] 遇到 [什麼問題]。

現有的解決方案是 [競品或現有方法]，
但它的問題是 [爲什麼不夠好]。

我做的 [產品/工具] 不同，因爲 [你的獨特價值]。
```

**示例：**

> 對於每週需要彙總多個部門數據的運營人員，他們在做週報時要花2小時手動複製粘貼Excel。現有的方法是一個個打開文件手動操作，但這太浪費時間而且容易出錯。我寫的這個腳本不同，因爲它可以一鍵合併所有文件，5分鐘完成原來2小時的工作。


## 先調研再動手

在回答"爲什麼選你"之前，先做一個簡單的調研：

**調研清單：**

- [ ] 搜索一下：有沒有現成的工具/App 能解決這個問題？
- [ ] 如果有，下載試用：它哪裏好？哪裏不行？
- [ ] 問問周圍的人：他們現在怎麼解決這個問題？
- [ ] 如果現有方案夠用：是否真的需要自己做？

很多時候，調研完你會發現：已經有很好的工具了，直接用就行。

這不是失敗，這是節省時間。


## 快速自檢

在進入下一節之前，確認你能完成這個填空：

```
用戶應該選擇我做的東西，而不是 [現有方案]，
因爲 [你的獨特價值/差異點]。
```

如果填不出來，可能有兩個情況：
1. 你還沒調研現有方案是什麼
2. 你做的東西確實沒有獨特價值，需要重新思考定位
