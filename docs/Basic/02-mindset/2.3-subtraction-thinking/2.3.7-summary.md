# 2.3.7 本節核心要點


## 三條核心原則

```
✓ MVP 是實驗，不是產品。它的目的是驗證假設，而不是取悅用戶。

✓ 做減法比做加法更難，但更重要。功能多不等於價值高。

✓ 「不做清單」和「要做清單」同樣重要。明確邊界才能聚焦核心。
```


## 本節思維模型回顧

| 概念 | 核心要點 | 應用場景 |
|-----|---------|---------|
| MVP 重定義 | Minimum = 最小投入，Viable = 能驗證假設 | 任何需要決定「做什麼」的時候 |
| 三個靈魂問題 | 能用嗎？驗證假設嗎？首次需要嗎？ | 篩選功能優先級 |
| P0/P1/P2 | P0 ≤ 5 個，其他都可以暫緩 | 確定 MVP 範圍 |
| 不做清單 | 功能、用戶羣、指標三個維度 | 明確邊界、抵抗誘惑 |
| 核心假設 | 明確假設 + 驗證標準 | 判斷功能是否該做 |


## 可複用的模板清單

本節提供了以下可直接使用的模板：

1. **核心假設模板**：明確你在驗證什麼
2. **驗證標準模板**：定義成功/失敗的判斷標準
3. **功能篩選表格**：用三個問題快速分類
4. **不做清單模板**：三個維度的完整格式
5. **AI 輔助 Prompt**：讓 AI 幫你砍功能和驗證判斷


## 常見錯誤 vs 正確做法

| 常見錯誤 | 正確做法 |
|---------|---------|
| 列了 14 個功能就開始做 | 先明確核心假設，再確定 P0 |
| MVP = 功能最少 | MVP = 投入最小且能驗證假設 |
| 做完再給用戶看 | 儘早讓用戶參與驗證 |
| 競品有什麼我也做什麼 | 只做驗證假設必須的功能 |
| 不做清單放腦子裏 | 寫下來，形成正式的決策記錄 |


## 檢查清單

在開始動手開發之前，確認以下事項：

- [ ] 我有明確的核心假設（一句話能說清楚）
- [ ] 我有可衡量的驗證標準（X 個用戶在 Y 時間內做了 Z）
- [ ] 我的 P0 功能不超過 5 個
- [ ] 我寫下了「不做清單」
- [ ] 我知道這個 MVP 要驗證什麼，不驗證什麼


## 承上啓下

回顧一下我們在第二章學到的內容：

- **2.1 思維升級**：從「功能視角」轉向「任務視角」，問「用戶要完成什麼任務」
- **2.2 逆向思維**：先想「什麼會導致失敗」，提前避坑
- **2.3 減法思維**：用最小投入驗證核心假設，學會說「這個不做」

這三個思維工具幫你解決了「做什麼」和「不做什麼」的問題。

但還有一個問題：**你真的瞭解你的用戶嗎？**

下一節，我們將學習「故事思維」——把用戶當作故事的主角，深入理解他們的需求和場景。

當你能像講故事一樣描述用戶時，你寫給 AI 的需求會清晰十倍。
