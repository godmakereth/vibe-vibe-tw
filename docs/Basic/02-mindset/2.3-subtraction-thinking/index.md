# 2.3 減法思維：MVP不是「最小功能」，而是「最小可驗證價值」

在上一節，我們學會了用「逆向思維」提前識別可能的失敗原因。

現在你已經知道什麼會讓項目失敗，下一個問題是：**如何在有限的時間和精力下，做出真正有價值的東西？**

答案是：學會做減法。


## 經過本節學習，你將掌握

- 理解 MVP 的真正含義，避免「功能堆砌」的陷阱
- 學會用三個問題把 20 個功能砍到 3 個核心功能
- 掌握「不做清單」這個被低估的決策工具
- 獲得可直接使用的 AI 輔助砍功能 Prompt 模板


## 本節核心洞見

> "完美不是無以復加，而是無可刪減。"
> —— Antoine de Saint-Exupéry，《小王子》作者、飛行員

這句話道出了一個反直覺的真相：

- 真正的高手不是能加更多功能的人，而是能識別什麼不該做的人
- 做減法比做加法難得多，因爲它需要你直面「這個功能可能沒人用」的殘酷現實
- 但正是這種殘酷的誠實，才能幫你做出真正有價值的東西


## 這不只是「做產品」的事

和前面幾節一樣，減法思維適用於你想用 AI 做的任何事情：

| 你想做的事情 | 減法思維幫你解決的問題 |
|-------------|---------------------|
| 做一個小工具 | 從 20 個功能中找到真正必須的 3 個 |
| 數據分析報告 | 從 50 個圖表中挑出能回答業務問題的 5 個 |
| 自動化腳本 | 從「萬能工具」收斂到「解決一個痛點」 |
| 給家人做網頁 | 從「完美設計」收斂到「他們會用」 |

無論你的目標是什麼，減法思維都能幫你聚焦在真正重要的事情上。


## 爲什麼這對 Vibe Coding 特別重要

使用 AI 編程有一個特點：**AI 可以幫你快速實現任何功能——包括你其實不需要的功能**。

這是一把雙刃劍：

- 好處是，實現功能的成本大大降低了
- 壞處是，你更容易陷入「反正做起來也不難，那就都加上吧」的陷阱

根據 Y Combinator 的數據，創業公司失敗的首要原因是「沒有市場需求」——換句話說，做了一堆沒人要的功能。

在 AI 讓「做功能」變得越來越容易的時代，「想清楚該做什麼功能」反而變得更加重要。


## 本節結構

接下來，我們將通過以下內容，幫你掌握這個思維工具：

1. **重新定義 MVP**：理解 MVP 到底是什麼，走出「最小功能」的誤區
2. **真實案例對比**：看看失敗項目和成功項目的區別
3. **MVP 的真正含義**：拆解 Minimum、Viable、Product 三個關鍵詞
4. **如何砍功能**：用三個問題和 P0/P1/P2 框架做決策
5. **「不做清單」的魔力**：學會用「不做」來定義「做」
6. **實戰練習**：爲你自己的項目做一次減法
7. **核心要點**：帶走可以立即應用的原則

準備好了嗎？讓我們從一個常見的錯誤開始。
