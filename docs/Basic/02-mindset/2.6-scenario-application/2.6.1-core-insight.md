# 2.6.1 核心洞見：一套思維，多種場景

## 爲什麼思維工具不是「產品經理專用」

前面五節講的思維工具，名字聽起來都很「產品化」：JTBD、MVP、用戶畫像、PRD……

但如果你仔細看它們的本質，會發現一個共同點：

**它們都在解決同一個問題——如何把模糊的想法變成清晰的定義。**

| 思維工具 | 表面看是 | 本質是 |
|---------|---------|--------|
| JTBD思維 | 理解用戶需求 | 搞清楚「我要完成什麼任務」 |
| 逆向思維 | 預防產品失敗 | 提前想清楚「什麼會出錯」 |
| 減法思維 | 砍掉不必要的功能 | 識別「什麼是真正重要的」 |
| 故事思維 | 設計用戶旅程 | 描述「誰在什麼情況下做什麼」 |
| 靈魂三問 | 驗證產品方向 | 確認「問題定義是否清晰」 |

**「定義問題」這個能力，不管你做什麼都需要。**

你做數據分析，需要定義「這個分析要回答什麼問題」。
你寫自動化腳本，需要定義「這個腳本要解決什麼重複勞動」。
你給自己做工具，需要定義「我真正的痛點是什麼」。
你給家人做網頁，需要定義「他們會在什麼場景下使用」。


## 一個貫穿示例：同一個人，四種場景

讓我們用一個虛構人物「小王」來說明，同一套思維工具如何應用於不同場景。

**小王的背景**：28歲，在一家電商公司做運營，會用 Excel，不會寫代碼。最近開始嘗試用 AI 編程工具。


### 場景 A：小王要做一份數據分析

**情況**：老闆要求小王分析「爲什麼上個月退貨率上升了」。

**沒有思維工具時**，小王可能會這樣告訴 AI：

```
幫我分析一下這份退貨數據
```

AI 會生成一堆圖表，但小王不知道該看哪個，也不知道怎麼向老闆彙報。

**用了思維工具後**，小王會先問自己：

| 問題 | 小王的回答 |
|-----|-----------|
| 誰要看這個分析？（靈魂三問） | 老闆，他需要在週會上彙報給CEO |
| 他們要做什麼決策？（JTBD思維） | 決定是否調整退貨政策 |
| 什麼會讓這個分析沒用？（逆向思維） | 只說「退貨率上升了」但不說原因 |
| 最核心的問題是什麼？（減法思維） | 退貨率上升的主要原因是什麼？ |

然後小王會這樣告訴 AI：

```
我是電商運營，老闆要在週會上向CEO彙報「上月退貨率上升原因」。

核心問題：退貨率上升的主要原因是什麼？

需要的分析：
1. 按商品類別分析，哪類退貨最多
2. 按退貨原因分析，最常見的3個原因
3. 對比上月和上上月，變化最大的指標

輸出要求：3張圖表，每張配一句結論，最後給出2-3個可能的原因假設。

數據在這裏：[上傳Excel]
```


### 場景 B：小王要寫一個自動化腳本

**情況**：小王每天早上要從5個平臺下載銷售數據，手動彙總到一張表裏，花費約40分鐘。

**沒有思維工具時**，小王可能會這樣告訴 AI：

```
幫我寫一個Python腳本，自動彙總銷售數據
```

AI 會問「數據從哪裏來」「格式是什麼」，小王答不上來，項目就卡住了。

**用了思維工具後**，小王會先評估：

| 問題 | 小王的回答 |
|-----|-----------|
| 這個任務多久做一次？（頻率） | 每天 |
| 每次花多少時間？（耗時） | 40分鐘 |
| 步驟是否固定？（規則性） | 是，每天都一樣 |
| 什麼情況下會出錯？（逆向思維） | 某個平臺改版了、文件格式變了 |
| 先自動化哪一步？（減法思維） | 先做「合併5個Excel」，下載暫時手動 |

然後小王會這樣告訴 AI：

```
我的重複任務：每天早上從5個平臺下載銷售數據Excel，手動合併到一張彙總表。

目前我的工作流程：
1. 手動下載5個Excel（暫時不自動化）
2. 打開每個Excel，複製「日期」「商品名」「銷量」「金額」這四列
3. 粘貼到彙總表
4. 按日期排序

希望自動化的部分：
第2-4步，把5個Excel合併成一個彙總表。

文件說明：
- 5個Excel都放在 D:\銷售數據\每日\ 文件夾
- 文件名格式：平臺名_日期.xlsx（如 淘寶_20241201.xlsx）
- 每個文件的數據在Sheet1，第一行是表頭

請幫我寫一個Python腳本，實現自動合併。
```


### 場景 C：小王要給自己做一個小工具

**情況**：小王想做一個「每日覆盤」小工具，記錄每天做了什麼、學到什麼。

**沒有思維工具時**，小王可能會想：

```
我要做一個覆盤工具，要有：
- 每日記錄
- 週報生成
- 月度統計
- 標籤分類
- 搜索功能
- 數據導出
- 圖表可視化
……
```

然後這個項目永遠做不完。

**用了思維工具後**，小王會先問自己：

| 問題 | 小王的回答 |
|-----|-----------|
| 我真正的痛點是什麼？（靈魂三問） | 每天忙完不知道時間花在哪了，第二天又重複同樣的錯誤 |
| 我會在什麼場景下用？（故事思維） | 晚上睡前，躺在牀上用手機，花2分鐘記一下 |
| 什麼功能是第一天就必須有的？（減法思維） | 能快速記錄「今天做了什麼」「明天要改進什麼」 |
| 什麼會讓我做完就不用？（逆向思維） | 太複雜、打開慢、需要填很多東西 |

然後小王會這樣告訴 AI：

```
我要做一個極簡每日覆盤網頁。

使用場景：
- 晚上睡前用手機打開
- 2分鐘內完成記錄
- 不需要登錄

核心功能（只需要這些）：
1. 兩個輸入框：「今天做了什麼」「明天要改進什麼」
2. 一個保存按鈕
3. 能看到最近7天的記錄
4. 數據保存在瀏覽器本地

不需要的功能：
- 不需要標籤分類
- 不需要搜索
- 不需要統計圖表
- 不需要導出
```


### 場景 D：小王要給爸媽做一個工具

**情況**：小王的媽媽60歲，經常忘記和老朋友視頻聊天的時間。

**沒有思維工具時**，小王可能會做一個功能很全的「日程管理App」——但媽媽根本不會用。

**用了思維工具後**，小王會先觀察和詢問：

| 問題 | 小王的回答 |
|-----|-----------|
| 媽媽是誰？技術水平如何？（故事思維） | 60歲，只會用微信，不會裝App |
| 她的真實痛點是什麼？（靈魂三問） | 不是「管理日程」，而是「記住週三下午3點要和李阿姨視頻」 |
| 什麼會讓她用不了？（逆向思維） | 字太小、步驟太多、需要登錄 |
| 她會在什麼場景下用？（故事思維） | 週三下午，提前看一眼今天要做什麼 |

然後小王會這樣告訴 AI：

```
我要給我媽做一個「今日提醒」網頁。

用戶情況：
- 60歲，只會用微信
- 不會安裝App，但會打開微信裏的鏈接
- 視力一般，需要大字

使用場景：
- 我把網頁鏈接發到家庭羣
- 媽媽點開就能看到「今天要做什麼」

核心功能（只需要這些）：
1. 大字顯示「今天是星期X」
2. 顯示今天的提醒事項（如「下午3點和李阿姨視頻」）
3. 我能遠程幫她添加/修改提醒

設計要求：
- 字體至少20px
- 按鈕要大，容易點到
- 不需要登錄
- 頁面打開就能看到信息，不需要任何操作
```


## 共性：都是在「定義問題」

四個場景，表面上完全不同：

- 場景A是數據分析
- 場景B是自動化腳本
- 場景C是個人工具
- 場景D是給家人做工具

但用到的思維工具是完全一樣的：

| 步驟 | 思維工具 | 在四個場景中都在問 |
|-----|---------|------------------|
| 第一步 | JTBD + 靈魂三問 | 這個東西要幫誰完成什麼任務？ |
| 第二步 | 故事思維 | 在什麼場景下使用？ |
| 第三步 | 逆向思維 | 什麼情況下會失敗/出錯/用不了？ |
| 第四步 | 減法思維 | 最核心的功能是什麼？先做哪個？ |

**思維工具是「操作系統」，場景只是「應用程序」。**

掌握了思維工具，你就能應對任何場景。
