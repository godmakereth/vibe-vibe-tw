# 2.6.2 數據分析場景：讓 AI 做出有價值的分析

## 數據分析的 JTBD 本質

很多人以爲數據分析就是「做幾張圖表」。但如果你用 JTBD 思維來看，數據分析真正要完成的任務是：

> **幫助某個人回答某個業務問題，從而做出更好的決策。**

| 表面需求 | JTBD 視角的真實任務 |
|---------|-------------------|
| 做一份銷售報表 | 幫老闆判斷「Q3業績是否達標」 |
| 分析用戶行爲數據 | 幫產品經理決定「下一版優先做什麼功能」 |
| 統計問卷結果 | 幫市場部判斷「用戶對新品的接受度」 |

**如果你不知道分析要回答什麼問題，AI 生成的圖表再漂亮也沒用。**


## 思維工具在數據分析中的應用

| 思維工具 | 在數據分析中要問的問題 |
|---------|---------------------|
| JTBD思維 | 這個分析要回答什麼業務問題？誰會用這個分析做什麼決策？ |
| 逆向思維 | 什麼會讓這個分析沒有價值？數據質量有問題？選錯了指標？圖表看不懂？ |
| 減法思維 | 最核心的 1-3 個問題是什麼？不要做 20 張圖，先回答最重要的問題 |
| 故事思維 | 看這個分析的人是誰？他們的背景是什麼？用什麼表達方式他們最容易理解？ |
| 靈魂三問 | 誰要看？他們關心什麼？爲什麼我的分析能幫到他們？ |


## 常見誤區：「圖表越多越專業」

❌ **錯誤做法**：

```
幫我分析這份銷售數據，做成圖表
```

AI 可能會生成 15 張圖表：柱狀圖、折線圖、餅圖、熱力圖……

問題是：
- 你不知道該看哪張
- 老闆問「所以結論是什麼」，你答不上來
- 做了很多工作，但沒有回答任何業務問題

✅ **正確做法**：

先想清楚要回答什麼問題，再讓 AI 幫你分析。


## 數據分析需求描述模板

在開始分析之前，先填寫這個模板：

```markdown
## 分析背景
- 我的角色：[你在公司的職位/角色]
- 分析目的：[這個分析要支持什麼決策]
- 彙報對象：[誰會看這個分析]
- 使用場景：[在什麼會議/報告中使用]

## 核心問題
這個分析要回答的核心問題是：
[用一句話描述，如「Q3業績下滑的主要原因是什麼」]

## 數據說明
- 數據來源：[Excel/數據庫/問卷/等]
- 數據範圍：[時間範圍、樣本量等]
- 關鍵字段：[列出最重要的幾個字段]

## 輸出要求
- 圖表數量：[建議 3-5 張，聚焦核心問題]
- 圖表類型偏好：[柱狀圖/折線圖/等，可選]
- 結論格式：[每張圖配一句結論 / 最後給總結 / 等]

## 數據文件
[上傳你的數據]
```


## 實戰案例：Q3 銷售分析

### 場景

小李是一家電商公司的銷售運營，老闆下週要向 CEO 彙報 Q3 業績。老闆讓小李「分析一下爲什麼 Q3 比 Q2 下滑了」。

### 小李的思考過程

| 思維工具 | 小李的思考 |
|---------|-----------|
| JTBD思維 | 老闆要向 CEO 彙報，他需要的是「原因解釋」而不是「數據羅列」 |
| 逆向思維 | 如果我只說「下滑了 15%」但不說原因，老闆在 CEO 面前會很尷尬 |
| 減法思維 | 核心問題就一個：下滑的主要原因是什麼。其他都是輔助 |
| 故事思維 | CEO 時間寶貴，5 分鐘內要講清楚，圖表要簡潔明瞭 |
| 靈魂三問 | 老闆 → 想知道原因 → 我的分析能給出可能的原因假設 |

### 小李給 AI 的 Prompt

```markdown
## 分析背景
- 我的角色：電商公司銷售運營
- 分析目的：幫老闆向CEO解釋Q3業績下滑原因
- 彙報對象：老闆（他會轉述給CEO）
- 使用場景：下週的高管週會，老闆有5分鐘彙報時間

## 核心問題
Q3銷售額比Q2下滑了，主要原因是什麼？

## 需要的分析
1. 整體對比：Q3 vs Q2 的總銷售額，確認下滑幅度
2. 拆分分析：按產品線/區域/客戶類型拆分，找到下滑最嚴重的部分
3. 原因探索：是客單價下降？還是訂單量下降？還是某個大客戶流失？

## 輸出要求
- 3-5張清晰的圖表
- 每張圖表配一句話結論
- 最後給出 2-3 個可能的原因假設（不需要100%確定，給方向就行）

## 數據說明
- 數據是Q2和Q3的銷售明細
- 包含字段：日期、產品線、區域、客戶名、訂單金額
- 共約5000條記錄

## 數據文件
[上傳Excel]
```

### AI 會給出什麼

基於這樣清晰的需求描述，AI 可能會：

1. 生成 Q3 vs Q2 的總銷售額對比圖，標註下滑 15%
2. 按產品線拆分的柱狀圖，發現「家電品類」下滑最嚴重（-30%）
3. 進一步分析家電品類，發現是「訂單量」下降而非「客單價」下降
4. 給出假設：可能是夏季促銷力度不夠 / 競品有大促活動 / 某款主力產品缺貨

**這樣的分析老闆拿到就能直接用。**


## 進階場景：問卷數據分析

問卷分析和銷售分析的思路相同，但有一些特殊注意點：

| 注意點 | 說明 |
|-------|------|
| 樣本量 | 要說明「共收集 X 份有效問卷」，讓結論有依據 |
| 交叉分析 | 不只看總體，還要看「不同人羣的差異」 |
| 開放題 | 文字回答要做關鍵詞提取或情感分析 |

### 問卷分析 Prompt 示例

```markdown
## 分析背景
- 我的角色：市場部實習生
- 分析目的：瞭解用戶對新產品概念的接受度
- 彙報對象：市場總監
- 使用場景：新品立項會議

## 核心問題
1. 用戶對新產品的整體接受度如何？（看購買意願評分）
2. 不同年齡段的用戶，態度有什麼差異？
3. 用戶最擔心的問題是什麼？（從開放題中提取）

## 數據說明
- 問卷共 20 題，包含單選、多選、評分題、開放題
- 有效樣本 326 份
- 關鍵字段：年齡、性別、購買意願（1-5分）、擔心的問題（開放題）

## 輸出要求
- 關鍵指標用大字展示（如「平均購買意願 3.8 分」）
- 交叉分析用分組柱狀圖
- 開放題用詞雲或關鍵詞列表

## 數據文件
[上傳Excel]
```


## 數據分析自檢清單

在把分析結果發出去之前，檢查這幾點：

- [ ] 我能用一句話說清楚「這個分析回答了什麼問題」嗎？
- [ ] 看這個分析的人，能在 30 秒內抓到重點嗎？
- [ ] 每張圖表都有明確的結論，而不只是展示數據嗎？
- [ ] 我有沒有做太多圖表，導致重點被淹沒？
- [ ] 我的分析能幫對方做出決策嗎？還是隻是「展示了數據」？


## 核心要點

✓ 數據分析的本質是「回答業務問題」，而不是「做漂亮的圖表」

✓ 開始分析之前，先用靈魂三問確認：誰要看？他們想知道什麼？

✓ 圖表貴精不貴多，3-5 張聚焦核心問題的圖表，比 20 張散亂的圖表有價值得多
