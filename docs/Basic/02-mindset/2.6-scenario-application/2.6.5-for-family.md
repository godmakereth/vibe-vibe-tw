# 2.6.5 家人工具場景：站在他們的角度設計

## 這個場景的特殊挑戰

給家人做工具，和給自己做完全不同。

| 給自己做 | 給家人做 |
|---------|---------|
| 你是用戶，知道自己要什麼 | 你要猜他們需要什麼 |
| 你會自己調試、自己修 | 他們遇到問題只會說「用不了」 |
| 你可以接受一定的複雜度 | 他們可能完全不能接受複雜度 |
| 你知道「這個按鈕是什麼意思」 | 他們可能看不懂界面 |

**核心挑戰：你習以爲常的東西，對他們可能完全陌生。**


## 思維工具在這個場景的特殊應用

| 思維工具 | 給家人做工具時的特別關注 |
|---------|----------------------|
| JTBD思維 | 他們真正要完成的任務是什麼？（往往比你想的簡單）|
| 逆向思維 | 什麼會讓他們用不了？（字太小、步驟太多、需要登錄）|
| 減法思維 | 只保留他們真正需要的功能（可能只有1個）|
| 故事思維 | 站在他們的角度想象使用場景：什麼時候用？在哪用？|
| 靈魂三問 | 他們是誰？（年齡、技術水平）他們的真實痛點？|


## 用戶調研：觀察 > 詢問

給家人做工具，不能只靠「問」。因爲：

- 他們可能說不清自己的需求
- 他們可能說「什麼都想要」（因爲不知道開發成本）
- 他們可能說「你做的我都喜歡」（因爲不想麻煩你）

**更好的方法是觀察。**

| 觀察點 | 你能發現什麼 |
|-------|------------|
| 他們用手機做什麼最多 | 微信？看視頻？通話？ |
| 他們遇到什麼問題會求助你 | 這就是真痛點 |
| 他們怎麼用現有的App | 步驟是否順利？哪裏會卡住？ |
| 他們的手機字體設置多大 | 反映他們需要的字號 |


## 非技術用戶的設計原則

### 原則1：字要大

| 用戶年齡 | 建議基礎字號 |
|---------|------------|
| 40-50歲 | 16-18px |
| 50-60歲 | 18-20px |
| 60歲以上 | 20px以上 |

**在 Prompt 裏明確說：「字體至少 20px」。**

### 原則2：步驟要少

| 好的設計 | 不好的設計 |
|---------|-----------|
| 打開就能看到信息 | 需要登錄才能看 |
| 點一下就完成操作 | 需要選擇多個選項 |
| 操作後立刻有反饋 | 操作後沒有變化 |

**理想情況：打開→看到→操作→完成，總共不超過3步。**

### 原則3：反饋要明確

| 好的反饋 | 不好的反饋 |
|---------|-----------|
| 「已保存」文字明確出現 | 只是某個圖標變了顏色 |
| 成功後有「✓」的動畫 | 什麼提示都沒有 |
| 出錯時說「請重試」 | 只顯示錯誤代碼 |

### 原則4：容錯要高

| 好的設計 | 不好的設計 |
|---------|-----------|
| 點錯了可以撤回 | 點錯了沒有後悔的機會 |
| 填錯了有提示「請輸入數字」 | 填錯了直接報錯 |
| 支持多種輸入格式 | 格式稍有不對就不認 |


## 家人工具需求描述模板

```markdown
## 用戶是誰
- 關係：[如「我媽」]
- 年齡：[如「62歲」]
- 技術水平：[如「只會用微信，不會裝App」]
- 視力狀況：[如「需要大字」]
- 常用設備：[如「華爲手機」]

## 他們的痛點
用一句話描述：[如「經常忘記喫藥時間」]

這個問題多久出現一次：[如「每天」]
現在怎麼解決：[如「我每天打電話提醒，但有時我也忘了」]
爲什麼現有App不行：[如「藥物提醒App太複雜，她不會設置」]

## 使用場景
什麼時候用：[如「早中晚飯後」]
在什麼設備上用：[如「手機」]
怎麼打開這個工具：[如「我把鏈接發到家庭羣，她點開」]

## 核心功能（只需要這些）
1. [如「顯示今天要喫的藥」]
2. [如「喫完了點一下標記爲已喫」]

## 設計要求
- 字體大小：[如「至少20px」]
- 按鈕大小：[如「容易點到，不容易誤觸」]
- 登錄要求：[如「不需要登錄」]
- 反饋方式：[如「操作後有明確的文字提示」]

## 不需要的功能
- [如「不需要歷史記錄」]
- [如「不需要雲同步」]
- [如「不需要多用戶」]
```


## 實戰案例：喫藥提醒

### 小明的情況

小明的媽媽62歲，每天要喫3種藥（早上、中午、晚上各一次）。媽媽經常忘記喫藥，小明想做一個提醒工具。

### 小明的用戶調研（觀察）

| 觀察 | 發現 |
|-----|------|
| 媽媽手機字體設置 | 開到最大 |
| 媽媽最常用的功能 | 微信、打電話 |
| 媽媽怎麼記事情 | 用紙條貼在冰箱上 |
| 媽媽之前用過提醒App嗎 | 用過，但不會設置，放棄了 |

### 小明給 AI 的 Prompt

```markdown
## 用戶是誰
- 我媽，62歲
- 只會用微信，不會安裝和設置App
- 視力一般，手機字體開到最大

## 痛點
每天要喫3種藥，經常忘。我不在身邊，沒法每天提醒。

## 使用場景
- 我把網頁鏈接發到家庭微信羣
- 媽媽點開鏈接，看到今天要喫什麼藥
- 喫完了點一下「已喫」

## 核心功能（只需要這些）
1. 顯示今天要喫的藥，分早/中/晚三欄
2. 每種藥旁邊有一個大按鈕「已喫」
3. 點了「已喫」後，那一行變成綠色，顯示「✓ 已喫」
4. 第二天自動重置（所有藥變回「未喫」狀態）

## 設計要求
- 字體至少 22px
- 按鈕要大，寬度至少佔屏幕一半
- 顏色對比要明顯（深色字，淺色背景）
- 不需要登錄
- 打開就能看到今天的藥，不需要任何操作
- 數據保存在瀏覽器本地

## 不需要的功能
- 不需要添加/編輯藥物功能（我幫她在代碼裏寫死）
- 不需要提醒推送（她不會設置權限）
- 不需要歷史記錄
- 不需要多用戶

## 藥物列表（直接寫在代碼裏）
- 早上：降壓藥
- 中午：維生素D
- 晚上：鈣片
```

### 這個設計好在哪裏

| 設計點 | 爲什麼這樣做 |
|-------|------------|
| 不需要登錄 | 減少一個步驟，減少出錯可能 |
| 藥物寫死在代碼裏 | 媽媽不需要學會「添加藥物」 |
| 不需要推送提醒 | 避免權限設置問題，由小明定時發消息提醒 |
| 點擊後變綠色+文字提示 | 明確的反饋，媽媽知道操作成功了 |
| 第二天自動重置 | 不需要手動操作 |


## 進階場景

### 場景1：視頻通話快捷入口

```markdown
## 用戶
爸媽，60多歲，想和孫子視頻但找不到微信視頻通話入口。

## 痛點
每次視頻都要問「怎麼弄」，其實就是微信裏點幾下，但他們記不住。

## 解決方案
做一個簡單網頁，上面就一個大按鈕「給孫子打視頻」。

## 功能
1. 一個大大的按鈕，寫着「給孫子視頻」
2. 點擊後彈出提示：
   「1. 打開微信
    2. 找到「小明」的聊天
    3. 點右下角的「+」
    4. 點「視頻通話」」
3. 配上截圖說明

（其實不能直接調起微信視頻，但可以做一個圖文教程）
```

### 場景2：生日相冊

```markdown
## 用戶
媽媽，過生日想給她一個驚喜。

## 功能
1. 打開看到大字「媽媽生日快樂」
2. 下面是家人的祝福語和照片
3. 自動播放背景音樂（可選）
4. 支持左右滑動看不同的祝福

## 設計要求
- 字要大
- 照片要清晰
- 操作要簡單（最好只需要滑動）
- 發一個鏈接就能打開，不需要裝任何東西
```


## 可訪問性檢查清單

發給家人之前，檢查這幾點：

- [ ] 我在自己手機上測試過了嗎？
- [ ] 我把字體調大後還能正常顯示嗎？
- [ ] 所有按鈕都足夠大，容易點到嗎？
- [ ] 操作後有明確的反饋嗎？
- [ ] 我媽/爸能理解界面上的每個文字嗎？
- [ ] 網絡不好的時候能正常打開嗎？
- [ ] 我寫了簡單的使用說明發給他們了嗎？


## 核心要點

✓ 給家人做工具，要用「觀察」而不只是「詢問」來了解需求

✓ 字要大、步驟要少、反饋要明確、容錯要高

✓ 不要做「完整」的工具，做「他們能用」的工具

✓ 測試時把自己想象成從沒用過智能手機的人
