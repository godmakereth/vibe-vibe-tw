# 2.6 場景應用：這些思維工具不只是用來「做產品」的

前面五節，我們學習了五種思維工具：JTBD思維、逆向思維、減法思維、故事思維、靈魂三問。

你可能會想：這些不都是產品經理、創業者用的嗎？和我有什麼關係？

這一節要告訴你：**這些思維工具的本質是「想清楚問題」的能力，而這個能力適用於你能想到的任何場景。**


## 經過本節學習，你將掌握

- 理解思維工具的通用性：同一套方法，不同場景的應用
- 掌握數據分析場景的需求描述方法和 Prompt 模板
- 掌握自動化腳本場景的任務評估和 Prompt 模板
- 掌握個人工具和家人工具場景的設計原則
- 獲得一套可直接複製使用的場景化 Prompt 模板庫


## 核心洞見

> **不管你想做什麼——小工具、數據分析、自動化腳本——核心思維方式都是一樣的：先想清楚問題，再動手解決。**

前面的思維工具不是「產品經理專用」，而是「定義問題專用」。

- 你做數據分析，也需要問「誰要看這個分析？他們要做什麼決策？」——這是 JTBD 思維
- 你寫自動化腳本，也需要想「什麼情況下會出錯？」——這是逆向思維
- 你給自己做小工具，也需要剋制「功能越多越好」的衝動——這是減法思維

**思維工具是通用的，變化的只是應用場景。**


## 四大場景速覽

本節覆蓋四類最常見的 Vibe Coding 場景：

| 場景 | 典型需求 | 核心挑戰 | 思維工具重點 |
|-----|---------|---------|------------|
| **數據分析** | 分析Excel、做圖表、寫報告 | 不知道要回答什麼問題 | JTBD（爲誰分析）、減法（聚焦核心問題） |
| **自動化腳本** | 批量處理文件、定時任務 | 不知道從哪開始自動化 | 逆向（識別出錯場景）、減法（先做最痛的一步） |
| **個人工具** | 記賬、番茄鍾、筆記 | 容易過度設計 | 減法（只做核心功能）、靈魂三問（自己的真實痛點） |
| **家人工具** | 喫藥提醒、相冊、快捷入口 | 不瞭解真實使用場景 | 故事思維（站在他們角度）、逆向（什麼會讓他們用不了） |
