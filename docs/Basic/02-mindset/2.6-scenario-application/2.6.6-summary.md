# 2.6.6 本節總結：場景化思維速查表

## 核心洞見回顧

本節的核心觀點是：

> **前面學的思維工具不是「產品經理專用」，而是「定義問題專用」。不管你想做什麼，核心思維方式都是一樣的。**


## 場景速查表

| 場景 | 核心問題 | 首要思維工具 | 常見陷阱 |
|-----|---------|------------|---------|
| **數據分析** | 這個分析要回答什麼業務問題？ | JTBD思維 | 做了50張圖但不知道結論是什麼 |
| **自動化腳本** | 這個任務值得花時間自動化嗎？ | 逆向思維+減法思維 | 一次性想自動化整個流程 |
| **個人工具** | 我的真實痛點是什麼？ | 減法思維 | 過度設計，做完就不用 |
| **家人工具** | 他們會在什麼場景下怎麼用？ | 故事思維 | 按自己的習慣設計，家人用不了 |


## 思維工具 × 場景 應用矩陣

| 思維工具 | 數據分析 | 自動化腳本 | 個人工具 | 家人工具 |
|---------|---------|-----------|---------|---------|
| **JTBD思維** | 誰要看？要做什麼決策？ | 這個任務幫我完成什麼？ | 我要完成什麼任務？ | 他們要完成什麼任務？ |
| **逆向思維** | 什麼會讓分析沒價值？ | 什麼情況下腳本會出錯？ | 什麼會讓我做完就不用？ | 什麼會讓他們用不了？ |
| **減法思維** | 最核心的3個問題是什麼？ | 先自動化最痛的那一步 | 第一版只做1個功能 | 只保留他們需要的功能 |
| **故事思維** | 誰在什麼場景下看這個分析？ | 什麼時候運行？結果存哪？ | 我什麼時候用？用什麼設備？ | 他們什麼時候用？怎麼打開？ |
| **靈魂三問** | 誰？關心什麼？爲什麼有價值？ | 誰用？什麼最痛？爲什麼值得做？ | 我是誰？痛點是什麼？ | 他們是誰？真實痛點是什麼？ |


## 通用 Prompt 框架

不管你做什麼場景，這個框架都適用：

```markdown
## 背景
- 我是：[你的角色/身份]
- 我要做：[一句話描述目標]
- 給誰用/誰會看：[用戶/受衆]
- 使用場景：[什麼時候、在哪、用什麼設備]

## 核心問題/任務
[用一句話描述要解決的核心問題或完成的核心任務]

## 詳細說明
[根據場景填寫具體信息：數據說明/當前流程/功能需求等]

## 第一版只需要
[列出1-3個核心功能/核心輸出]

## 不需要
[明確說不做什麼，避免AI過度發揮]

## 可能出錯的情況
[如果適用，列出異常情況及期望的處理方式]
```


## 開始新項目前的5個問題

在開始任何項目之前，花5分鐘問自己這5個問題：

| # | 問題 | 如果答不上來 |
|---|------|------------|
| 1 | 這個東西要幫誰完成什麼任務？ | 任務不明確，先想清楚再動手 |
| 2 | 我/用戶會在什麼場景下使用？ | 場景不清，設計出來可能沒人用 |
| 3 | 什麼情況下會失敗/出錯/用不了？ | 沒想過這個問題，容易踩坑 |
| 4 | 第一版最核心的1個功能是什麼？ | 想做的太多，可能做不完 |
| 5 | 我能用1句話向別人解釋這是什麼嗎？ | 如果說不清，可能自己也沒想清楚 |

**5個問題都能回答，就可以開始了。**


## 場景選擇決策樹

```
我有一個想法，想用AI幫我做
    ↓
這是什麼類型的任務？

→ 我有一堆數據，想看出點東西 → 數據分析場景（2.6.2）
→ 我有一個重複做的事情，想讓它自動 → 自動化場景（2.6.3）
→ 我想做一個給自己用的小工具 → 個人工具場景（2.6.4）
→ 我想做一個給家人/朋友用的東西 → 家人工具場景（2.6.5）
→ 我想做一個給很多人用的產品 → 回到2.1-2.5，走完整的產品思維流程
```


## 本節核心要點

✓ **思維工具是通用的**——不管你做數據分析、自動化腳本還是小工具，核心思維方式都是「先想清楚問題」

✓ **場景不同，側重不同**——數據分析側重JTBD（回答什麼問題），自動化側重逆向思維（什麼會出錯），個人工具側重減法思維（避免過度設計），家人工具側重故事思維（站在他們角度）

✓ **Prompt質量取決於問題定義**——AI是一個超級執行者，但「做什麼」只有你能決定。問題定義得越清楚，AI的輸出越有用

✓ **先做最小版本**——不管什麼場景，第一版都只做最核心的1-2個功能，驗證了再加
