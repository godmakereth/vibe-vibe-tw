# 2.8.3 從心法到技法


## 你已經學會了什麼

完成第二章的學習後，你具備了以下能力：

| 能力 | 具體表現 | 對應章節 |
|-----|---------|---------|
| **發現問題** | 能從日常生活中識別值得解決的問題 | 2.7 |
| **定義任務** | 能用"任務視角"描述用戶需求，而不是列功能清單 | 2.1 |
| **預判風險** | 能提前識別可能導致失敗的因素 | 2.2 |
| **聚焦核心** | 能把想法收斂到可驗證的最小版本 | 2.3 |
| **理解用戶** | 能構建有血有肉的用戶畫像 | 2.4 |
| **自我檢驗** | 能用三個問題判斷是否準備好開始 | 2.5 |
| **場景適配** | 能根據不同場景調整思考重點 | 2.6 |

這些能力的共同點是：**幫你想清楚"做什麼"和"爲什麼做"**。


## 心法與技法的關係

本章叫"心法篇"，下一章叫"技法篇"。它們解決的是不同的問題：

| 維度 | 心法（第2章） | 技法（第3章） |
|-----|-------------|-------------|
| **核心問題** | 做什麼？爲什麼做？ | 怎麼讓AI理解我要做什麼？ |
| **輸入** | 模糊的想法 | 清晰的需求 |
| **輸出** | 清晰的需求 | AI能執行的指令 |
| **關鍵能力** | 產品思維、用戶洞察 | Prompt工程、需求表達 |

**心法是技法的前提。**

如果你自己都沒想清楚要做什麼，寫出來的Prompt也會是模糊的。AI會給你一個"看起來像那麼回事"的結果，但不是你真正想要的。

反過來，如果你想得很清楚，但不知道怎麼表達給AI，AI也無法幫你。

兩者缺一不可。


## 從"想清楚"到"說清楚"

### 3.2 用戶旅程地圖

## 一個預告：本章思考 → 下章Prompt

看看本章的輸出如何變成下一章的輸入：

**本章你完成的思考：**

```
用戶：25歲職場新人，怕遺漏任務被批評
任務：快速記錄今天要做的事，不遺漏重要任務
場景：早上到公司，打開電腦
MVP：添加任務、完成任務、查看今日任務
不做：分類、標籤、提醒、統計
```

**下一章你將寫出的Prompt：**

```markdown
## 背景
我是一個25歲的職場新人，經常因爲忘記任務被領導批評。
我需要一個極簡的每日待辦工具。

## 用戶場景
每天早上到公司，打開電腦，快速記下今天要做的事。
下班前檢查是否都完成了。

## 核心功能（P0）
1. 添加任務：輸入框 + 回車即可添加
2. 完成任務：點擊任務左側的圓圈打勾
3. 查看任務：打開頁面就能看到今日所有任務

## 這一版不需要
- 任務分類或標籤
- 截止日期或提醒
- 歷史記錄或統計
- 用戶登錄

## 設計要求
- 界面極簡，打開就是任務列表
- 添加任務必須在3秒內完成
- 手機也能正常使用
```

**看到區別了嗎？**

同樣是"做一個待辦清單"，經過本章的思考，你的Prompt從一句模糊的想法，變成了一份AI能精確執行的需求說明。


## 寫在最後

如果你認真完成了這一章的學習和練習，恭喜你——你已經比大多數Vibe Coding初學者領先了一大步。

大多數人的做法是：有了想法 → 直接告訴AI → 發現不對 → 不斷修改Prompt → 越改越亂 → 放棄。

你的做法將是：有了想法 → 想清楚 → 寫出清晰的Prompt → AI一次給出接近預期的結果 → 微調 → 完成。

**區別不在於你用了什麼AI工具，而在於你在開口之前想了多少。**


> **記住**
> 
> AI是一個超級執行者，但"做什麼"只有你能決定。
> 
> 想清楚再動手，永遠是最快的路。
