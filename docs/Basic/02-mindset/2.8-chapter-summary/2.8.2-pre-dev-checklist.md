# 2.8.2 開發前檢查清單

在你打開AI工具、寫第一行Prompt之前，用這份清單做最後的檢查。


## 60秒快速版：紅綠燈判斷法

沒有時間做完整檢查？用這三個問題快速判斷：

| # | 問題 | 你的回答 |
|---|------|---------|
| 1 | 這個東西是給誰用的？（能說出具體的人或自己） | |
| 2 | 它解決什麼問題？（能描述一個具體的煩惱場景） | |
| 3 | 第一版只做什麼？（能用一句話說清楚） | |

**判斷標準：**

- 🟢 **三個問題都能一句話回答** → 可以開始
- 🟡 **有的問題說不清楚** → 需要再想想，回顧對應章節
- 🔴 **有的問題完全答不上來** → 先停下來，從2.1節重新開始


## 5分鐘完整版：分維度檢查

如果你有5分鐘，用這份更完整的清單：

### 問題定義（30秒）

```markdown
□ 我知道這個問題來自哪裏（自己的煩惱/他人的抱怨/工作流斷點）
□ 這個問題重複出現，不是一次性的
□ 我能用一句話描述這個問題
```

### 用戶理解（60秒）

```markdown
□ 我知道用戶是誰（具體到能叫出名字或描述特徵）
□ 我知道用戶現在怎麼解決這個問題（替代方案）
□ 我理解用戶的情緒（焦慮？煩躁？無奈？）
□ 我知道用戶會在什麼場景下使用
```

### 風險預判（60秒）

```markdown
□ 我想過什麼情況下會失敗
□ 我有一個"避坑清單"或預防措施
□ 我知道這個項目的邊界（什麼不做）
```

### MVP範圍（60秒）

```markdown
□ 我的核心假設是什麼（一句話）
□ 我知道怎麼驗證這個假設（什麼標準算成功）
□ 第一版的核心功能不超過3個
□ 我有一個明確的"不做清單"
```

### 最終確認（30秒）

```markdown
□ 我能用靈魂三問的格式總結：
  - 用戶是誰：_______________
  - 痛點在哪：_______________
  - 爲什麼選我/爲什麼值得做：_______________
```


## 問題診斷表

如果你發現自己卡住了，對照這張表找到問題：

| 你的狀態 | 可能的原因 | 建議行動 |
|---------|-----------|---------|
| 不知道做什麼項目 | 沒有系統地觀察生活中的問題 | 回到2.7節，用"煩惱日記"記錄一週 |
| 想做的功能太多，不知道砍哪個 | 沒有明確核心假設 | 回到2.3節，問自己"第一版要驗證什麼" |
| 說不清楚用戶是誰 | 用戶畫像太模糊 | 回到2.4節，用三維畫像法重新描述 |
| 擔心做出來沒人用 | 沒有做風險預判 | 回到2.2節，做一次Pre-mortem分析 |
| 解釋不清楚這個東西有什麼用 | 任務定義不清晰 | 回到2.1節，用JTBD模板重新描述 |
| 不知道自己的項目和別人有什麼區別 | 沒想清楚差異化 | 回到2.5節，回答"爲什麼選你" |
| 場景特殊，不知道怎麼調整 | 沒有場景化思考 | 回到2.6節，找到對應場景的側重點 |


## 可複製的檢查清單模板

在開始每個新項目時，複製這份模板填寫：

```markdown
# 項目啓動檢查清單

## 基本信息
- 項目名稱：
- 開始日期：
- 預計完成第一版時間：

## 問題定義
- 問題來源：□ 自己的煩惱 □ 他人的抱怨 □ 工作流斷點 □ 其他
- 問題描述（一句話）：

## 靈魂三問
1. 用戶是誰：
2. 痛點在哪：
3. 爲什麼做這個/爲什麼選我：

## MVP定義
- 核心假設：
- 驗證標準（什麼情況算成功）：
- P0功能（不超過3個）：
  1. 
  2. 
  3. 

## 不做清單
- 功能不做：
- 用戶不服務：
- 場景不覆蓋：

## 風險預判
- 最可能導致失敗的原因：
- 預防措施：

## 最終確認
- [ ] 我能用一句話向別人解釋這是什麼
- [ ] 我知道第一版要驗證什麼
- [ ] 我有信心在 ___ 天內完成第一版
```


## 不同場景的簡化檢查

並非所有項目都需要完整檢查。根據你的場景，選擇合適的深度：

| 場景 | 必須檢查 | 可選檢查 |
|-----|---------|---------|
| **給自己用的小工具** | 問題定義 + MVP範圍 | 用戶理解（自己就是用戶） |
| **數據分析/報告** | 問題定義 + 用戶理解（誰看） | 風險預判 |
| **自動化腳本** | 問題定義 + 風險預判（容錯性） | 用戶理解 |
| **給家人/朋友做** | 用戶理解 + MVP範圍 | 風險預判 |
| **想給很多人用的產品** | 全部檢查 | 無 |


## 本節要點

✓ **60秒版是底線**：無論多急，三個問題必須能回答。

✓ **診斷表幫你定位問題**：卡住了不要硬衝，找到問題根源再繼續。

✓ **不同場景深度不同**：給自己用的工具不需要完整的用戶畫像，但核心假設必須清楚。
