# 2.7.2 問題就在你身邊，只是你沒注意

## 一個思維實驗

回想一下過去一週，你有沒有：

- 做過重複的手工操作？
- 在多個地方複製粘貼同樣的信息？
- 等待某個人給你發送某個文件？
- 用 Excel 做過讓你覺得"好煩"的事？
- 想找某個信息但找不到？
- 覺得某個 App 不好用但又不得不用？

**每一個"好煩"的瞬間，都是一個潛在的問題。**

問題是：大多數人在"煩"完之後就忘了。他們繼續忍受，繼續抱怨，但從不停下來想一想——"這個問題，能不能解決？"


## 五種發現問題的渠道

問題不會主動跳出來告訴你"我在這裏"。你需要知道去哪裏找。

以下是五種最有效的問題發現渠道：

### 渠道一：自己的重複勞動

**觀察點**：什麼事情你每天/每週都在做，而且做的時候感覺"又來了"？

| 信號 | 示例 |
|-----|------|
| "每次都要……" | 每次開會前都要手動整理會議紀要模板 |
| "又要……" | 又要從5個Excel裏彙總數據做週報 |
| "總是……" | 總是忘記給家人發提醒 |

**真實案例：Timesheetr 的誕生**

一位獨立開發者厭倦了傳統的時間表系統：每次記錄工作時間都要打開復雜的軟件，填寫繁瑣的表單，然後導出數據提交給 ERP 系統。他問自己："有沒有可能做一個極簡的工具，打開就能記，記完自動同步？"

於是他用幾周時間做了一個輕量級的時間追蹤工具 Timesheetr。核心功能只有三個：快速記錄任務、自動計算時長、一鍵同步到 ERP。這個工具解決了他自己的痛點，也幫助了成千上萬有同樣困擾的自由職業者和諮詢師。

**問自己**：過去一週，你重複做了什麼事情超過 3 次？


### 渠道二：身邊人的抱怨

**觀察點**：家人、朋友、同事在抱怨什麼？

| 信號 | 示例 |
|-----|------|
| "我媽說……" | 我媽說手機上的圖標太多了，找不到想用的 |
| "同事吐槽……" | 同事吐槽每次找歷史文件都要翻半天 |
| "朋友問我……" | 朋友問我有沒有什麼工具可以記錄讀書筆記 |

**關鍵技巧：聽"過去的行爲"，而非"未來的意願"**

這是《The Mom Test》一書的核心洞見：當你問別人"你覺得這個工具怎麼樣"或"你會用嗎"，得到的答案往往是禮貌性的肯定。但如果你問"你上次遇到這個問題是什麼時候？你當時怎麼解決的？"，你會得到真實的信息。

| 不要問 | 要問 |
|-------|------|
| "你覺得這個想法好不好？" | "你上次遇到這個問題是什麼時候？" |
| "你會用這樣的工具嗎？" | "你現在怎麼解決這個問題？花了多少時間？" |
| "這個功能有用嗎？" | "你之前試過什麼方法？爲什麼不管用？" |


### 渠道三：現有工具的不滿

**觀察點**：什麼 App 或網站你在用，但用得很不爽？

| 信號 | 示例 |
|-----|------|
| "如果它能……就好了" | 如果這個日曆能自動同步多個賬戶就好了 |
| "爲什麼它不能……" | 爲什麼這個筆記軟件不能一鍵導出 PDF |
| "它什麼都好，就是……" | 它什麼都好，就是每次打開太慢了 |

**真實案例：Tag Timelines 的誕生**

一位父親在使用 Apple AirTag 追蹤孩子的書包和家裏的狗時，發現了一個問題：AirTag 只能顯示當前位置，不能查看歷史軌跡。他想知道"孩子放學後去了哪裏"，但蘋果的 Find My 應用做不到。

他沒有抱怨完就算了，而是問自己："我能不能自己做一個？"

於是他開發了 Tag Timelines，一個可以記錄和展示 AirTag 位置歷史的應用。這個工具不僅解決了他自己的問題，還吸引了大量有同樣需求的用戶——家長想追蹤孩子的行蹤，寵物主人想看狗狗在院子裏跑了多遠，商家想監控貨物的運輸路線。

**問自己**：你最近用什麼工具時，心裏嘀咕過"要是能……就好了"？


### 渠道四：工作流程的斷點

**觀察點**：在完成一件事的過程中，哪裏"卡"了？

| 信號 | 示例 |
|-----|------|
| "我得先……然後……再……" | 我得先從系統A導出，然後手動整理，再導入系統B |
| "每次到這一步就卡住" | 每次到審批環節就要等三天 |
| "如果這兩個能連起來……" | 如果微信羣消息能自動同步到待辦清單就好了 |

這種"斷點"往往是最有價值的問題。因爲它不是某個工具不好用，而是**兩個工具之間沒有打通**。

**常見的工作流斷點**：
- 從會議錄音到會議紀要（需要手動轉錄和整理）
- 從 Excel 數據到可視化圖表（需要手動複製粘貼到另一個工具）
- 從想法到待辦事項（需要打開另一個 App 手動輸入）
- 從收據照片到報銷單（需要手動填寫信息）


### 渠道五：信息獲取的困難

**觀察點**：什麼信息你經常需要，但每次獲取都很麻煩？

| 信號 | 示例 |
|-----|------|
| "我得去哪裏查……" | 我得去三個網站分別查比價 |
| "這個信息存在哪來着" | 這個客戶的聯繫方式存在哪來着 |
| "每次都要問別人" | 每次都要問同事這個流程怎麼走 |

**問自己**：什麼信息你每週至少要找一次，每次都要花超過 2 分鐘？


## 問題觀察清單

用這份清單掃描你的日常生活，找出潛藏的問題：

```markdown
## 問題觀察清單

### 重複勞動
- [ ] 我每天重複做的事情：________________
- [ ] 我每週重複做的事情：________________
- [ ] 做這些事時，最煩的環節是：________________

### 身邊人的抱怨
- [ ] 家人最近抱怨過什麼？________________
- [ ] 同事/朋友吐槽過什麼工具？________________
- [ ] 有人向我求助過什麼問題？________________

### 現有工具的不滿
- [ ] 哪個常用 App 讓我覺得"要是能……就好了"？________________
- [ ] 哪個網站/軟件我用得很不順手？________________

### 工作流斷點
- [ ] 完成某件事時，哪兩個步驟之間需要"手動搬運"數據？________________
- [ ] 哪些操作我總覺得"應該能自動化"？________________

### 信息獲取困難
- [ ] 什麼信息我經常要找，但總是記不住在哪？________________
- [ ] 什麼數據我需要從多個地方拼湊？________________
```


## 一個重要提醒

發現問題只是第一步。不是所有問題都值得解決，也不是所有問題都適合用 Vibe Coding 解決。

下一節，我們會學習如何用"煩惱日記"系統性地記錄問題，然後在 2.7.4 節學習如何篩選出真正值得投入的問題。

現在，先完成上面的觀察清單。給自己 10 分鐘，靜下心來掃描一下你的日常生活。

你可能會驚訝地發現：問題一直都在，只是你之前沒注意。
