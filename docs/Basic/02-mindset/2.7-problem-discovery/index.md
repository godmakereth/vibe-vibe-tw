# 2.7 問題發現：如何找到值得解決的問題

## 本節核心洞見

> "Live in the future, then build what's missing."
> （活在未來，然後構建缺失的東西。）
> —— Paul Graham，Y Combinator 聯合創始人

很多人在學完前面的章節後，會遇到一個尷尬的問題：

**"道理我都懂，但我不知道做什麼。"**

JTBD 思維告訴你要理解用戶任務，但你連用戶是誰都不知道。靈魂三問要求你回答"痛點在哪"，但你連問題都沒想好。這就像學會了開車，卻不知道要去哪裏。

本節要解決的，正是這個"從 0 到 0.1"的問題——**如何從日常生活中發現值得解決的問題**。


## 經過本節學習，你將掌握

- 五種從日常生活中發現問題的渠道
- "煩惱日記"方法：系統性記錄和篩選問題的工具
- 問題篩選的評分標準：判斷什麼問題適合用 Vibe Coding 解決
- 一套可立即使用的問題發現練習


## 你可能正處於這三種狀態

### 狀態一："我有很多想法，但不知道哪個值得做"

你的腦子裏有一堆零散的念頭：做個記賬工具、做個習慣打卡、做個讀書筆記……但每個都感覺"好像可以做"，又"好像沒必要做"。

**本節能幫你**：用評分表篩選出最值得投入的那一個。

### 狀態二："我想做點什麼，但完全沒想法"

你被 Vibe Coding 的可能性吸引，想親手做出點什麼，但盯着空白的屏幕，腦子也是空白的。

**本節能幫你**：用"煩惱日記"和"五種發現渠道"，從日常生活中挖掘出你沒注意到的問題。

### 狀態三："我有痛點，但不知道怎麼轉化成項目"

你知道自己每週做彙報很煩、每天找文件很痛苦，但不知道這些"煩"能變成什麼。

**本節能幫你**：通過真實案例，展示從"煩惱"到"項目"的完整轉化路徑。

## 一個關鍵認知

很多人以爲，好的項目 idea 是"靈光一閃"想出來的。

但真相是：**好的問題不是想出來的，是觀察出來的。**

那些成功的獨立開發者、產品經理、創業者，他們最強的能力不是"創意"，而是"觀察"——觀察自己的生活，觀察身邊人的困擾，觀察現有工具的不足。

本節要教你的，就是這種"觀察"的能力。
