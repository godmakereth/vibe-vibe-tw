# 2.7.7 本節核心要點

## 三條核心原則

✓ **問題不是想出來的，是觀察出來的。** 每一個"好煩"的瞬間，都是發現問題的線索。通過五種渠道（重複勞動、身邊人抱怨、現有工具不滿、工作流斷點、信息獲取困難），系統性地掃描你的日常生活。

✓ **用"煩惱日記"捕捉問題，用五維評分篩選問題。** 連續 7 天記錄煩惱，然後用重複性、規則性、可驗證性、敏感性、容錯性五個維度打分，選出最適合用 Vibe Coding 解決的問題。

✓ **知道什麼不做，和知道什麼要做一樣重要。** 涉及資金、隱私、監管、高併發、長期維護的問題，不適合用 Vibe Coding 解決。識別邊界，避免踩坑。


## 問題發現檢查清單

在動手做項目之前，確認你已經完成以下步驟：

```markdown
## 問題發現自檢清單

### 問題來源
- [ ] 我已經識別出這個問題來自哪個渠道（重複勞動/他人抱怨/工具不滿/流程斷點/信息困難）
- [ ] 我記錄了這個問題發生的頻率和痛苦程度

### 問題篩選
- [ ] 我已經用五維評分法評估了這個問題
- [ ] 總分達到 15 分以上
- [ ] 這個問題不在"紅線清單"內（不涉及資金/隱私/監管）

### 問題定義
- [ ] 我能用一句話描述"誰有這個問題"
- [ ] 我能用一句話描述"痛點具體是什麼"
- [ ] 我能用一句話描述"爲什麼現有方案解決不了"

### MVP 描述
- [ ] 我已經寫出了 MVP 的一句話描述
- [ ] 這個描述足夠具體，AI 能理解我要做什麼
```

## 本節學到的關鍵概念

| 概念 | 一句話解釋 |
|-----|-----------|
| **五種問題發現渠道** | 重複勞動、身邊人抱怨、現有工具不滿、工作流斷點、信息獲取困難 |
| **煩惱日記法** | 連續 7 天記錄煩惱瞬間，用頻率×痛苦度計算優先級 |

| **The Mom Test** | 問過去的行爲而非未來的意願，獲得真實的問題驗證 |


## 常見問題

**Q：我做完練習，發現我的問題評分很低，怎麼辦？**

A：這是好事！說明你避免了一個不適合的問題。回到煩惱日記，換一個問題評估。也可以考慮把問題"降級"到更安全的範圍（比如只做個人使用的版本）。

**Q：我的問題只有我自己有，這樣值得做嗎？**

A：完全值得。給自己做一個好用的工具，本身就是 Vibe Coding 的價值所在。而且，解決自己問題的過程中，你會學到技能、建立信心，爲將來做更大的項目打下基礎。

**Q：我做完訪談，發現別人不太有這個問題，怎麼辦？**

A：兩種選擇：(1) 把它當作個人工具來做，不追求普遍性；(2) 重新理解問題，也許你問的方式需要調整，也許問題的本質和你想的不一樣。
