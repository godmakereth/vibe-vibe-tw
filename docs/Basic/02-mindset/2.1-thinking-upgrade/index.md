# 2.1 思維升級：從"我想做一個App"到"我要解決一個問題"

## 經過本節學習，你將掌握

- 理解爲什麼"想清楚問題"比"會用AI"更重要
- 學會用"任務視角"替代"功能視角"來思考需求
- 掌握一個硅谷產品經理都在用的思維框架
- 獲得一個可以立即應用的"問題發現"模板


## 小李的故事：一個典型的失敗開局

小李是一名互聯網公司的運營，每天要處理大量的工作事項。

某天晚上，他刷到一篇文章：《用AI三分鐘做一個App》。他興奮地打開了一個AI編程工具，輸入了這樣一段話：

> "幫我做一個待辦清單App，要有任務分類、優先級標籤、截止日期提醒、重複任務、子任務拆解、標籤系統、日曆視圖、統計報表、多設備同步、協作共享、暗黑模式……"

三個小時後，小李盯着屏幕上一堆報錯信息，徹底放棄了。

**問題出在哪裏？**

不是AI不夠聰明。問題在於，小李從一開始就走錯了方向。

他把時間花在"列功能清單"上，卻從未問過自己一個最基本的問題：

> **"我到底要解決什麼問題？"**


## 這不只是"做產品"的事

在繼續之前，我想先澄清一件事：

**本節內容適用於你想用AI做的任何事情**，不只是"做產品"或"創業"。

| 你想做的事情 | 本節對你的幫助 |
|-------------|---------------|
| 解決個人痛點的小工具 | 幫你想清楚"我到底要解決什麼問題" |
| 數據分析和可視化 | 幫你明確"我要回答什麼業務問題" |
| 自動化腳本 | 幫你識別"哪些重複勞動真正值得自動化" |
| 給家人做個小網頁 | 幫你理解"他們真正需要什麼" |

無論你想做什麼，核心邏輯都是一樣的：

**先想清楚要解決的問題，再告訴AI怎麼做。**


## 本節核心洞見

> "用戶不是在購買你的產品，他們是在'僱傭'你的產品來完成一項工作。"
> —— Clayton Christensen，哈佛商學院教授，《創新者的窘境》作者

這句話揭示了一個反直覺的真相：

- 人們買電鑽，不是因爲想要電鑽，而是因爲想要牆上的洞
- 人們下載待辦清單App，不是因爲想要App，而是因爲不想遺漏重要的事
- 人們寫自動化腳本，不是因爲喜歡寫代碼，而是因爲討厭重複勞動

理解這一點，是Vibe Coding成功與失敗的分水嶺。
