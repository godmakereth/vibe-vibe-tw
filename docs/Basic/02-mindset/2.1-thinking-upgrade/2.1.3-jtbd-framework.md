# 2.1.3 JTBD框架應用：四個場景實戰

學會了JTBD的理論，現在來看看它如何應用到你實際想做的事情上。

我們會通過四個不同場景的示例，展示如何用任務視角重新定義問題。


## 任務描述模板

在開始之前，先記住這個模板。後面所有的分析都會用到它：

```
當 [某類用戶] 在 [某個情境] 下，
他們想要 [完成某個任務]，
以便於 [獲得某種結果/感受]。

目前他們的替代方案是 [現有解決方式]，
但這個方案的問題是 [痛點]。
```


## 場景一：產品原型——待辦清單

### 原始想法
"我想做一個待辦清單App"

### 用JTBD重新分析

**用戶是誰？**
小李，互聯網公司運營，每天要處理20-30個工作事項

**情境是什麼？**
每天早上到公司，需要理清今天要做什麼

**要完成什麼任務？**
快速記錄和查看今天要做的事，確保不遺漏

**期望的結果是什麼？**
安心投入工作，不用擔心漏掉重要的事

**目前的替代方案？**
用便利貼、微信置頂消息、或腦子記

**替代方案的問題？**
便利貼容易丟，微信消息會被刷掉，腦子記不住

### JTBD視角的任務描述

```
當小李每天早上到公司時，
他想要快速記錄和查看今天要做的事，
以便於不遺漏重要任務，可以安心投入工作。

目前他用便利貼和微信置頂來管理，
但便利貼容易丟，微信消息會被別的聊天刷掉。
```

### 這個分析改變了什麼？

有了這個分析，你和AI的對話會完全不同：

**之前（功能視角）：**
> "幫我做一個待辦清單，要有分類、標籤、日曆視圖……"

**之後（任務視角）：**
> "幫我做一個簡單的每日待辦工具。用戶是職場人士，早上需要快速記錄今天要做的事，核心需求是'不遺漏'和'快速記錄'。不需要複雜功能，能添加任務、標記完成就夠了。"


## 場景二：數據分析——Q3銷售報告

### 原始想法
"我想做一個銷售數據可視化看板"

### 用JTBD重新分析

**用戶是誰？**
銷售部門負責人，需要向管理層彙報季度業績

**情境是什麼？**
下週要開季度會議，需要準備彙報材料

**要完成什麼任務？**
清楚展示Q3銷售表現，回答"我們做得怎麼樣？"

**期望的結果是什麼？**
讓管理層快速理解業績情況，爲Q4策略提供依據

**目前的替代方案？**
手動在Excel裏做圖表，再複製到PPT

**替代方案的問題？**
每次都要重新做，格式不統一，耗時2-3小時

### JTBD視角的任務描述

```
當銷售負責人準備季度彙報時，
他想要清楚展示Q3的銷售表現，
以便於讓管理層快速理解業績情況，做出Q4決策。

目前他用Excel做圖表再複製到PPT，
但每次都要重新做，格式不統一，耗時長。
```

### 這個分析改變了什麼？

**之前（功能視角）：**
> "幫我做一個銷售看板，展示趨勢、留存、轉化……"

**之後（任務視角）：**
> "幫我做一個Q3銷售彙報頁面。受衆是管理層，他們關心的核心問題是'Q3做得怎麼樣'和'Q4應該怎麼調整'。數據我會提供Excel，需要展示總銷售額、月度趨勢、和各區域對比。"


## 場景三：自動化腳本——Excel彙總

### 原始想法
"我想寫一個Python腳本處理Excel"

### 用JTBD重新分析

**用戶是誰？**
行政助理，負責彙總各部門週報

**情境是什麼？**
每週五下午，需要把5個部門的週報彙總成一份總表

**要完成什麼任務？**
把5個Excel合併成1個，計算彙總數據

**期望的結果是什麼？**
下班前完成彙總，不用加班

**目前的替代方案？**
手動複製粘貼每個部門的數據

**替代方案的問題？**
容易出錯，格式不統一，要花2小時

### JTBD視角的任務描述

```
當行政助理每週五下午需要彙總週報時，
她想要快速把5個部門的Excel合併成一份總表，
以便於準時下班，不用加班。

目前她手動複製粘貼每個部門的數據，
但容易出錯、格式不統一，每次要花2小時。
```

### 這個分析改變了什麼？

**之前（技術視角）：**
> "幫我寫Python腳本，用Pandas讀取Excel，合併數據……"

**之後（任務視角）：**
> "幫我自動化一個週報彙總流程。每週我會收到5個Excel文件，格式相同。需要把它們合併成一份總表，並計算合計行。輸出一個新的Excel文件。"


## 場景四：給家人做工具——喫藥提醒

### 原始想法
"我想給我爸做一個喫藥提醒App"

### 用JTBD重新分析

**用戶是誰？**
60歲的父親，每天需要喫3種藥

**情境是什麼？**
每天早晚飯後需要喫藥，經常忘記

**要完成什麼任務？**
按時喫藥，不漏服

**期望的結果是什麼？**
身體健康，子女放心

**目前的替代方案？**
讓家人提醒，或者在藥盒上貼紙條

**替代方案的問題？**
家人不總在身邊，紙條容易忽略

### JTBD視角的任務描述

```
當60歲的父親每天需要喫藥時，
他想要有東西提醒他按時服藥，
以便於保持健康，讓子女放心。

目前他靠家人提醒或藥盒上的紙條，
但家人不總在身邊，紙條容易被忽略。
```

### 這個分析改變了什麼？

**之前（功能視角）：**
> "幫我做一個喫藥提醒App，要有藥品管理、用量記錄、庫存追蹤……"

**之後（任務視角）：**
> "幫我給60歲的父親做一個喫藥提醒網頁。他不太會用手機App，所以要極簡設計：大字體、醒目顏色、每天早晚兩個時間點提醒。只需要響鈴提醒和'已服藥'按鈕，不需要其他功能。"


## 傳統思維 vs JTBD思維對比

| 維度 | 傳統思維 | JTBD思維 |
|-----|---------|---------|
| 核心問題 | "我要做什麼功能？" | "用戶要完成什麼任務？" |
| 起點 | 功能清單 | 用戶場景 |
| 關注點 | 產品有什麼 | 用戶需要什麼 |
| 競爭分析 | 別人有什麼功能 | 用戶現在怎麼解決 |
| 成功標準 | 功能做完了 | 任務完成得更好 |


## 識別"真正的任務"三步法

當你有了一個想法，用這三步來識別真正的任務：

### 第一步：問"爲什麼"

> 我想做一個XXX。
> → 爲什麼？

繼續問"爲什麼"，直到你找到真正的動機。

**示例：**
- 我想做一個習慣打卡App → 爲什麼？
- 因爲我想養成跑步習慣 → 爲什麼？
- 因爲我想減肥 → 爲什麼？
- 因爲我想更健康、更自信

真正的任務是"變得更健康更自信"，不是"做一個App"。

### 第二步：問"現在怎麼做"

> 你現在怎麼解決這個問題？

瞭解現有方案，能幫你理解真正的痛點。

### 第三步：問"哪裏不爽"

> 現在的方案有什麼問題？

痛點往往藏在這裏。你的解決方案只需要比現有方案好一點點，就能被"僱傭"。
