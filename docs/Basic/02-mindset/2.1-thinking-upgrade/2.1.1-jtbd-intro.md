# 2.1.1 JTBD框架：理解"僱傭"隱喻

## 什麼是JTBD

Jobs to be Done（待完成的任務），簡稱JTBD，是由哈佛商學院教授Clayton Christensen提出的產品思維框架。

**核心思想只有一句話：**

> 人們不是在購買產品本身，而是在"僱傭"產品來幫助他們完成某項任務。

"僱傭"這個詞用得很妙。想象一下：

- 你僱傭一個清潔工，是爲了讓家裏變乾淨
- 你僱傭一個會計，是爲了讓賬目清晰
- 你僱傭一個待辦清單App，是爲了不遺漏重要的事

如果清潔工打掃得不乾淨，你會解僱他。如果App沒能幫你記住事情，你也會卸載它。

**產品被"僱傭"是因爲它能完成任務，被"解僱"是因爲它完成得不好。**


## 奶昔的故事：JTBD的經典案例

這是JTBD框架最著名的案例，來自一家快餐連鎖店的真實經歷。

### 問題

這家連鎖店想要提高奶昔的銷量。他們最初的做法是：

- 調查顧客："你想要奶昔更甜還是更稠？"
- 增加口味選擇：草莓、巧克力、香草……
- 優化配料組合

結果呢？銷量幾乎沒有變化。

### 轉折

後來，研究團隊換了一個問題：

> "顧客在什麼情況下會'僱傭'奶昔？他們要完成什麼任務？"

他們派人去店裏觀察，發現了一個有趣的現象：

**大多數奶昔是在早上7點到9點之間賣出的，買奶昔的人通常是獨自開車上班的通勤者。**

### 洞察

通過訪談，團隊發現這些顧客"僱傭"奶昔的真正任務是：

| 顧客的真實任務 | 爲什麼選擇奶昔 |
|--------------|---------------|
| 在漫長的通勤路上有事可做 | 吸管喝奶昔可以單手操作，不影響開車 |
| 抵禦上午的飢餓感 | 奶昔比咖啡更頂餓，能撐到午飯 |
| 比麪包不那麼無聊 | 有味道，喫起來不枯燥 |
| 保持衣服整潔 | 不會像三明治那樣掉渣 |

### 解決方案

基於這個洞察，團隊的建議是：**讓奶昔更稠**。

更稠的奶昔需要更長時間才能吸完，這意味着：
- 通勤路上的"陪伴"時間更長
- 顧客會覺得更"值"

結果：奶昔銷量提升了超過40%。

### 這個故事告訴我們什麼

傳統思維問的是"你想要什麼口味"，這是**功能視角**。

JTBD思維問的是"你要完成什麼任務"，這是**任務視角**。

同樣是賣奶昔，視角不同，答案完全不同。


## 任務的三個層次

JTBD框架把用戶的任務分爲三個層次：

| 層次 | 定義 | 示例（待辦清單場景） |
|-----|------|-------------------|
| **功能任務** | 用戶要完成的具體事情 | 記錄今天要做的事、設置截止日期 |
| **情感任務** | 用戶想要的感受 | 感到安心、有掌控感、不焦慮 |
| **社會任務** | 用戶想要的社會形象 | 顯得有條理、專業、靠譜 |

**初學者常犯的錯誤是隻看到功能任務，忽略情感任務和社會任務。**

舉個例子：

一個職場新人下載待辦清單App，表面上是爲了"記錄任務"（功能任務），但深層原因可能是：
- 害怕遺漏重要事項導致被領導批評（情感任務）
- 想讓同事覺得自己是個靠譜的人（社會任務）

如果你只關注功能，可能會不斷加功能：分類、標籤、統計……

但如果你理解情感任務，可能會發現：一個簡單的"今日待辦"清單，加上完成時的小慶祝動畫，反而更能解決問題。


## 爲什麼JTBD對Vibe Coding特別重要

你可能會問：這個框架是給產品經理用的，和我用AI寫代碼有什麼關係？

關係太大了。

**AI是一個超級執行者，但它不會幫你思考"該做什麼"。**

當你告訴AI"幫我做一個待辦清單"，AI會忠實地執行你的指令。但它不會問你：

- 你爲什麼需要待辦清單？
- 你現在用什麼方式管理任務？
- 那個方式有什麼問題？

這些問題，只有你自己能回答。

**JTBD框架的價值，就是幫你在開口告訴AI之前，先想清楚這些問題。**


## 一個思維轉換練習

下次當你有了一個想法，試着完成這個轉換：

| 原始想法 | 問自己 | JTBD視角的重述 |
|---------|--------|---------------|
| 我想做一個待辦清單 | 我爲什麼需要它？ | 當我每天開始工作時，我想快速記錄今天要做的事，這樣我就不會遺漏重要任務，可以安心投入工作 |
| 我想做一個記賬App | 我爲什麼需要它？ | 當我月底發現錢不知道花哪了，我想看清楚錢都去了哪裏，這樣我可以調整消費習慣，存下更多錢 |
| 我想寫一個自動化腳本 | 我爲什麼需要它？ | 當我每週都要手動彙總5個部門的Excel表格，我想讓這個過程自動完成，這樣我可以把時間花在更有價值的事情上 |

注意這個轉換的結構：

```
當 [某個情境發生時]，
我想要 [完成某個任務]，
這樣我就可以 [獲得某種結果/感受]。
```

這個結構會在後面的練習中反覆使用。先記住它。
