# 2.2.6 本節核心要點

## 核心概念回顧

| 概念 | 定義 | 核心問題 |
|-----|------|---------|
| 逆向思維 | 通過思考"什麼會導致失敗"來規避風險 | 什麼會讓這個項目失敗？ |
| Pre-mortem | 假設項目已經失敗，追溯可能的原因 | 如果失敗了，是因爲什麼？ |
| 避坑清單 | 把失敗原因轉化爲預防措施 | 如何避免這些失敗？ |


## 核心要點

✓ **避免失敗比追求成功更容易**：你可能不知道成功的祕訣，但你通常能識別出明顯的錯誤。先排除錯誤，再追求正確。

✓ **Pre-mortem：假設已經失敗，追溯原因**：這個技術利用人類"後見之明"的能力，把"預測"變成"回顧"，可以將風險識別能力提高 30%。

✓ **把失敗原因轉化爲預防措施**：識別風險只是第一步，關鍵是把每個高優先級的失敗原因，轉化爲具體的預防措施，並落實到需求中。


## 本節檢查清單

在開始你的項目之前，確保你能回答以下問題：

- [ ] 我做過 Pre-mortem 分析嗎？
- [ ] 我列出了至少 5 個可能的失敗原因嗎？
- [ ] 我識別出了高可能性、高嚴重性的風險嗎？
- [ ] 我爲這些風險制定了具體的預防措施嗎？
- [ ] 我把預防措施落實到需求描述中了嗎？
