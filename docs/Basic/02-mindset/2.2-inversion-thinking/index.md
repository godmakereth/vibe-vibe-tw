# 2.2 逆向思維：先想清楚什麼會讓項目失敗

在上一節，我們學會了用"任務視角"來思考需求——先問"用戶要完成什麼任務"，再考慮"需要什麼功能"。

這一節，我們要學習另一個同樣重要的思維工具：**逆向思維**。


## 經過本節學習，你將掌握

- 理解逆向思維的核心原理，以及它爲何對 Vibe Coding 特別有效
- 學會使用"預演失敗"技術，在動手之前識別潛在風險
- 獲得一份"避坑清單"模板，可以直接應用到你的項目中
- 掌握一個讓 AI 幫你做風險分析的 Prompt 模板


## 本節核心洞見

> "告訴我我會死在哪裏，這樣我就永遠不會去那裏。"
> —— Charlie Munger，伯克希爾·哈撒韋副董事長

這句話聽起來像玩笑，卻是 Munger 幾十年投資生涯的核心智慧。

與其絞盡腦汁思考"如何成功"，不如先想清楚"什麼會導致失敗"，然後避開那些坑。

這個方法有一個正式的名字：**逆向思維**。


## 這不只是"做產品"的事

和上一節一樣，逆向思維適用於你想用 AI 做的任何事情：

| 你想做的事情 | 逆向思維幫你思考的問題 |
|-------------|---------------------|
| 做一個小工具 | 什麼會讓這個工具沒人用？ |
| 數據分析報告 | 什麼會讓分析結果毫無價值？ |
| 自動化腳本 | 什麼情況會讓腳本出錯或失控？ |
| 給家人做網頁 | 什麼會讓他們根本用不起來？ |

無論你的目標是什麼，逆向思維都能幫你提前看到盲點。


## 爲什麼這對 Vibe Coding 特別重要

使用 AI 編程有一個特點：**AI 會幫你做任何你要求的事——包括錯誤的事**。

如果你的需求方向本身就有問題，AI 只會幫你更快地走向失敗。

研究表明，在項目開始前使用逆向思維技術，可以將風險識別能力提高 **30%**（根據認知心理學家 Gary Klein 的研究）。

提前預見失敗，比事後補救便宜一百倍。
