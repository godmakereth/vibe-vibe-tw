# 3.1.3 初學者最常犯的五個錯誤

## 經過本節學習，你將掌握

- 識別五種最常見的提示詞錯誤
- 理解每種錯誤「差在哪裏」
- 學會對應的修正方法
- 獲得一個可反覆使用的自檢清單


## 爲什麼要學「錯誤示範」

與其告訴你「什麼是好的提示詞」，不如先看看「什麼是差的提示詞」。

原因很簡單：避免錯誤比追求完美更容易做到，效果也更立竿見影。

下面是初學者最常犯的五個錯誤，每個錯誤都配有真實案例和修正方法。


## 錯誤 1：一次說太多（信息過載）

### 問題場景

你想讓 AI 幫你做一個待辦清單應用，於是一股腦把所有想法都倒出來：

### ❌ 差提示詞

```
幫我做一個待辦清單應用，要有添加任務、刪除任務、編輯任務、
標記完成、設置優先級、設置截止日期、分類標籤、搜索功能、
數據統計、導出功能，還要有暗黑模式，最好能同步到雲端，
支持多設備使用，界面要好看，用 React 寫，要有動畫效果。
```

### 問題分析

- AI 面對 15+ 個需求，無法判斷優先級
- 容易顧此失彼，每個功能都做得不完整
- 你收到的代碼會非常複雜，難以理解和修改
- 違背了第二章學的「減法思維」

### ✅ 修正方法：分步驟拆解

```
【第一步：先做核心功能】
幫我實現待辦清單的添加任務功能。
要求：
- 輸入框 + 添加按鈕
- 按回車也能添加
- 添加後清空輸入框
- 空內容不能添加

技術棧：React + TypeScript
先只做這一個功能，其他功能後面再加。
```

### 📌 一句話法則

**一次只讓 AI 做一件事。複雜任務拆成多輪對話。**


## 錯誤 2：一次說太少（上下文缺失）

### 問題場景

你想讓 AI 寫一段代碼，但只說了最簡單的需求：

### ❌ 差提示詞

```
寫一個登錄頁面
```

### 問題分析

- AI 不知道用什麼技術（HTML？React？Vue？）
- AI 不知道要什麼樣式（簡約？炫酷？企業風？）
- AI 不知道有什麼功能（只有登錄？還要註冊？忘記密碼？）
- AI 不知道你的技術水平（給你複雜代碼你看不懂）

結果：AI 只能給你一個「通用模板」，大概率不是你想要的。

### ✅ 修正方法：補充三層上下文

```
【項目背景】
我在做一個個人博客網站，需要一個簡單的管理員登錄頁面。
技術棧：Next.js + Tailwind CSS
風格：簡約乾淨，白色背景，居中卡片式佈局

【功能需求】
- 只需要用戶名和密碼兩個輸入框
- 一個登錄按鈕
- 不需要註冊、忘記密碼等功能
- 輸入框要有基本的校驗提示（不能爲空）

【關於我】
我是前端初學者，請給我完整可運行的代碼，關鍵部分加註釋。
```

### 📌 一句話法則

**把 AI 當作剛入職的實習生：你不說清楚，它就只能猜。**


## 錯誤 3：期待 AI 讀心（不說清楚目標）

### 問題場景

你讓 AI 幫你優化代碼，但沒說「優化」是什麼意思：

### ❌ 差提示詞

```
幫我優化一下這段代碼：
[粘貼了一段代碼]
```

### 問題分析

「優化」可以有很多種理解：
- 讓代碼跑得更快（性能優化）？
- 讓代碼更容易讀（可讀性優化）？
- 讓代碼更短（精簡代碼）？
- 讓代碼更安全（安全性優化）？
- 讓代碼更易維護（架構優化）？

AI 不知道你想要哪種「優化」，只能按自己的理解來。

### ✅ 修正方法：明確目標和驗收標準

```
幫我優化這段代碼的可讀性：

[粘貼代碼]

優化目標：
1. 讓其他開發者能快速看懂這段代碼在做什麼
2. 變量命名更清晰
3. 添加必要的註釋
4. 如果有重複邏輯，提取成函數

請保持功能不變，只改善代碼結構。
```

### 📌 一句話法則

**說清楚「什麼算完成」。給 AI 一個可以對照檢查的驗收標準。**


## 錯誤 4：忽視格式要求（讓 AI 自由發揮）

### 問題場景

你讓 AI 整理一些信息，但沒說想要什麼格式：

### ❌ 差提示詞

```
幫我整理一下 React 常用的 Hooks
```

### 問題分析

AI 可能給你：
- 一大段文字描述
- 或者一個簡單列表
- 或者過於詳細的教程
- 或者只有名字沒有解釋

每次結果都不一樣，可能都不是你想要的格式。

### ✅ 修正方法：指定輸出格式

```
幫我整理 React 最常用的 5 個 Hooks。

輸出格式：
| Hook 名稱 | 一句話解釋 | 使用場景 |
|----------|-----------|---------|

要求：
- 只列最常用的 5 個，不要太多
- 解釋要通俗易懂，初學者能看懂
- 使用場景給出具體例子
```

### 📌 一句話法則

**想要什麼格式，就先畫出格式的「樣子」給 AI 看。**


## 錯誤 5：不做約束（AI 容易「加戲」）

### 問題場景

你只要一個簡單功能，但 AI 給了你一堆額外的東西：

### ❌ 差提示詞

```
幫我寫一個顯示當前時間的組件
```

### AI 可能的「加戲」

- 加上了時區選擇功能
- 加上了 12/24 小時切換
- 加上了自定義格式功能
- 加上了暗黑模式適配
- 用了你沒聽說過的第三方庫

你只想要一個簡單的時鐘，卻收到了一個「全功能時間管理器」。

### ✅ 修正方法：用「不做清單」約束範圍

```
幫我寫一個顯示當前時間的 React 組件。

需要：
- 顯示小時:分鐘:秒（如 14:30:45）
- 每秒自動更新

不需要：
- 不要時區選擇
- 不要格式切換
- 不要用任何第三方庫
- 不要添加樣式（我自己加）

給我最簡單的實現，越簡單越好。
```

### 📌 一句話法則

**明確告訴 AI「不要什麼」，和「要什麼」同樣重要。**


## 五大錯誤速查表

| 錯誤 | 症狀 | 修正法則 |
|-----|------|---------|
| 一次說太多 | AI 輸出很長但每個功能都不完整 | 一次只讓 AI 做一件事 |
| 一次說太少 | AI 給的是「通用模板」不符合你的場景 | 補充三層上下文 |
| 期待讀心 | AI 的理解和你想的不一樣 | 說清楚驗收標準 |
| 忽視格式 | 輸出格式每次都不一樣 | 先畫出格式的「樣子」 |
| 不做約束 | AI「加戲」，做了很多你不需要的 | 用「不做清單」限制範圍 |


## 提問前自檢清單

在按下發送鍵之前，快速過一遍：

- [ ] **範圍是否聚焦**：這次只讓 AI 做一件事？
- [ ] **上下文是否充足**：AI 有足夠的背景信息嗎？
- [ ] **目標是否明確**：說清楚了「什麼算完成」？
- [ ] **格式是否指定**：告訴 AI 想要什麼樣的輸出格式？
- [ ] **邊界是否清晰**：說明了「不要什麼」？

不需要每次都完美，但避開這五個常見錯誤，你的 AI 對話質量會提升一大截。


## 本節要點

✓ **錯誤 1**：一次說太多 → 分步驟拆解，一次只做一件事

✓ **錯誤 2**：一次說太少 → 補充三層上下文，把 AI 當實習生

✓ **錯誤 3**：期待讀心 → 明確目標，給出驗收標準

✓ **錯誤 4**：忽視格式 → 先畫出格式的「樣子」

✓ **錯誤 5**：不做約束 → 用「不做清單」限制 AI 的發揮空間
