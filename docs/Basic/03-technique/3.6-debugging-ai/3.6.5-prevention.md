# 3.6.5 預防勝於治療：減少「不聽話」的概率

## 經過本節學習，你將掌握

- 發送前的 5 條快速自檢
- 「讓 AI 先複述」的預防技巧
- 「小步快跑」的迭代策略
- 及時糾偏的最佳時機

## 發送前自檢清單

在按下發送鍵之前，花 10 秒過一遍這個清單，可以避免大部分問題：

### 5 條快速自檢

- [ ] **任務是否聚焦？** 這次只讓 AI 做一件事？
- [ ] **上下文是否充足？** AI 有足夠的背景信息理解任務？
- [ ] **目標是否明確？** 說清楚了「什麼算完成」？
- [ ] **約束是否清晰？** 說明了「不要什麼」？
- [ ] **格式是否指定？** 告訴 AI 期望的輸出格式？

這和 3.1.3 中學的「五大錯誤」相對應。養成習慣後，這個檢查只需要幾秒鐘。

### 提問質量自評表

在開始重要任務前，用這個表格評估你的提示詞質量：

| 維度 | 差（1分） | 中（2分） | 好（3分） |
|-----|---------|---------|---------|
| **任務範圍** | 包含多個不相關的功能 | 有主次之分但都要做 | 只聚焦一個功能 |
| **上下文** | 只說了要做什麼 | 說了技術棧或部分背景 | 技術棧+項目背景+約束都有 |
| **驗收標準** | 沒說什麼算完成 | 模糊描述了期望 | 給出了具體的驗收條件 |
| **約束條件** | 沒有任何限制 | 有技術棧限制 | 技術棧+不做清單+格式要求 |

**評分解讀**：
- 10-12 分：很棒，發送吧
- 7-9 分：可以發送，但準備好迭代
- 4-6 分：建議補充信息後再發送
- 4 分以下：重新組織提示詞

## 讓 AI 先複述

這是一個簡單但非常有效的預防技巧：在讓 AI 開始寫代碼之前，先讓它複述對任務的理解。

### 複述 Prompt 模板

```markdown
在開始寫代碼之前，請先回答以下問題：

1. **你理解的任務目標是什麼？**
   用一句話描述。

2. **你計劃使用什麼技術方案？**
   列出關鍵技術點和實現思路。

3. **有哪些地方你不確定，需要我澄清？**
   如果沒有請說"無"。

請先回答這三個問題，我確認理解一致後，你再開始寫代碼。
```

### 爲什麼這招有效

- **提前暴露誤解**：AI 對任務的理解如果有偏差，在複述階段就能發現
- **統一預期**：你和 AI 對「做什麼」達成一致後再動手
- **節省時間**：糾正一個誤解比修改一堆錯誤代碼快得多

### 使用時機

- ✅ 開始一個新功能時
- ✅ 任務比較複雜時
- ✅ 你不確定 AI 是否理解了你的意思時
- ❌ 簡單的小修改不需要

## 小步快跑策略

「小步快跑」是指：把大任務拆成小任務，每完成一步就驗證一下，確認沒問題再繼續。

### 爲什麼要小步快跑

| 一次做完 | 小步快跑 |
|---------|---------|
| 問題累積到最後才發現 | 每步都能及時發現問題 |
| 出錯後難以定位是哪步出的問題 | 問題範圍明確，容易定位 |
| 修復成本高（可能要大改） | 修復成本低（只改最近一步） |
| AI 上下文容易丟失 | 每步上下文都是新鮮的 |

### 拆分粒度建議

| 任務類型 | 建議拆分粒度 |
|---------|-------------|
| 單個功能 | 核心邏輯 → 錯誤處理 → 邊界情況 → 樣式 |
| 多個功能 | 每個功能單獨一輪對話 |
| 完整頁面 | 佈局骨架 → 各組件 → 交互邏輯 → 數據獲取 |
| 數據流程 | 數據模型 → API 接口 → 前端調用 → 狀態管理 |

### 小步快跑的節奏

```
第一步：讓 AI 做最小核心功能
   ↓
驗證：能跑嗎？結果對嗎？
   ↓
確認 OK → 第二步：加一個小功能
   ↓
驗證：還能跑嗎？新功能對嗎？之前的功能還正常嗎？
   ↓
重複...
```

## 及時糾偏

發現問題後立即反饋，不要等到最後。

### 爲什麼要及時

- **上下文新鮮**：AI 剛輸出完，上下文還在，修改最容易
- **問題隔離**：知道問題就出在最近這一步
- **避免連鎖反應**：一個錯誤不糾正，後面可能越錯越多

### 何時該立即糾偏

| 情況 | 行動 |
|-----|------|
| 發現理解偏差 | 立即澄清，讓 AI 重新理解 |
| 技術方案不對 | 立即指出，給出正確方向 |
| 代碼有明顯錯誤 | 立即反饋，在繼續之前先修正 |
| 功能缺失 | 列出缺失項，讓 AI 補充 |

### 糾偏 Prompt 示例

```markdown
暫停一下。

上面的代碼有個問題：[描述問題]

在繼續之前，請先修正這個問題。其他部分可以保留。
```

```markdown
等等，這不是我想要的。

我要的是 [正確的需求]，不是 [AI 做的]。

請重新理解我的需求後再繼續。
```

## 預防檢查清單

每次開始和 AI 對話前，快速過一遍：

### 開始前

- [ ] 我清楚知道這次要讓 AI 做什麼（一件事）
- [ ] 我準備好了必要的上下文信息
- [ ] 我知道什麼算「完成」

### 發送後

- [ ] AI 的複述和我的意圖一致嗎？
- [ ] AI 的技術方案合理嗎？
- [ ] AI 用的庫和 API 我認識嗎？

### 收到代碼後

- [ ] 代碼能跑嗎？
- [ ] 結果符合預期嗎？
- [ ] 邊界情況處理了嗎？

## 本節要點

✓ **發送前自檢**：任務聚焦、上下文充足、目標明確、約束清晰、格式指定

✓ **複述技巧**：讓 AI 先說它的理解，確認一致後再寫代碼

✓ **小步快跑**：拆分任務，每步驗證，及時發現問題

✓ **及時糾偏**：發現問題立即反饋，不要等到最後
