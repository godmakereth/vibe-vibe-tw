# A.1 項目創建類模板

本節提供從零開始創建項目的 Prompt 模板，覆蓋網頁應用、數據分析、自動化腳本等常見場景。


## 模板一：網頁應用從零搭建

適用於：個人工具、產品原型、小型網站

```markdown
## 項目背景

我想創建一個 [項目類型]。

**目標用戶**：[誰會使用這個應用]
**核心需求**：[用一句話描述這個應用要解決什麼問題]

## 技術要求

**技術棧偏好**：
- 前端：[React/Vue/純 HTML+CSS+JS/不限]
- 樣式：[Tailwind CSS/原生 CSS/不限]
- 是否需要後端：[是/否]
- 數據存儲：[localStorage/無需存儲/需要數據庫]

**我的技術水平**：[零基礎/有一點基礎/熟悉前端開發]

## 功能需求

**必須實現（P0）**：
- [ ] [功能1]
- [ ] [功能2]
- [ ] [功能3]

**暫不需要**：
- [功能A]（原因：[爲什麼不需要]）
- [功能B]（原因：[爲什麼不需要]）

## 界面要求

**風格**：[簡潔/現代/可愛/專業/參考某網站]
**配色**：[主色調偏好，如"藍白配色"/"暗色主題"]
**響應式**：[需要適配手機/只需桌面端]

## 輸出要求

請提供：
1. 項目的文件結構說明
2. 完整的代碼文件
3. 如何在本地運行的說明
```

### 填寫示例：個人記賬本

```markdown
## 項目背景

我想創建一個極簡記賬網頁應用。

**目標用戶**：我自己（想養成記賬習慣但嫌其他 App 太複雜）
**核心需求**：快速記錄每筆支出，月底能看到總花費

## 技術要求

**技術棧偏好**：
- 前端：純 HTML+CSS+JS（我是初學者，想從簡單的開始）
- 樣式：原生 CSS
- 是否需要後端：否
- 數據存儲：localStorage

**我的技術水平**：零基礎，剛學完 Hello World

## 功能需求

**必須實現（P0）**：
- [ ] 記錄支出：金額 + 簡單備註
- [ ] 顯示今日支出列表
- [ ] 顯示本月總支出
- [ ] 刪除錯誤記錄

**暫不需要**：
- 收入記錄（原因：我只想管支出）
- 分類統計（原因：先跑起來，以後再加）
- 圖表可視化（原因：數字足夠，不需要花哨）

## 界面要求

**風格**：極簡，類似 iOS 備忘錄的乾淨感
**配色**：白色背景，深灰文字，綠色強調色
**響應式**：需要適配手機（我主要在手機上用）

## 輸出要求

請提供：
1. 項目的文件結構說明
2. 完整的代碼文件
3. 如何在本地運行的說明
```


## 模板二：數據分析項目

適用於：Excel 數據處理、銷售分析、問卷統計、可視化報告

```markdown
## 分析背景

我需要分析一份 [數據類型] 數據。

**數據來源**：[Excel/CSV/數據庫/API]
**數據規模**：約 [行數] 行，[列數] 列
**分析目的**：[回答什麼業務問題/支持什麼決策]

## 數據描述

**主要字段**：
| 字段名 | 含義 | 數據類型 | 示例值 |
|-------|------|---------|-------|
| [字段1] | [說明] | [文本/數字/日期] | [示例] |
| [字段2] | [說明] | [文本/數字/日期] | [示例] |

**數據質量問題**（如有）：
- [問題1，如"部分日期格式不統一"]
- [問題2，如"有些行缺少金額數據"]

## 分析需求

**想要回答的問題**：
1. [問題1]
2. [問題2]
3. [問題3]

**輸出形式**：
- [ ] 數據清洗後的表格
- [ ] 統計數字（如總和、平均值、佔比）
- [ ] 可視化圖表（柱狀圖/折線圖/餅圖）
- [ ] 分析結論文字說明

## 技術要求

**使用工具**：[Python + Pandas / Excel 公式 / SQL]
**運行環境**：[本地 Python / Jupyter Notebook / Google Colab]
**我的水平**：[零基礎/會基本 Python/熟悉數據分析]

## 輸出要求

請提供：
1. 完整的代碼（帶註釋說明每一步在做什麼）
2. 代碼運行後預期看到的結果示例
3. 如何修改代碼以適應我的實際數據
```

### 填寫示例：銷售數據月度分析

```markdown
## 分析背景

我需要分析一份電商銷售數據。

**數據來源**：Excel 文件（從後臺導出）
**數據規模**：約 5000 行，8 列
**分析目的**：瞭解本月銷售情況，找出暢銷商品和問題商品

## 數據描述

**主要字段**：
| 字段名 | 含義 | 數據類型 | 示例值 |
|-------|------|---------|-------|
| 訂單號 | 唯一標識 | 文本 | ORD20241201001 |
| 商品名 | 商品名稱 | 文本 | 無線藍牙耳機 |
| 分類 | 商品類別 | 文本 | 數碼配件 |
| 金額 | 訂單金額 | 數字 | 299 |
| 下單時間 | 訂單時間 | 日期 | 2024-12-01 14:30 |

**數據質量問題**：
- 部分訂單金額爲 0（可能是退款訂單）
- 下單時間格式有兩種：有的帶時分，有的只有日期

## 分析需求

**想要回答的問題**：
1. 本月總銷售額是多少？日均銷售額？
2. 哪個商品類別賣得最好？佔比多少？
3. 哪些商品是 Top 10 暢銷品？
4. 銷售額的日趨勢是什麼樣的？

**輸出形式**：
- [x] 統計數字（總額、日均、類別佔比）
- [x] 可視化圖表（類別餅圖、日趨勢折線圖、Top10 柱狀圖）
- [x] 分析結論文字說明

## 技術要求

**使用工具**：Python + Pandas + Matplotlib
**運行環境**：本地 Jupyter Notebook
**我的水平**：會基本 Python，Pandas 用得不熟

## 輸出要求

請提供：
1. 完整的代碼（帶註釋說明每一步在做什麼）
2. 代碼運行後預期看到的結果示例
3. 如何修改代碼以適應我的實際數據
```

```markdown

**目前的痛點**：
[描述現在手動做這件事有多麻煩]

**期望效果**：
[運行腳本後，應該發生什麼]

## 輸入輸出

**輸入**：
- 來源：[文件夾/單個文件/網頁/...]
- 格式：[Excel/CSV/TXT/圖片/...]
- 位置：[具體路徑或描述]
- 示例：[舉一個具體例子]

**輸出**：
- 格式：[Excel/CSV/新文件/...]
- 位置：[保存到哪裏]
- 命名規則：[如"原文件名_processed.xlsx"]

## 處理邏輯

請按以下步驟處理：
1. [第一步：做什麼]
2. [第二步：做什麼]
3. [第三步：做什麼]

**特殊情況處理**：
- 如果遇到 [情況A]，則 [處理方式]
- 如果遇到 [情況B]，則 [處理方式]

## 技術要求

**語言**：[Python/批處理腳本/PowerShell]
**運行環境**：[Windows/Mac/Linux]
**依賴限制**：[儘量少用第三方庫/可以用常見庫]
**我的水平**：[會運行腳本即可/想學習理解代碼]

## 輸出要求

4. 常見問題及解決方法
```

### 填寫示例：批量重命名圖片

```markdown
## 自動化需求

我想自動批量重命名文件夾中的圖片。

**目前的痛點**：
從相機導出的照片命名是 IMG_0001.jpg 這種格式，很難找到特定日期的照片。
手動重命名幾百張圖片太費時間。

**期望效果**：
運行腳本後，所有圖片按拍攝日期重命名爲 2024-12-01_001.jpg 格式

## 輸入輸出

**輸入**：
- 來源：指定文件夾
- 格式：jpg、png 圖片
- 位置：用戶指定（運行時輸入）
- 示例：IMG_0001.jpg、IMG_0002.jpg...

**輸出**：
- 格式：原圖片（只改名，不改內容）
- 位置：原地重命名
- 命名規則：拍攝日期_序號.擴展名（如 2024-12-01_001.jpg）

## 處理邏輯

請按以下步驟處理：
1. 讀取文件夾中所有圖片
2. 從圖片 EXIF 信息中提取拍攝日期
3. 按日期排序，同一天的圖片按時間順序編號
4. 重命名文件

**特殊情況處理**：
- 如果圖片沒有 EXIF 信息，使用文件修改時間
- 如果目標文件名已存在，添加 _dup 後綴

## 技術要求

**語言**：Python
**運行環境**：Windows 11
**依賴限制**：可以用 Pillow 庫讀取 EXIF
**我的水平**：會運行腳本即可，代碼不需要太詳細的註釋

## 輸出要求

請提供：
1. 完整可運行的腳本代碼
2. 需要安裝的依賴
3. 使用說明（如何運行）
```

## 常見填寫誤區

| 誤區 | 問題 | 正確做法 |
|-----|------|---------|
| 不寫技術水平 | AI 給的代碼太複雜或太簡單 | 明確說明自己的水平 |
| 功能需求太模糊 | "做一個好用的工具" | 列出具體的功能點 |
| 忘記寫"不需要" | AI 自作主張加了很多功能 | 明確邊界，寫清不做什麼 |
| 數據描述不清 | AI 無法寫出正確的處理邏輯 | 給出字段說明和示例數據 |


## 本節要點

- ✅ **網頁應用模板**：技術棧 + 功能需求 + 界面要求 + 輸出格式
- ✅ **數據分析模板**：數據描述 + 分析問題 + 輸出形式
- ✅ **自動化腳本模板**：輸入輸出 + 處理邏輯 + 異常處理
- ✅ **關鍵技巧**：說明技術水平，AI 會調整代碼複雜度
