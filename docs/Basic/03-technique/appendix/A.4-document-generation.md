# A.4 文檔生成類模板

本節提供生成各類文檔的 Prompt 模板，包括代碼註釋、README、API 文檔和使用說明。


## 模板一：代碼註釋生成

適用於：爲現有代碼添加註釋

```markdown
## 註釋需求

請爲以下代碼添加中文註釋：

```[語言]
[粘貼代碼]
```

## 註釋要求

**註釋類型**：
- [ ] 文件頭註釋（說明文件用途）
- [ ] 函數註釋（說明參數、返回值、功能）
- [ ] 關鍵邏輯註釋（解釋複雜邏輯）
- [ ] TODO 註釋（標記待完善的地方）

**註釋風格**：
- 語言：中文
- 格式：[JSDoc 風格 / 普通註釋 / 你的規範]
- 詳細程度：[簡潔/適中/詳盡]

**特別說明**：
- [其他要求，如"不需要給簡單的變量加註釋"]

## 輸出要求

請輸出添加註釋後的完整代碼，保持原有代碼邏輯不變。
```

### 填寫示例

```markdown
## 註釋需求

請爲以下代碼添加中文註釋：

```typescript
interface Task {
  id: string;
  title: string;
  completed: boolean;
  createdAt: Date;
}

function useTasks() {
  const [tasks, setTasks] = useState<Task[]>([]);
  
  const addTask = useCallback((title: string) => {
    const newTask: Task = {
      id: crypto.randomUUID(),
      title,
      completed: false,
      createdAt: new Date(),
    };
    setTasks(prev => [...prev, newTask]);
  }, []);
  
  const toggleTask = useCallback((id: string) => {
    setTasks(prev => 
      prev.map(task => 
        task.id === id ? { ...task, completed: !task.completed } : task
      )
    );
  }, []);
  
  return { tasks, addTask, toggleTask };
}
```

## 註釋要求

**註釋類型**：
- [x] 文件頭註釋
- [x] 函數註釋（JSDoc 格式）
- [x] 關鍵邏輯註釋

**註釋風格**：
- 語言：中文
- 格式：JSDoc 風格
- 詳細程度：適中

**特別說明**：
- interface 的每個字段需要註釋
- useCallback 需要解釋爲什麼用它

## 輸出要求

請輸出添加註釋後的完整代碼。
```

// POST /api/tasks
export async function POST(request: Request) {
  const body = await request.json();
  const { title, priority } = body;
  
  if (!title || title.trim() === '') {
    return Response.json(
      { error: 'Title is required' },
      { status: 400 }
    );
  }
  
  const task = {
    id: crypto.randomUUID(),
    title: title.trim(),
    priority: priority || 'medium',
    completed: false,
    createdAt: new Date().toISOString(),
  };
  
  // 保存到數據庫...
  await db.tasks.create(task);
  
  return Response.json(task, { status: 201 });
}
```

## 文檔生成技巧

### 技巧一：提供足夠的上下文

AI 需要理解代碼的用途才能寫出好的文檔。簡單說明這段代碼"是做什麼的"，能讓文檔更準確。

### 技巧二：指定目標讀者

給開發者看的文檔和給普通用戶看的說明，寫法完全不同。明確讀者是誰。

### 技巧三：給出格式示例

如果你有特定的文檔格式要求，給一個示例比文字描述更有效。


## 常見填寫誤區

| 誤區 | 問題 | 正確做法 |
|-----|------|---------|
| 不說明用途 | AI 不理解代碼做什麼 | 簡要說明功能和場景 |
| 不指定格式 | 輸出格式隨機 | 明確要 Markdown/JSDoc 等 |
| 不指定語言 | 可能輸出英文 | 明確要求中文 |
| 要求過於籠統 | "寫個文檔" | 具體說明包含哪些內容 |


## 本節要點

- ✅ **代碼註釋**：指定註釋類型 + 風格 + 詳細程度
- ✅ **README**：提供項目信息 + 功能列表 + 技術細節
- ✅ **API 文檔**：提供接口代碼 + 指定文檔格式
- ✅ **使用說明**：明確目標用戶 + 指定語言風格
- ✅ **關鍵技巧**：提供上下文 + 指定目標讀者 + 給出格式示例
