# 3.5.1 爲什麼一次提問往往不夠

## 經過本節學習，你將掌握

- 理解「一次搞定」爲什麼是不現實的預期
- 認識導致首次輸出不完美的三個原因
- 建立正確的迭代心態：AI 是對話夥伴，不是讀心術

## 你可能有過這樣的經歷

你花了十分鐘寫了一個詳細的提示詞，滿懷期待地按下回車。結果 AI 給你的東西——雖然看起來很專業——卻完全不是你想要的。

你開始懷疑：是不是我的提示詞寫得不夠好？是不是我不夠聰明？是不是 AI 根本沒用？

**都不是。**

這只是說明：你和 AI 之間需要更多對話。就像你和一個新同事合作，第一次交接任務時，對方很難完全理解你的意圖。這不是誰的錯，而是溝通的本質。

## 首次輸出不完美的三個原因

### 原因一：信息不對稱

你腦子裏有完整的畫面，但 AI 只能看到你寫下來的文字。

```
你心裏想的：
"我要一個簡潔的輸入框，放在頁面頂部，按回車就能添加任務，
 輸入框要有圓角，placeholder 寫'添加新任務...'，
 添加成功後自動清空輸入框"

你實際寫的：
"幫我做一個添加任務的功能"
```

**你以爲自己說清楚了，但 AI 只收到了 10% 的信息。**

### 原因二：需求本身模糊

很多時候，你自己也不完全清楚想要什麼——直到看到第一版輸出。

這很正常。產品設計領域有句話：「用戶不知道自己要什麼，直到你展示給他看。」你和 AI 的關係也類似。第一輪輸出的價值，往往不是「直接能用」，而是「幫你想清楚」。

### 原因三：AI 的理解偏差

即使你寫得很詳細，AI 也可能理解錯。因爲自然語言是模糊的。

```
你說："做一個簡單的表單"

AI 可能理解爲：
- 只有一個輸入框的表單？
- 沒有驗證邏輯的表單？
- 樣式樸素的表單？
- 功能精簡的表單？
```

「簡單」對不同人意味着不同的事情。AI 只能猜測你的意圖。

## 一個類比：裝修房子

假設你請了一個裝修師傅，你說：「我要一個舒適的客廳。」

師傅不可能一次就裝修出你心中完美的樣子。你們需要：

1. **第一輪**：師傅給你看設計方案草圖（方向對不對？）
2. **第二輪**：你提出修改意見，師傅調整（沙發換個位置，窗簾換個顏色）
3. **第三輪**：細節打磨（燈光亮度、抱枕擺放）

AI 編程也是一樣的過程。**期望一次到位，是不合理的預期。**

## 迭代思維：每次對話都在逼近目標

換一個角度看問題：

| 傳統思維 | 迭代思維 |
|---------|---------|
| 第一次沒做對 = 失敗 | 第一次輸出 = 有價值的起點 |
| 需要多次修改 = AI 不行 | 多輪對話 = 協作的正常過程 |
| 追求一次完美 | 追求持續改進 |

**核心轉變**：把 AI 當作一個聰明的實習生，而不是全知全能的神。你需要和它「聊」，而不是「命令」。

## 差提示詞 vs 好提示詞

理解了迭代思維後，你會發現：**好的提示詞不是追求一次完美，而是爲迭代留下空間。**

### ❌ 差提示詞：試圖一次說完所有事

```
幫我用 React + TypeScript + Tailwind CSS 實現一個待辦清單的添加任務功能，
要有輸入框和按鈕，輸入框要有 placeholder，按鈕要有 hover 效果，
點擊按鈕或按回車都能添加任務，要驗證空輸入，空輸入時顯示錯誤提示，
添加成功後清空輸入框，任務數據存到 state 裏，用 useState hook，
組件命名爲 AddTaskForm，導出爲默認導出...
```

**問題**：信息量太大，AI 容易遺漏細節。而且你可能還沒想清楚所有需求。

### ✅ 好提示詞：先建立方向，再逐步細化

**第一輪提示詞：**
```
我正在開發一個待辦清單應用，使用 React + TypeScript + Tailwind CSS。

請幫我實現「添加任務」功能：
- 一個輸入框 + 一個添加按鈕
- 點擊按鈕可以添加任務到列表

先給我一個基礎實現，我們後面再逐步完善。
```

**看到第一輪輸出後，第二輪提示詞：**
```
基礎功能可以了。請加上以下改進：
1. 支持按回車鍵添加任務
2. 添加空輸入驗證，空輸入時按鈕禁用
3. 添加成功後自動清空輸入框
```

**第三輪提示詞：**
```
功能完整了。最後優化一下代碼結構：
- 把驗證邏輯抽成一個獨立的 validateTaskInput 函數
- 把樣式類名用變量統一管理
```

**優勢**：每一輪都有明確焦點，AI 不容易遺漏。而且你可以根據上一輪的輸出調整下一輪的需求。

## 實戰心態調整

| 場景 | 錯誤反應 | 正確反應 |
|-----|---------|---------|
| AI 第一次輸出不完美 | "AI 太笨了" | "這是正常的起點" |
| 需要多次修改 | "太麻煩了" | "這是協作的過程" |
| 發現自己需求變了 | "我怎麼沒想清楚" | "第一輪輸出幫我想清楚了" |

## 本節要點

✓ **一次搞定是幻覺**：首次輸出不完美是常態，不是失敗

✓ **三個原因**：信息不對稱、需求模糊、AI 理解偏差

✓ **裝修類比**：AI 編程就像裝修房子，需要「看效果 → 提意見 → 再調整」

✓ **迭代心態**：把 AI 當作對話夥伴，而不是命令執行者

✓ **分階段提問**：先建立方向，再逐步細化，比一次說完更有效
