# 3.5 迭代對話的藝術：從「一次搞定」到「逐步逼近」

> **本節目標**：掌握多輪對話技巧，學會在對話中逐步逼近理想結果

你已經學會了寫結構化的提示詞（3.2），掌握了進階技巧（3.3），甚至能寫出一份完整的 PRD（3.4）。但當你把這些都交給 AI 後，很可能發現：**第一次輸出往往不是你想要的**。

這不是你的問題，也不是 AI 的問題。這是對話的本質。

## 核心洞見

> "追求「一次完美」是幻覺，「逐步逼近」纔是正道。"

想象你在裝修房子。你不會期望油漆工刷完第一遍就完美無瑕。你會看效果、提意見、再調整。AI 對話也是如此——它是一個**協作過程**，不是一次**命令執行**。

## 經過本節學習，你將掌握

- 理解爲什麼「一次提問」往往不夠
- 掌握迭代對話的三階段模型：看方向 → 改問題 → 磨細節
- 學會用 SBI 框架給 AI 有效反饋
- 管理長對話的上下文，避免 AI「失憶」
- 知道何時該開始新對話，以及如何「搬家」

## 延續案例：小李的待辦清單

本節繼續使用小李的待辦清單項目。你會看到一個完整的迭代過程：

```
初始需求："幫我實現添加任務的功能"
    ↓
第一輪：AI 給出基礎實現（方向對，但細節差）
    ↓
反饋："輸入框太小，而且沒有驗證空輸入"
    ↓
第二輪：AI 修正，加入驗證（功能對，但結構不滿意）
    ↓
反饋："驗證邏輯放在了組件裏，我希望抽成獨立函數"
    ↓
第三輪：最終版本（結構清晰，可維護）
```

三輪對話，從「能用」到「好用」。這就是迭代的力量。

## 學習建議

1. **調整預期**：第一輪輸出不完美是正常的，不要沮喪
2. **邊學邊練**：每個小節都有可複製的模板，建議實際使用
3. **記錄經驗**：把有效的反饋句式收集起來，形成自己的「武器庫」
