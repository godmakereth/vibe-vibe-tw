# 3.7.5 從本章到下一章

## 能力盤點：你現在具備什麼

完成第二章和第三章的學習後，你已經具備了以下能力：

### 思維能力（第二章）

| 能力 | 你能做到 |
|-----|---------|
| **問題發現** | 從日常煩惱中識別值得解決的問題 |
| **任務定義** | 用 JTBD 描述用戶真正要完成的任務 |
| **風險預判** | 用逆向思維提前識別失敗原因 |
| **範圍收斂** | 用減法思維確定 MVP 和「不做清單」 |
| **用戶理解** | 構建用戶畫像，描述用戶旅程 |
| **自我檢驗** | 用靈魂三問確認準備就緒 |

### 表達能力（第三章）

| 能力 | 你能做到 |
|-----|---------|
| **上下文構建** | 給 AI 提供完整的項目、任務、個人上下文 |
| **結構化表達** | 用 S.C.A.F.F. 或 R.G.C. 框架組織提示詞 |
| **技巧選擇** | 根據任務類型選擇 Zero-shot/Few-shot/CoT 等技巧 |
| **PRD 編寫** | 整合思考成果，寫出 AI 能理解的需求文檔 |
| **迭代對話** | 通過精確反饋逐步逼近理想結果 |
| **問題處理** | 識別 AI 幻覺，知道何時需要人工干預 |

## 第四章預告：從 0 到 1 開發個人工具

### 你將學到

| 階段 | 內容 | 你會得到 |
|-----|------|---------|
| **準備** | 整合第二章 + 第三章成果 | 完整的項目 PRD |
| **靜態層** | 搭建頁面框架和樣式 | 可以看的界面 |
| **交互層** | 實現核心功能 | 可以用的功能 |
| **存儲層** | 讓數據持久化 | 刷新不丟失 |
| **調試** | 處理 AI 代碼報錯 | Debug 實戰經驗 |
| **收尾** | 完整項目回顧 | 你的第一個作品 |

### 貫穿案例

第四章將繼續使用小李的待辦清單項目。你會看到：

- 如何把第二章的 PRD 轉化爲第一個 Prompt
- 如何通過迭代對話實現三個 P0 功能
- 如何處理 AI 代碼中的 Bug
- 最終得到一個完整可用的待辦清單應用

### 你需要準備的

進入第四章之前，請確認：

```markdown
□ 我有一個 AI 編程工具可用（Cursor、Windsurf、Bolt.new 等任意一個）
□ 我完成了第二章的項目思考（或者準備用小李的待辦清單跟着做）
□ 我準備好了 2-3 小時的集中學習時間
□ 我有心理準備：AI 可能會犯錯，但我知道怎麼修正
```


## 課後作業

在進入第四章之前，請完成以下作業：

### 作業一：選擇你的項目

從以下選項中選擇一個，或者用你自己的想法：

| 項目類型 | 示例 | 難度 |
|---------|------|------|
| **個人工具** | 待辦清單、讀書筆記、習慣打卡 | ⭐ |
| **數據可視化** | 個人開支分析、健身數據追蹤 | ⭐⭐ |
| **自動化腳本** | 文件批量重命名、數據格式轉換 | ⭐⭐ |
| **給家人的工具** | 老人用的大字時鐘、簡單相冊 | ⭐ |

**新手建議**：選擇「待辦清單」跟着教程走完第一遍，然後再做自己的項目。

### 作業二：完成項目 PRD

用本章學到的方法，爲你選擇的項目完成 PRD。

**PRD 模板**：

```markdown
# [項目名稱] PRD

## 1. 項目背景
### 問題來源
[描述你發現的問題]

### JTBD 任務描述
當我 [場景] 時，我想 [任務]，這樣我就能 [價值]。

## 2. 目標用戶
- 身份：
- 場景：
- 核心需求：

## 3. 功能範圍
### P0 功能（第一版必須有）
1. 
2. 
3. 

### 不做清單
- 
- 
- 

## 4. 技術約束
- 技術棧：
- 數據存儲：
- 其他：

## 5. 成功標準
[如何驗證項目成功]
```

### 作業三：寫出第一個 Prompt

基於你的 PRD，用 S.C.A.F.F. 框架寫出實現第一個 P0 功能的 Prompt。

**不需要真的發給 AI**，寫完保存好，第四章會用到。


## 自我檢查清單

在進入第四章之前，確認你能回答這些問題：

### 關於第二章

```markdown
□ 我能用一句話描述項目要解決的問題
□ 我知道目標用戶是誰
□ 我明確了第一版要做什麼、不做什麼
□ 靈魂三問我都能回答
```

### 關於第三章

```markdown
□ 我知道好提示詞包含哪些要素
□ 我能選擇合適的提示技巧（Zero-shot/Few-shot/CoT）
□ 我知道 AI 輸出不滿意時如何給反饋
□ 我知道如何驗證 AI 推薦的包和 API
```

如果有的問題答不上來，建議回顧對應章節再繼續。


## 寫在最後

恭喜你完成了第二章和第三章的學習！

這兩章是 Vibe Coding 的核心基礎。很多人急於動手，跳過思考和表達的訓練，結果在開發時反覆踩坑、不斷返工。

你選擇了更紮實的路徑：**先想清楚，再說明白，最後才動手**。

這條路看似慢，實際上最快。

下一章，你將看到這些積累如何轉化爲實實在在的成果——你的第一個 AI 編程作品。


> **記住**
> 
> Vibe Coding 的核心不是「讓 AI 寫代碼」，而是「把你想要的說清楚」。
> 
> 你已經學會了怎麼說。現在，讓我們開始做。
