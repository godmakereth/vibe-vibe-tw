# 3.7.3 提示詞速查清單

本節提供一系列可直接使用的速查清單和模板，建議收藏或打印備用。


## 一頁紙速查表：提問前必查

在向 AI 提問之前，快速檢查這九項：

```
┌─────────────────────────────────────────────────────────────┐
│                    提示詞自檢清單 v1.0                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  □ 1. 我說清楚【背景】了嗎？                                  │
│       （項目類型、技術棧、當前進度）                          │
│                                                             │
│  □ 2. 我說清楚【任務】了嗎？                                  │
│       （要做什麼、輸入輸出是什麼）                            │
│                                                             │
│  □ 3. 我說清楚【約束】了嗎？                                  │
│       （必須遵守的限制、不能做什麼）                          │
│                                                             │
│  □ 4. 我說清楚【格式】了嗎？                                  │

│  □ 8. 我準備好【迭代】了嗎？                                  │
│       （不期待一次完美）                                      │
│                                                             │
│  □ 9. 我整合了【第二章的思考成果】嗎？                        │

│                                                             │

## 分場景 Prompt 模板

### 場景一：代碼生成

適用於：創建新組件、實現新功能、生成完整模塊

```markdown
【背景】
項目類型：[網頁應用/數據分析/自動化腳本/...]
技術棧：[React/Python/Node.js/...]
當前進度：[項目剛開始/已有基礎結構/...]

【任務】
請幫我實現 [功能名稱]：
- 功能描述：[用戶做什麼，系統做什麼]
- 輸入：[用戶輸入什麼]
- 輸出：[系統輸出什麼]

【約束】
必須：
- [約束1]
- [約束2]

不能：
- [約束3]
- [約束4]

【格式要求】
- [ ] 完整代碼文件
- [ ] 包含 TypeScript 類型
- [ ] 包含關鍵註釋
- [ ] 包含使用說明
```

**填寫示例**：

```markdown
【背景】
項目類型：個人待辦清單網頁應用
技術棧：React + TypeScript + Tailwind CSS
當前進度：項目已初始化，已有 AddTask 組件

【任務】
請幫我實現「刪除任務」功能：
- 功能描述：用戶點擊任務右側的刪除圖標，該任務從列表中移除
- 輸入：用戶點擊刪除圖標
- 輸出：任務從列表中消失，顯示簡短的撤銷提示

【約束】
必須：
- 刪除前顯示確認提示（防止誤刪）
- 刪除後 3 秒內可撤銷
- 樣式用 Tailwind CSS

不能：
- 不用任何 UI 組件庫
- 不用 confirm() 彈窗（體驗不好）

【格式要求】
- [x] 完整代碼文件
- [x] 包含 TypeScript 類型
- [x] 包含關鍵註釋
- [ ] 包含使用說明
```


### 場景二：代碼修改

適用於：修復 Bug、優化性能、重構代碼

```markdown
【當前代碼】
```[語言]
[粘貼當前代碼]
```

【問題描述】
現在的問題是：[具體描述問題現象]
期望的行爲是：[描述正確的行爲]

【修改要求】
- [要求1]
- [要求2]

【約束】
- 只修改必要的部分，保持其他代碼不變
- [其他約束]
```

**填寫示例**：

```markdown
【當前代碼】
```typescript
const handleSubmit = () => {
  if (inputValue) {
    addTask(inputValue);
    setInputValue('');
  }
}
```

【問題描述】
現在的問題是：輸入空格也能添加任務
期望的行爲是：只有真正有內容時才能添加，空格應該被忽略

【修改要求】
- 添加前去除首尾空格
- 去除空格後如果爲空，顯示提示

【約束】
- 只修改必要的部分，保持其他代碼不變
- 提示用 toast 而不是 alert
```


### 場景三：問題排查

適用於：代碼報錯、功能異常、性能問題

```markdown
【問題現象】
[描述你看到了什麼]

【期望行爲】
[描述你期望看到什麼]

【相關代碼】
```[語言]
[粘貼相關代碼]
```

【已嘗試的方法】
1. [嘗試1]：[結果]
2. [嘗試2]：[結果]

【環境信息】
- 瀏覽器/運行環境：[版本]
- 相關庫版本：[版本]

【請幫我】
1. 分析可能的原因
2. 給出解決方案
3. 解釋爲什麼會出現這個問題
```

**填寫示例**：

```markdown
【問題現象】
點擊「完成任務」按鈕後，任務狀態沒有變化，控制檯沒有報錯

【期望行爲】
點擊後任務左側的圓圈應該變成勾選狀態

【相關代碼】
```typescript
const toggleComplete = (id: string) => {
  setTasks(tasks.map(task => 
    task.id === id ? { ...task, complete: !task.complete } : task
  ));
}
```

【已嘗試的方法】
1. 檢查 id 是否正確傳遞：console.log 確認 id 是對的
2. 檢查 tasks 狀態：更新前後 tasks 數組沒有變化

【環境信息】
- 瀏覽器：Chrome 120
- React 版本：18.2.0

【請幫我】
1. 分析可能的原因
2. 給出解決方案
3. 解釋爲什麼會出現這個問題
```


### 場景四：技術選型

適用於：選擇庫、框架、架構方案

```markdown
【項目背景】
[簡要描述項目]

【選型需求】
我需要選擇 [選型類型]，主要考慮：
- [考慮因素1]
- [考慮因素2]
- [考慮因素3]

【候選方案】（可選）
1. [方案A]
2. [方案B]
3. [方案C]

【請幫我】
1. 分析每個方案的優缺點
2. 給出推薦方案及理由
3. 說明什麼情況下應該選擇其他方案
```

**填寫示例**：

```markdown
【項目背景】
個人待辦清單應用，目前數據存在 localStorage，想支持多設備同步

【選型需求】
我需要選擇後端數據存儲方案，主要考慮：
- 免費或低成本（個人項目）
- 上手簡單（我是後端小白）
- 支持用戶認證

【候選方案】
1. Supabase
2. Firebase
3. 自己搭建 Node.js + MongoDB

【請幫我】
1. 分析每個方案的優缺點
2. 給出推薦方案及理由
3. 說明什麼情況下應該選擇其他方案
```


## 常見錯誤自檢清單

在發送 Prompt 前，檢查是否犯了這五個常見錯誤：

| # | 錯誤 | 檢查方法 | 修正方式 |
|---|-----|---------|---------|
| 1 | **沒有背景** | 刪掉背景後 AI 還能理解嗎？ | 加上項目類型和技術棧 |
| 2 | **太模糊** | 「優化代碼」優化什麼？ | 具體說明優化哪個方面 |
| 3 | **沒有約束** | AI 可能「加戲」嗎？ | 加上「不做清單」 |
| 4 | **沒有格式** | AI 輸出格式隨機？ | 指定要代碼還是解釋 |
| 5 | **一次要太多** | 任務超過 3 個要點？ | 拆分成多輪對話 |


## 幻覺驗證快速指南

當 AI 推薦了你不熟悉的包或 API 時，用這個流程驗證：

```
AI 推薦了一個包或 API
        │
        ▼
┌─────────────────────────┐
│ 1. 驗證包是否存在         │
│    npm search <包名>     │
│    或訪問 npmjs.com      │
└─────────────────────────┘
        │
        ├─ 不存在 ──────────▶ 🔴 包幻覺，要求 AI 重新推薦
        │
        ▼ 存在
┌─────────────────────────┐
│ 2. 檢查包的可靠性         │
│    - 周下載量 > 1000?    │
│    - 最近更新 < 1年?     │
│    - GitHub stars > 100? │
└─────────────────────────┘
        │
        ├─ 不滿足 ──────────▶ 🟡 謹慎使用，考慮替代方案
        │
        ▼ 滿足
┌─────────────────────────┐
│ 3. 驗證 API 是否正確      │
│    查閱官方文檔          │
└─────────────────────────┘
        │
        ├─ API 不存在 ─────▶ 🔴 API 幻覺，查文檔找正確 API
        │
        ▼ API 存在
        ✅ 可以使用
```


## 迭代反饋模板

當 AI 的輸出不滿意時，使用這個模板給反饋：

```markdown
【問題】
[具體指出哪裏不對]

【原因】
[解釋爲什麼這樣不行]

【期望】
[明確說明你想要什麼]

【示例】（可選）
[給一個你期望的示例]
```

**填寫示例**：

```markdown
【問題】
你給的代碼中，刪除任務後沒有撤銷功能

【原因】
用戶可能誤刪，需要有後悔的機會

【期望】
刪除後顯示一個 3 秒的「撤銷」提示，點擊可以恢復任務

【示例】
類似 Gmail 刪除郵件後底部出現的「撤銷」按鈕
```
