# 3.4 編寫你的第一份 PRD：整合第二章思考

> **本節目標**：學會將第二章的產品思考成果，轉化爲 AI 能理解和執行的 PRD 文檔
>
> **核心定位**：PRD 是連接"想清楚"和"做出來"的橋樑。本節不重複講解產品思維（那是第二章的內容），而是聚焦於如何把思考結果"翻譯"給 AI。

## 經過本節學習，你將掌握

- 理解 PRD 在 Vibe Coding 中的獨特價值（與傳統 PRD 的區別）
- 學會把第二章的思考成果（靈魂三問、減法思維、用戶旅程）填入 PRD 模板
- 獲得一份可直接複製使用的精簡版 PRD 模板
- 掌握用 P0/P1/P2 優先級引導 AI 分階段開發的方法
- 通過完整案例，從"我想做一個..."走到"AI 開始執行"

## 本節與其他章節的關係

```
第二章（心法）          第三章前半部分           本節
    ↓                      ↓                   ↓
 想清楚要做什麼    →    學會結構化表達    →    整合成 AI 任務書
 靈魂三問、減法思維      S.C.A.F.F. 框架         PRD 文檔
 用戶畫像、旅程地圖      提示技巧               分階段開發
```

如果說 3.1-3.3 是教你"說話的方法"，本節就是把這些方法**應用到一份完整文檔上**。

## 本節結構

| 小節 | 核心內容 | 你會獲得 |
|-----|---------|---------|
| 3.4.1 | 什麼是 PRD，爲什麼需要 | 正確認知：PRD 是任務書，不是報告 |
