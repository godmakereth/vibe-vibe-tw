# 3.4.4 用優先級引導 AI 分階段開發

你在第二章用減法思維劃分了 P0/P1/P2 功能優先級。現在的問題是：**如何讓 AI 按這個順序分階段工作？**

## 爲什麼要分階段開發

### 問題場景

小李把完整的 PRD 一次性發給 AI：

```
請幫我實現這些功能：
- 添加任務
- 查看列表
- 勾選完成
- 刪除任務
- 分類標籤
- 優先級標記
- 定時提醒
- 數據統計
```

AI 返回了一大堆代碼。小李運行後發現：

- "添加任務"有 bug（輸入爲空也能添加）
- "分類標籤"功能做得很完整
- "定時提醒"用了他看不懂的技術

小李花了兩小時調試，發現核心功能有問題，但錦上添花的功能卻很完善。

**這就是"一次性給太多"的典型後果。**

### 分階段的三大好處

| 好處 | 說明 |
|-----|------|
| **減少遺漏** | AI 一次處理 3-5 個功能比處理 10 個更不容易出錯 |
| **及時驗證** | 每輪結束後可以測試，發現問題及時修正 |
| **靈活調整** | P0 做完後可能發現 P1 不需要了，省去無用功 |

## 三輪對話模式

分階段開發的核心是：**每輪只做一個優先級的功能，驗證通過後再進入下一輪。**

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   第一輪    │────▶│   第二輪    │────▶│   第三輪    │
│  實現 P0    │     │  添加 P1    │     │  添加 P2    │
│             │     │             │     │  (可選)     │
└─────────────┘     └─────────────┘     └─────────────┘
      │                   │                   │
      ▼                   ▼                   ▼
   驗證測試            驗證測試            驗證測試
```

## 第一輪：實現 P0 功能

### Prompt 模板

```markdown
請根據以下 PRD 實現第一版，**只做 P0 功能，不要做任何額外的事情**。

[粘貼你的 PRD]

特別強調：
- 只實現"核心功能"部分列出的功能
- 不要添加任何 PRD 中沒有提到的功能
- 代碼要簡潔，能跑起來就行
```

### 小李的實際 Prompt

```markdown
請根據以下 PRD 實現第一版，只做 P0 功能。

# 極簡待辦

## 一句話描述
一個給自己用的極簡待辦清單網頁

## 目標用戶
我自己（職場人士，每天處理5-10件事務）

## 要解決的問題
怕遺漏重要事項，需要一個打開就能用的極簡工具

## 核心功能（P0，只做這些）
- 添加任務：輸入框 + 添加按鈕
- 查看任務：列表顯示所有待辦
- 完成任務：點擊勾選，顯示刪除線

## 不做的功能
- 不要登錄註冊
- 不要刪除功能（P1，下一輪做）
- 不要分類標籤
- 不要雲同步

## 技術約束
- 純前端：HTML + CSS + JavaScript
- 數據存儲：瀏覽器 localStorage
- 界面語言：中文

特別強調：只做上面列出的 P0 功能，不要添加任何額外功能。
```

### 第一輪驗證清單

AI 返回代碼後，按這個清單驗證：

- [ ] 能添加任務嗎？
- [ ] 任務能顯示在列表嗎？
- [ ] 能勾選完成嗎？
- [ ] 刷新後數據還在嗎？
- [ ] 有沒有多做了什麼功能？

**全部通過後，才進入第二輪。**

## 第二輪：添加 P1 功能

### Prompt 模板

```markdown
P0 功能已完成並驗證通過。現在請添加 P1 功能。

## 當前已有功能
- 添加任務 ✓
- 查看列表 ✓
- 勾選完成 ✓

## 本輪要添加的 P1 功能
- [P1 功能1]
- [P1 功能2]

## 注意事項
- 不要修改已有的 P0 功能代碼（除非是 bug 修復）
- 不要添加任何 P1 以外的功能
- 保持代碼風格一致
```

### 小李的實際 Prompt

```markdown
P0 功能已完成並驗證通過。現在請添加 P1 功能。

## 當前已有功能
- 添加任務 ✓
- 查看列表 ✓
- 勾選完成 ✓

## 本輪要添加的 P1 功能
- 刪除任務：每個任務右邊有個刪除按鈕，點擊後移除
- 任務計數：顯示"待辦 X 項，已完成 Y 項"

## 注意事項
- 不要修改已有的 P0 功能
- 不要添加分類、標籤等其他功能
- 刪除按鈕樣式要和整體風格一致
```

### 第二輪驗證清單

- [ ] P0 功能還能正常工作嗎？
- [ ] 刪除功能正常嗎？
- [ ] 計數顯示正確嗎？
- [ ] 刪除後計數會更新嗎？

## 第三輪：添加 P2 功能（可選）

第三輪是可選的。如果 P0 + P1 已經滿足需求，可以到此爲止。

### Prompt 模板

```markdown
P1 功能已完成。如果時間允許，現在可以添加 P2 功能。

## 當前已有功能
- [P0 功能列表] ✓
- [P1 功能列表] ✓

## 可選的 P2 功能（按優先級選擇 1-2 個）
- [P2 功能1]
- [P2 功能2]
- [P2 功能3]

請先問我想優先做哪個，然後再實現。
```

## 階段切換的關鍵句式

以下是一些在階段切換時很有用的句式：

### 開始新階段

```
"P0 驗證通過，請繼續添加 P1 功能..."
"上一輪功能已確認無誤，現在進入下一階段..."
"第一版完成，現在迭代第二版..."
```

### 發現問題時

```
"在測試 P0 時發現一個問題：[描述問題]。請先修復這個，然後再繼續 P1。"
"添加任務時輸入爲空也能添加，這是個 bug。請加上非空校驗。"
```

### 想調整計劃時

```
"P0 做完後我發現其實不需要 P1 的刪除功能了，請直接跳到 P2 的分類標籤。"
"這個功能比想象中複雜，我們先暫停，我重新想想需求。"
```

## 完整案例：小李的三輪開發

### 第一輪對話

**小李**：
```
請根據以下 PRD 實現第一版...（P0 功能）
```

**AI**：返回 HTML + CSS + JS 代碼

**小李測試**：
- 添加任務 ✓
- 查看列表 ✓
- 勾選完成 ✓
- 刷新數據保留 ✓

### 第二輪對話

**小李**：
```
P0 驗證通過。請添加 P1 功能：
- 刪除任務
- 任務計數
```

**AI**：返回更新後的代碼

**小李測試**：
- P0 功能正常 ✓
- 刪除功能正常 ✓
- 計數顯示正確 ✓

### 第三輪對話

**小李**：
```
P1 完成。P2 功能裏我只想做"任務優先級標記"，其他不要。
```

**AI**：返回帶優先級功能的代碼

**小李測試**：全部通過，項目完成。

## 常見問題

### Q：P0 還沒做完，能不能先做個 P1 功能？

A：不建議。P0 是核心功能，如果核心有問題，後面做的都可能白做。老老實實按順序來。

### Q：AI 自作主張加了功能怎麼辦？

A：告訴它刪掉，或者重新生成。
```
"我看到你添加了排序功能，但這不在 PRD 裏。請刪掉這個功能，只保留我要求的。"
```

### Q：每輪都要把 PRD 全發一遍嗎？

A：第一輪發完整 PRD，後面只發變化的部分就行。AI 通常能記住上下文。

### Q：分階段會不會太慢？

A：表面上多對話了幾輪，但減少了大量返工時間。一次性做完往往要花更多時間調試。

## 驗證檢查點模板

每輪結束後，用這個模板做檢查：

```markdown
## 第 [N] 輪驗證檢查

### 功能驗證
- [ ] [功能1] 正常工作
- [ ] [功能2] 正常工作
- [ ] [功能3] 正常工作

### 迴歸測試（確保之前的功能沒壞）
- [ ] P0 功能仍然正常
- [ ] P1 功能仍然正常（如果已實現）

### 邊界情況
- [ ] 空輸入處理正確
- [ ] 刷新後數據保留
- [ ] 多次操作不會出錯

### 決策
- [ ] 全部通過 → 進入下一輪
- [ ] 有問題 → 先修復，再繼續
```

## 本節核心要點

- ✅ 分階段開發能減少遺漏、及時驗證、靈活調整
- ✅ 三輪模式：P0 → 驗證 → P1 → 驗證 → P2（可選）
- ✅ 每輪開始時明確"只做這些，不做其他"
- ✅ 每輪結束時用檢查清單驗證
- ✅ 發現問題先修復，確認無誤再進入下一輪
