# 3.2.4 萬能提示詞模板

## 經過本節學習，你將掌握

- 一個適用於大多數場景的通用模板
- 完整版和精簡版的使用場景
- 三種不同場景的填寫示例
- 讓模板更好用的實用技巧


## 爲什麼需要"萬能模板"

S.C.A.F.F. 適合技術開發，R.G.C. 適合快速提問，但有些任務不那麼容易歸類：

- 既不是純技術，也不是簡單提問
- 不確定該用哪個框架
- 想要一個"怎麼填都不會錯"的通用方案

這就是萬能模板存在的意義：**覆蓋面廣，怎麼填都不會太差**。


## 模板結構

萬能模板包含七個要素，按重要性排序：

| 序號 | 要素 | 必填程度 | 作用 |
|-----|------|---------|------|
| 1 | 角色 | 推薦 | 讓 AI 進入專家模式 |
| 2 | 背景 | 必填 | 提供上下文信息 |
| 3 | 任務 | 必填 | 說清楚要做什麼 |
| 4 | 要求 | 推薦 | 明確成功標準 |
| 5 | 約束 | 推薦 | 劃定邊界，防止"加戲" |
| 6 | 格式 | 視情況 | 指定輸出形式 |
| 7 | 示例 | 可選 | 用例子教 AI 你想要什麼 |


## 完整版模板

```markdown
你是一個 [角色描述]。

## 背景
[項目/任務的背景信息]

## 任務
[具體要做什麼]

## 要求
- [要求1：具體的成功標準]
- [要求2]
- [要求3]

## 約束
- [不能做什麼1]
- [不能做什麼2]

## 輸出格式
[期望的輸出形式]

## 示例（可選）
[一個理想輸出的示例，幫助 AI 理解你的期望]
```


## 精簡版模板

日常使用時，可以保留核心三要素：

```markdown
你是一個 [角色]。

背景：[一句話說明情境]

任務：[要做什麼]

要求：
- [要求1]
- [要求2]
```

**選擇標準**：

| 任務類型 | 推薦版本 |
|---------|---------|
| 重要的、會進入項目的代碼 | 完整版 |
| 快速驗證想法、臨時查詢 | 精簡版 |
| 第一次做某類任務 | 完整版（幫你理清思路） |
| 重複做過的任務 | 精簡版 |


## 場景示例

### 場景一：產品開發（待辦清單）

```markdown
你是一個擅長 React 和用戶體驗設計的前端開發者。

## 背景
我在做一個個人待辦清單應用，目標用戶是容易忘事的職場新人。
技術棧：React + TypeScript + Tailwind CSS
設計原則：極簡，添加任務要在 3 秒內完成

## 任務
實現任務的"拖拽排序"功能，讓用戶可以通過拖拽調整任務順序。

## 要求
- 拖拽時被拖動的任務半透明
- 放置位置有明顯的視覺指示線
- 拖拽完成後順序立即更新
- 移動端也能用（長按觸發拖拽）

## 約束
- 不使用第三方拖拽庫（如 react-beautiful-dnd）
- 用原生 HTML5 拖拽 API
- 代碼要有詳細註釋，我是拖拽 API 的初學者

## 輸出格式
完整的 React 組件代碼，包含：
1. 組件文件（TypeScript）
2. 關鍵邏輯的中文註釋
3. 簡單的使用說明
```

### 場景二：數據分析

```markdown
你是一個擅長數據可視化的分析師，特別關注如何讓非技術人員也能看懂數據。

## 背景
我是電商公司的運營，手裏有三個月的銷售數據（Excel 格式）。
老闆每週一早上只有 15 分鐘看報告，上次的報告太長被批評了。

## 任務
幫我設計一份 Q3 銷售分析報告的結構。

## 要求
- 老闆能在 5 分鐘內抓住重點
- 第一頁就能看到最重要的結論
- 有明確的行動建議，不只是展示數據
- 圖表要簡潔，一圖說明一件事

## 約束
- 報告不超過 5 頁
- 不使用複雜的統計術語
- 顏色使用公司品牌色（藍色系）

## 輸出格式
1. 報告的頁面結構（每頁標題和內容要點）
2. 推薦的圖表類型及原因
3. 建議的配色方案
```

### 場景三：自動化腳本

```markdown
你是一個注重代碼健壯性的 Python 工程師。

## 背景
我是公司財務，每週五要彙總 5 個部門的週報 Excel。
當前流程：手動打開 5 個文件 → 複製數據 → 粘貼到彙總表 → 調整格式
每週花 2 小時，經常出錯要反覆檢查。

## 任務
寫一個 Python 腳本，自動完成這個彙總工作。

## 要求
- 讀取指定文件夾下所有 .xlsx 文件
- 每個文件提取"彙總"工作表的 A1:E20 區域
- 合併到一個新的 Excel，每個部門一個工作表
- 運行結束後顯示處理了多少個文件

## 約束
- 只用 openpyxl 庫（公司電腦已安裝）
- 如果文件格式不對，跳過並記錄錯誤，不要中斷整個流程
- 代碼要有中文註釋，我是 Python 初學者

## 輸出格式
完整的 Python 腳本，可以直接運行

## 示例
文件夾結構：
```
/週報/
  ├── 銷售部_第1周.xlsx
  ├── 市場部_第1周.xlsx
  ├── ...
```
輸出文件：彙總_2024年第1周.xlsx
```


## 讓模板更好用的技巧

### 技巧一：先寫"任務"，再補其他

很多人會卡在"背景"不知道怎麼寫。建議：

1. 先在"任務"裏寫你最想說的
2. 然後問自己：AI 理解這個任務需要知道什麼背景？
3. 把答案填入"背景"

### 技巧二："約束"用"不要"開頭

"約束"最容易寫成"要求"的重複。記住：

- **要求**：告訴 AI "要什麼"
- **約束**：告訴 AI "不要什麼"

約束的句子儘量用"不要"、"不使用"、"不需要"開頭，這樣更容易區分。

### 技巧三：示例勝過千言萬語

當你發現很難用語言描述期望的輸出時，直接給一個示例：

```markdown
## 示例
輸入："2024-01-15"
期望輸出："2024年1月15日 星期一"
```

一個精準的示例，比三段描述更有效。


## 本節要點

✓ **萬能模板七要素**：角色、背景、任務、要求、約束、格式、示例

✓ **完整版 vs 精簡版**：重要任務用完整版，快速驗證用精簡版

✓ **填寫順序建議**：先寫任務，再補背景，最後加約束

✓ **關鍵區分**：要求說"要什麼"，約束說"不要什麼"
