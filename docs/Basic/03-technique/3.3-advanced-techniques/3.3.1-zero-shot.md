# 3.3.1 Zero-shot Prompting：直接提問的藝術

## 經過本節學習，你將掌握

- Zero-shot 的定義和工作原理
- 適合使用 Zero-shot 的場景
- 如何寫出高質量的 Zero-shot 提示詞
- Zero-shot 的侷限性及應對方法

## 什麼是 Zero-shot Prompting

Zero-shot（零樣本）的意思是：**不給任何示例，直接讓 AI 執行任務**。

這是最直接的提問方式。你描述任務，AI 根據它在訓練中學到的知識直接給出答案。

```
將以下 Python 2 代碼轉換爲 Python 3 語法：
print "Hello, World!"
```

AI 不需要你演示「怎麼轉換」，它已經「知道」Python 2 和 Python 3 的區別。這就是 Zero-shot 的核心：**利用 AI 已有的知識完成任務**。

## 爲什麼 Zero-shot 有效

現代 AI 模型在訓練時已經「見過」海量數據：代碼、文檔、教程、問答。對於常見任務，它已經具備執行能力，只需要你**清晰地描述任務**即可。

就像請一位經驗豐富的同事幫忙——你不需要教他怎麼寫代碼，只需要說清楚你要什麼。

## 適用場景

| 場景 | 示例 | 爲什麼適合 |
|-----|------|-----------|
| 代碼轉換 | Python 2 → Python 3 | AI 熟悉兩種語法 |
| 文本分類 | 判斷評論情感 | 常見 NLP 任務 |
| 格式轉換 | JSON → YAML | 標準格式，規則明確 |
| 內容生成 | 寫一段產品描述 | 創意任務，無需示例 |
| 代碼解釋 | 解釋這段代碼做了什麼 | AI 理解代碼能力強 |

**一句話判斷**：如果這個任務「AI 應該會」，就用 Zero-shot。

## 差提示詞 vs 好提示詞

### ❌ 差提示詞

```
幫我處理一下這個數據
```

**問題**：任務不明確。處理是什麼意思？清洗？轉換？分析？

### ✅ 好提示詞

```
將以下 CSV 數據轉換爲 JSON 格式，每行作爲一個對象：

name,age,city
張三,25,北京
李四,30,上海
```

**改進點**：
- 明確任務：CSV → JSON
- 明確格式：每行一個對象
- 提供輸入數據

### 輸出對比

差提示詞可能得到：「請提供具體數據和處理需求」（AI 無法執行）

好提示詞得到：
```json
[
  {"name": "張三", "age": 25, "city": "北京"},
  {"name": "李四", "age": 30, "city": "上海"}
]
```

## 寫好 Zero-shot 的四個要素

### 1. 明確的任務動詞

| 模糊 | 清晰 |
|-----|------|
| 處理這段代碼 | **重構**這段代碼，提取重複邏輯爲函數 |
| 看看這個數據 | **分析**這個數據的異常值 |
| 弄一下格式 | **轉換**爲 Markdown 表格格式 |

### 2. 具體的輸出要求

```
❌ "給我一個函數"
✅ "給我一個 JavaScript 函數，接收數組參數，返回去重後的數組"
```

### 3. 必要的約束條件

```
❌ "寫一個排序算法"
✅ "寫一個排序算法，要求：時間複雜度 O(n log n)，不使用內置排序方法"
```

### 4. 清晰的輸入數據

如果任務需要處理數據，直接在提示詞中提供。

## 進階技巧：增強 Zero-shot 效果

### 角色設定（Role Prompting）

給 AI 一個角色，可以讓它從特定視角回答問題。

```
你是一位有 10 年經驗的前端工程師。
請審查以下 React 代碼，指出潛在的性能問題：
[代碼]
```

### 輸出格式指定

明確告訴 AI 你想要的格式。

```
分析以下用戶反饋，提取關鍵問題。
輸出格式：
- 問題1：[描述]
- 問題2：[描述]
...
```

## 實戰示例：待辦清單項目

小李想讓 AI 幫他寫一個日期格式化函數。

### Zero-shot 提示詞

```
寫一個 JavaScript 函數 formatDate，功能如下：
- 輸入：Date 對象
- 輸出：格式化的字符串，格式爲 "YYYY年MM月DD日"
- 示例：輸入 new Date('2024-03-15')，輸出 "2024年03月15日"
- 要求：月份和日期不足兩位時補零
```

### AI 輸出

```javascript
function formatDate(date) {
  const year = date.getFullYear();
  const month = String(date.getMonth() + 1).padStart(2, '0');
  const day = String(date.getDate()).padStart(2, '0');
  return `${year}年${month}月${day}日`;
}
```

## Zero-shot 的侷限性

| 侷限 | 表現 | 解決方案 |
|-----|------|---------|
| 格式不穩定 | 每次輸出格式不一致 | 用 Few-shot 提供示例 |
| 複雜推理出錯 | 多步驟邏輯容易跳步 | 用 Chain of Thought |
| 不理解特定需求 | 輸出不符合項目規範 | 在提示詞中補充上下文 |

**判斷標準**：如果 Zero-shot 連續兩次輸出不滿意，考慮換用其他技巧。

## 可複製模板

### 代碼轉換模板

```markdown
將以下 [源語言/版本] 代碼轉換爲 [目標語言/版本]：

[代碼內容]

要求：
- 保持功能一致
- 遵循 [目標語言] 的最佳實踐
```

### 內容生成模板

```markdown
你是 [角色定位]。

任務：[具體任務描述]

要求：
- [要求1]
- [要求2]

輸出格式：[格式說明]
```

### 代碼審查模板

```markdown
審查以下代碼，從 [審查維度] 角度指出問題：

[代碼內容]

輸出格式：
1. 問題：[描述] | 位置：[行號] | 建議：[修改方案]
```

## 本節要點

✓ **Zero-shot 的本質**：利用 AI 已有知識，直接執行任務

✓ **適用場景**：AI「應該會」的任務——轉換、分類、生成

✓ **寫好 Zero-shot 的關鍵**：明確動詞 + 具體輸出 + 必要約束

✓ **侷限性**：格式不穩定、複雜推理容易出錯

✓ **增強技巧**：角色設定、輸出格式指定
