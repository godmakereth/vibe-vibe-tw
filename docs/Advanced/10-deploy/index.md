# 10 ｜部署與運維

代碼寫完只是萬里長征第一步，讓用戶真正用上纔是終點。

## 本章導讀

部署不是把代碼丟到服務器就完事。它是一個系統工程，涉及**基礎設施選擇**、**容器化編排**、**流量管理**和**運行監控**四大環節。本章將帶你從零開始，用現代化的工具鏈完成一次完整的生產部署。

```mermaid
flowchart LR
    A[本地開發] --> B[容器化打包]
    B --> C[服務器部署]
    C --> D[反向代理]
    D --> E[監控告警]
    E --> F[持續運維]
```

## 核心理念

| 原則 | 說明 |
|------|------|
| **基礎設施即代碼** | 所有配置都應版本化，可復現 |
| **不可變部署** | 容器鏡像一旦構建，不再修改 |
| **可觀測性優先** | 日誌、指標、追蹤缺一不可 |
| **故障預案先行** | 上線前就要想好掛了怎麼辦 |

## 本章目錄

- **10.1 上線前必須知道的事** — 雲服務、域名、備案，這些前置工作不能跳過
- **10.2 點點鼠標就上線** — 用 1Panel 可視化完成從零到上線
- **10.3 一鍵啓動所有服務** — Docker Compose 多服務編排實戰
- **10.4 網站的交通指揮官** — Nginx 反向代理與負載均衡配置
- **10.5 網站生病了怎麼辦** — 監控、日誌與告警體系搭建

## 技術棧

本章涉及的核心工具：

| 工具 | 用途 |
|------|------|
| Docker | 容器化運行時 |
| Docker Compose | 多容器編排 |
| 1Panel | 可視化服務器管理面板 |
| Nginx | 反向代理與負載均衡 |
| PostgreSQL | 生產數據庫 |

## 學完本章你將能夠

1. 獨立完成一個 Next.js + NestJS 應用的生產部署
2. 使用 Docker Compose 編排多服務應用
3. 配置 Nginx 反向代理和 HTTPS 證書
4. 搭建基礎的監控和日誌系統
5. 處理常見的部署問題和故障
