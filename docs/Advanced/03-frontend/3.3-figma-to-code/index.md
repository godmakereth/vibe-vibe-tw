# 3.3 能不能讓 AI 看懂設計稿——Figma 集成實戰與 AI 協作

### 一句話破題

將設計稿截圖或鏈接提供給 AI，配合清晰的結構描述，讓 AI 生成接近設計稿的代碼。

### 核心價值

傳統的設計到開發流程需要開發者逐像素還原設計稿，耗時且容易出錯。在 Vibe Coding 體系下，AI 可以"看懂"設計稿並生成基礎代碼，你的任務是**驗收和微調**。

### 設計稿到代碼的工作流

```mermaid
graph LR
    A["設計稿"] --> B["截圖/導出"]
    B --> C["結構化描述"]
    C --> D["AI 生成代碼"]
    D --> E["Review 驗收"]
    E --> F["微調完善"]
```

### 準備工作：從 Figma 獲取信息

**方式一：截圖法（推薦新手）**

1. 在 Figma 中選中目標組件或頁面
2. 截取清晰的截圖
3. 將截圖粘貼到 AI 對話中

**方式二：Dev Mode 導出**

Figma 的 Dev Mode 可以導出：
- CSS 屬性（顏色、間距、字體）
- 組件結構
- 資源文件

**方式三：Figma 插件**

一些插件可以直接生成代碼：
- **Anima**：導出 React/Vue 代碼
- **Locofy**：AI 驅動的代碼生成
- **Builder.io**：可視化到代碼

### AI 協作：結構化描述設計稿

僅僅給 AI 一張圖是不夠的。你需要用**結構化語言**描述設計意圖：

**有效的 Prompt 模板：**

```
請根據這張設計稿生成 React + Tailwind 代碼：

**頁面/組件名稱**：用戶資料卡片

**整體佈局**：
- 卡片容器，圓角陰影
- 左側：用戶頭像（圓形，64px）
- 右側：用戶信息（姓名、職位、簡介）

**交互要求**：
- 鼠標懸停時卡片輕微上浮
- 點擊卡片跳轉到用戶詳情頁

**樣式細節**：
- 背景色：白色
- 陰影：輕微
- 間距：內邊距 16px

**技術要求**：
- 使用 Tailwind CSS
- 組件接收 user 對象作爲 Props
```

### 關鍵技巧：分層描述

將設計稿分解爲多個層次：

| 層次 | 描述內容 | 示例 |
|------|----------|------|
| **佈局層** | 整體結構、容器、網格 | "兩欄佈局，左側固定 240px" |
| **組件層** | 獨立的 UI 單元 | "卡片、按鈕、輸入框" |
| **樣式層** | 顏色、字體、間距 | "主色調 #3B82F6，圓角 8px" |
| **交互層** | 懸停、點擊、動畫 | "hover 時放大 1.05 倍" |

### 常見場景實戰

**場景一：還原導航欄**

```
設計稿：[粘貼截圖]

請生成頂部導航欄代碼：
- 左側：Logo 圖片
- 中間：導航鏈接（首頁、產品、關於）
- 右側：登錄/註冊按鈕
- 移動端：漢堡菜單
- 使用 Next.js Link 組件
- 當前頁面鏈接高亮
```

**場景二：還原表單**

```
設計稿：[粘貼截圖]

請生成登錄表單：
- 郵箱輸入框（帶圖標）
- 密碼輸入框（帶顯示/隱藏切換）
- 記住我複選框
- 登錄按鈕（主色調）
- 忘記密碼鏈接
- 使用 shadcn/ui 組件
- 添加表單驗證提示
```

### 驗收清單

AI 生成代碼後，按以下清單驗收：

- [ ] **佈局正確**：元素位置、間距與設計稿一致
- [ ] **響應式**：在不同屏幕尺寸下正常顯示
- [ ] **交互完整**：懸停、點擊等狀態正確
- [ ] **無障礙**：有 alt 文本、正確的語義標籤
- [ ] **代碼質量**：無重複代碼、Props 類型正確

### 常見問題與解決

**問題：AI 生成的佈局偏差較大**

解決：
1. 提供更詳細的佈局描述（使用 Flexbox/Grid 術語）
2. 分步生成：先佈局，再填充內容
3. 指定具體的像素值或比例

**問題：顏色不準確**

解決：
1. 從 Figma 複製精確的顏色值
2. 在 Prompt 中明確指定：`背景色 #F3F4F6`
3. 建立設計令牌（Design Tokens）

**問題：組件風格不統一**

解決：
1. 先建立基礎組件庫（Button、Input 等）
2. 在 Prompt 中引用已有組件
3. 使用 shadcn/ui 保持一致性

### AI 協作指南

**核心意圖**：讓 AI 理解設計稿並生成可用代碼。

**需求定義公式**：
- 視覺描述：[截圖] + 佈局說明
- 交互描述：用戶操作 + 系統響應
- 技術約束：使用的框架/組件庫

**關鍵術語**：`Flexbox`、`Grid`、`Tailwind`、`響應式`、`設計令牌`

**交互策略**：
1. 先給 AI 看整體設計，讓它理解上下文
2. 分組件逐個生成，而非一次性生成整頁
3. 生成後立即在瀏覽器中驗證
4. 發現偏差時，給 AI 具體的修改指令

### 工具推薦

| 工具 | 用途 | 特點 |
|------|------|------|
| **v0.dev** | 自然語言生成 UI | Vercel 出品，生成 shadcn 組件 |
| **Claude** | 識圖 + 代碼生成 | 多模態，可直接看圖 |
| **Cursor** | IDE 內設計還原 | 可粘貼圖片到對話 |

### 最佳實踐總結

1. **建立設計系統**：統一顏色、字體、間距，減少每次描述的成本
2. **組件優先**：先生成可複用組件，再組合成頁面
3. **增量開發**：不要一次性生成整個頁面，分塊進行
4. **及時驗收**：每生成一個組件就驗證，避免錯誤累積
5. **保留原稿**：將設計稿截圖保存在項目中，方便後續對照
