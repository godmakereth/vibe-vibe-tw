# 11.2.2 質量檢查：測試/構建/安全掃描

## 一句話破題

質量門禁通過**自動化檢查**確保代碼符合標準，不合格的代碼無法合併到主分支。

## 核心價值

配置質量門禁能讓你：
- 自動攔截有問題的代碼
- 減少人工 Review 的負擔
- 保持代碼庫的整潔和一致性

## 完整的質量檢查配置

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  # 類型檢查
  typecheck:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - run: npm run typecheck

  # 代碼規範檢查
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - run: npm run lint

  # 單元測試
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - run: npm test -- --coverage
      - name: Upload coverage
        uses: codecov/codecov-action@v3

  # 構建驗證
  build:
    runs-on: ubuntu-latest
    needs: [typecheck, lint, test]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - run: npm run build
```

## 依賴安全掃描

```yaml
  # 依賴漏洞掃描
  security:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run npm audit
        run: npm audit --audit-level=high
      
      # 或使用 Snyk
      - name: Run Snyk
        uses: snyk/actions/node@master
        env:
          SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
```

## 配置分支保護規則

在 GitHub 倉庫設置中啓用分支保護：

```
Settings → Branches → Branch protection rules → Add rule

勾選：
☑ Require a pull request before merging
☑ Require status checks to pass before merging
  └─ 選擇: typecheck, lint, test, build
☑ Require branches to be up to date before merging
```

## 檢查失敗時的處理

```yaml
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci
      - name: Run ESLint
        run: npm run lint
        continue-on-error: false  # 失敗時阻斷流程
      
      # 可選：上傳錯誤報告
      - name: Upload lint report
        if: failure()
        uses: actions/upload-artifact@v4
        with:
          name: lint-report
          path: lint-results.json
```

## 避坑指南

::: danger 新手最容易犯的錯
1. 檢查太多導致 CI 時間過長（應並行執行）
2. 沒有配置分支保護規則（檢查可被繞過）
3. 忽略安全掃描結果
4. 測試覆蓋率閾值設置過高導致頻繁失敗
:::
