# 11.3.3 教學材料：案例研究與練習題

## 一句話破題

把項目經驗轉化爲**可教學的材料**，是知識傳承的最高形式——你不僅記住了，還能讓別人也學會。

## 核心價值

製作教學材料能讓你：
- 深化自己對知識的理解
- 形成可複用的培訓資源
- 建立技術影響力

## 案例研究模板

```markdown
# 案例研究：[問題名稱]

## 背景
項目/功能的簡要描述，讀者需要知道的上下文

## 挑戰
我們遇到了什麼具體問題？有什麼約束條件？

## 探索過程
1. 首先我們嘗試了 A 方案...
2. 發現 A 方案的問題是...
3. 然後我們考慮 B 方案...

## 最終方案
詳細描述最終採用的解決方案

```代碼示例```

## 效果驗證
方案實施後的效果數據

## 經驗總結
- 關鍵洞察 1
- 關鍵洞察 2
- 如果重來會怎麼做

## 延伸思考
- 這個方案還能用在哪裏？
- 有什麼侷限性？
```

## 案例研究示例

```markdown
# 案例研究：優化首頁加載速度

## 背景
電商首頁需要展示推薦商品、Banner、分類導航等多個模塊

## 挑戰
首頁加載時間超過 5 秒，LCP 指標不達標

## 探索過程
1. 首先用 Lighthouse 分析，發現圖片是主要瓶頸
2. 嘗試壓縮圖片，效果有限
3. 考慮懶加載，但首屏圖片不能懶加載

## 最終方案
採用漸進式圖片加載 + 骨架屏：

```tsx
<Image
  src={product.image}
  placeholder="blur"
  blurDataURL={product.thumbnail}
  priority={index < 4}  // 首屏前4張優先加載
/>
```

## 效果驗證
- LCP 從 5.2s 降到 1.8s
- 跳出率下降 23%

## 經驗總結
- 優化要有數據支撐
- 首屏優化比全局優化更有效
```

## 教學材料清單

| 類型 | 用途 | 示例 |
|------|------|------|
| **案例研究** | 展示真實問題解決過程 | 性能優化案例 |
| **練習題** | 讓學習者動手實踐 | 實現登錄功能 |
| **代碼模板** | 提供起點代碼 | 項目腳手架 |
| **Checklist** | 驗證學習成果 | 安全檢查清單 |

## 避坑指南

::: danger 新手最容易犯的錯
1. 案例過於簡單，沒有體現真實複雜度
2. 練習題沒有漸進式難度
3. 只給答案不解釋爲什麼
4. 忽略了"踩坑"過程的價值
:::
