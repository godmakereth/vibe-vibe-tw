# 1.1.4 親眼看到第一個頁面——Hello World 驗證：本地開發服務器啓動

### 一句話破題

這是全棧開發的"開機時刻"——啓動開發服務器，在瀏覽器中看到你的第一個頁面。

### 核心價值

Hello World 不僅僅是一個測試頁面，它驗證了整個開發環境的正確配置。如果你能看到這個頁面，說明：

- Node.js 運行正常
- 項目依賴安裝成功
- Next.js 編譯器工作正常
- 本地服務器可以啓動

### 啓動開發服務器

確保你在項目目錄下，運行：

```bash
pnpm dev
```

你會看到類似這樣的輸出：

```
  ▲ Next.js 16.x.x
  - Local:        http://localhost:3000
  - Environments: .env.local

 ✓ Ready in 2.1s
```

### 訪問你的頁面

打開瀏覽器，訪問 `http://localhost:3000`。你應該能看到 Next.js 的默認歡迎頁面。

**恭喜！你已經成功跑通了第一個 Next.js 項目。**

### 修改頁面內容

讓我們做一個簡單的修改來驗證熱更新：

1. 用編輯器打開 `src/app/page.tsx`
2. 找到頁面中的文字，改成你想要的內容
3. 保存文件

你會發現**瀏覽器自動刷新**，顯示你修改後的內容。這就是 Next.js 的熱模塊替換功能，它讓你的開發體驗非常絲滑。

### 第一次修改示例

把 `src/app/page.tsx` 的內容替換爲：

```tsx
export default function Home() {
  return (
    <main className="flex min-h-screen flex-col items-center justify-center p-24">
      <h1 className="text-4xl font-bold">我的第一個 Next.js 應用</h1>
      <p className="mt-4 text-gray-600">Hello, Vibe Coding!</p>
    </main>
  );
}
```

保存後，瀏覽器會自動顯示新內容。

### 開發服務器的常用操作

| 操作 | 快捷鍵/命令 |
|------|-------------|
| 啓動服務器 | `pnpm dev` |
| 停止服務器 | `Ctrl + C` |
| 重啓服務器 | 停止後再啓動 |
| 清除緩存重啓 | `rm -rf .next && pnpm dev` |

### 常見問題排查

**問題 1：端口被佔用**

```
Error: listen EADDRINUSE: address already in use :::3000
```

解決方案：換一個端口啓動

```bash
pnpm dev -p 3001
```

**問題 2：依賴未安裝**

```
Cannot find module 'xxx'
```

解決方案：重新安裝依賴

```bash
pnpm install
```

**問題 3：頁面白屏**

打開瀏覽器開發者工具（F12），查看 Console 面板的錯誤信息，複製給 AI 幫你排查。
