# 1.1 先讓 Hello World 跑起來——工具裝配與最小項目驗證

### 本節目標

在動手寫任何業務代碼之前，你需要先擁有一套能用的開發環境。本節的唯一目標是：**讓你在本地瀏覽器裏看到一個 Next.js 頁面**。

這看似簡單，卻是整個全棧開發的第一個里程碑。一旦你跑通了這個最小閉環，後面所有的功能開發都只是在這個基礎上"疊加"。

### 你需要準備的工具

| 工具類型 | 推薦選擇 | 作用 |
|----------|----------|------|
| **代碼編輯器** | Cursor / VS Code | 編寫和管理代碼 |
| **AI 助手** | Claude / GPT-4 | 輔助編程、答疑解惑 |
| **運行時環境** | Node.js (LTS) | 運行 JavaScript/TypeScript |
| **包管理器** | pnpm | 管理項目依賴 |

### 子章節導航

- **1.1.1 編輯器選擇**：VS Code 與 Cursor 的功能對比，幫你做出選擇
- **1.1.2 AI 助手配置**：讓你的 IDE "長出大腦"
- **1.1.3 Next.js 初始化**：用一條命令生成項目骨架
- **1.1.4 Hello World 驗證**：啓動服務器，見證第一個頁面

### 預計耗時

對於完全零基礎的同學，完成本節全部內容大約需要 **30-60 分鐘**。如果你已經有一定開發經驗，可以跳過熟悉的部分，直接進入 1.1.3。
