# 1.1.1 神器如何選——編輯器選擇：VSCode vs Cursor 功能對比

### 一句話破題

Cursor 是 VS Code 的 AI 增強版——它在 VS Code 的基礎上深度集成了 AI 能力，讓你可以用自然語言和代碼"對話"。

### 核心價值

選對編輯器，決定了你後續的開發體驗。在 Vibe Coding 時代，編輯器不僅是寫代碼的地方，更是你與 AI 協作的"指揮部"。

### 快速對比

| 特性 | VS Code | Cursor |
|------|---------|--------|
| **基礎編輯** | ✅ 完善 | ✅ 完善（繼承 VS Code） |
| **插件生態** | ✅ 海量 | ✅ 兼容 VS Code 插件 |
| **AI 對話** | ⚠️ 需安裝插件 | ✅ 原生支持 |
| **代碼補全** | ⚠️ 需 Copilot 訂閱 | ✅ 內置 AI 補全 |
| **上下文理解** | ⚠️ 有限 | ✅ 可索引整個代碼庫 |
| **價格** | 免費 | 免費版 + Pro 訂閱 |

### 我的建議

**如果你是 Vibe Coding 的初學者，直接選 Cursor。**

原因很簡單：Cursor 把 AI 能力做成了"一等公民"。你不需要折騰插件配置，開箱即用。它的核心功能包括：

1. **Cmd/Ctrl + K**：選中代碼後，用自然語言描述你想要的修改
2. **Cmd/Ctrl + L**：打開 AI 對話面板，討論代碼問題
3. **Tab 補全**：AI 預測你接下來要寫的代碼

### 快速上手

1. 訪問 [cursor.com](https://cursor.com) 下載安裝
2. 首次啓動時，它會詢問是否導入 VS Code 配置，選擇"是"
3. 登錄賬號，免費版每月有一定的 AI 調用額度

### 避坑指南

- **別糾結於工具選擇**：無論選哪個，都能完成本課程。工具是爲了解決問題，不是用來比較的
- **VS Code 老用戶**：如果你已經有深度定製的 VS Code 配置，可以先用 GitHub Copilot 插件體驗 AI 編程，之後再考慮遷移
- **網絡問題**：Cursor 需要聯網調用 AI 服務，確保你的網絡環境暢通
