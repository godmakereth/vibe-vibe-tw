# 1.2.3 如何向 AI 下達指令——提示工程基礎：如何與 AI 有效溝通

### 一句話破題

提示工程的本質是**用結構化的語言表達清晰的意圖**。好的提示詞不是華麗的修辭，而是精準的需求定義。

### 核心原則：清晰、具體、結構化

#### 原則 1：清晰

AI 無法讀心，它只能理解你表達出來的內容。

```
❌ 模糊：幫我優化一下這個代碼
✅ 清晰：幫我優化這個函數的性能，目前它處理 10000 條數據需要 5 秒，目標是降到 1 秒以內
```

#### 原則 2：具體

提供足夠的上下文，讓 AI 知道你在什麼場景下需要什麼。

```
❌ 泛泛：寫一個登錄功能
✅ 具體：用 Next.js 16 App Router 寫一個登錄頁面，使用 Server Actions 處理表單提交，
        支持郵箱+密碼登錄，使用 Tailwind CSS 做樣式
```

#### 原則 3：結構化

用清晰的結構組織你的需求，讓 AI 更容易理解和執行。

```
## 需求描述
實現一個用戶評論組件

## 技術要求
- 框架：Next.js 16
- 樣式：Tailwind CSS
- 數據：從 API 獲取評論列表

## 功能要點
1. 顯示評論列表（頭像、用戶名、內容、時間）
2. 支持點贊
3. 支持回覆

## 約束條件
- 需要處理加載狀態和空狀態
- 需要處理錯誤情況
```

### 提示詞模板：功能描述 + 交互方式 + 預期效果

這是一個萬能的需求定義公式：

```
[功能描述]：我想要實現什麼功能
[交互方式]：用戶如何與這個功能交互
[預期效果]：成功後應該看到什麼結果
```

**示例**：

```
功能描述：實現一個搜索框組件
交互方式：用戶輸入關鍵詞，按回車或點擊搜索按鈕觸發搜索
預期效果：搜索時顯示加載動畫，結果返回後展示搜索結果列表，無結果時顯示"未找到相關內容"
```

### 分步對話策略

不要試圖一次讓 AI 生成所有代碼。分步走，每一步都確認無誤後再進行下一步：

```mermaid
graph LR
    A["第一步：數據結構"] --> B["確認 OK"]
    B --> C["第二步：核心邏輯"]
    C --> D["確認 OK"]
    D --> E["第三步：UI 樣式"]
    E --> F["確認 OK"]
    F --> G["第四步：錯誤處理"]
```

**對話示例**：

```
你：幫我設計一個博客文章的數據結構，需要包含標題、內容、作者、發佈時間、標籤

AI：[生成數據結構]

你：很好，現在基於這個數據結構，幫我寫一個獲取文章列表的 API

AI：[生成 API 代碼]

你：API 沒問題，現在幫我寫一個展示文章列表的組件，使用剛纔的 API

AI：[生成組件代碼]
```

### 關鍵術語清單

在與 AI 討論代碼時，使用準確的技術術語可以大幅提高溝通效率：

| 場景 | 關鍵術語 |
|------|----------|
| **Next.js** | App Router, Server Component, Client Component, Server Actions, Middleware |
| **React** | useState, useEffect, props, children, 條件渲染, 列表渲染 |
| **TypeScript** | interface, type, 泛型, 聯合類型, 類型守衛 |
| **Tailwind** | flex, grid, responsive, dark mode, 自定義主題 |
| **數據庫** | CRUD, 關聯查詢, 事務, 索引, 遷移 |

### 常見問題的提示詞模板

#### 需求開發

```
請幫我實現 [功能名稱]。

技術棧：[Next.js/React/TypeScript 等]
要求：
1. [具體要求 1]
2. [具體要求 2]
3. [具體要求 3]

參考：[如果有類似功能的參考，貼上來]
```

#### Bug 修復

```
我遇到了一個問題：[描述問題現象]

相關代碼：
[粘貼相關代碼]

錯誤信息：
[粘貼完整錯誤信息]

我已經嘗試過：
1. [嘗試過的方法 1]
2. [嘗試過的方法 2]
```

#### 代碼優化

```
請幫我優化以下代碼：

[粘貼代碼]

優化目標：
- [性能優化 / 可讀性優化 / 安全性優化]

約束條件：
- [保持 API 不變 / 兼容舊版本 / 等]
```

### 避坑指南

- **不要一口氣要求太多**：任務越大，AI 出錯的概率越高
- **及時糾正錯誤**：發現問題立即反饋，不要等到最後
- **保存有效的提示詞**：好的提示詞可以複用
- **理解 AI 的侷限性**：它的知識有截止日期，新特性可能不瞭解
