# 1.5.1 磨刀不誤砍柴工——IDE 選擇：VS Code 配置與插件推薦

### 一句話破題

VS Code 是全球最流行的代碼編輯器，通過合理的插件配置，可以打造一個高效的全棧開發環境。

### 推薦配置

在 VS Code 設置（settings.json）中添加以下配置：

```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": "explicit",
    "source.organizeImports": "explicit"
  },
  "editor.tabSize": 2,
  "editor.wordWrap": "on",
  "files.autoSave": "onFocusChange",
  "typescript.preferences.importModuleSpecifier": "relative",
  "tailwindCSS.includeLanguages": {
    "typescript": "javascript",
    "typescriptreact": "javascript"
  },
  "emmet.includeLanguages": {
    "javascript": "javascriptreact",
    "typescript": "typescriptreact"
  }
}
```

### Cursor 用戶說明

如果你使用 Cursor，以上配置同樣適用。Cursor 基於 VS Code 構建，完全兼容 VS Code 的插件和配置。

額外的 Cursor 特有配置可以在 Cursor 設置中找到，主要是 AI 相關的選項（已在 1.1.2 節介紹）。

### 工作區配置

對於團隊項目，建議在項目根目錄創建 `.vscode/settings.json`，確保團隊成員使用統一的配置：

```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "[typescript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[typescriptreact]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  }
}
```

### 避坑指南

- **插件不要裝太多**：只裝需要的，太多插件會拖慢編輯器
- **保持插件更新**：過時的插件可能有兼容性問題
- **備份你的配置**：使用 Settings Sync 功能同步配置到雲端
