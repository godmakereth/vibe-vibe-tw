# 5.5.2 我們要解決什麼——問題陳述

### 一句話破題

問題陳述回答三個核心問題：**現狀是什麼、目標是什麼、約束是什麼**。

### 問題陳述的結構

```markdown
## 問題陳述

### 現狀（As-Is）
當前用戶面臨什麼問題？痛點是什麼？

### 目標（To-Be）
理想狀態是什麼？解決後會怎樣？

### 約束（Constraints）
有哪些限制條件？時間、資源、技術？
```

### 實際案例

```markdown
## 問題：技術博客內容難以管理

### 現狀
- 技術人員使用第三方平臺寫博客，受制於平臺審覈
- 內容分散在多個平臺，難以統一管理
- 擔心平臺關閉導致內容丟失
- 現有方案（如 Hexo）配置繁瑣

### 目標
- 擁有完全可控的個人博客系統
- 支持 Markdown 寫作，專注內容
- 良好的 SEO，被搜索引擎收錄
- 部署簡單，維護成本低

### 約束
- 獨立開發者，開發時間有限
- 預算有限，儘量使用免費資源
- 技術棧限定：Next.js + PostgreSQL
- 兩週內完成 MVP
```

### 好的問題陳述 vs 差的問題陳述

| 差的 | 好的 |
|------|------|
| "用戶體驗不好" | "用戶平均需要點擊 5 次才能完成註冊，導致 60% 的用戶放棄" |
| "性能需要優化" | "首頁加載時間超過 5 秒，在 3G 網絡下無法正常使用" |
| "功能不夠完善" | "用戶無法搜索歷史文章，只能翻頁瀏覽" |

### 讓 AI 幫你完善問題陳述

```
你：我想做一個博客系統，幫我寫問題陳述

AI：## 問題陳述：個人技術博客

### 現狀分析
1. **平臺依賴**
   - 技術文章發佈在 CSDN/掘金等平臺
   - 受制於平臺審覈，敏感內容可能被刪
   - 廣告干擾閱讀體驗

2. **內容管理困難**
   - 文章分散在多個平臺
   - 缺乏統一的分類和標籤體系
   - 無法控制內容的展示方式

3. **遷移風險**
   - 平臺可能調整政策或關閉
   - 導出功能有限，遷移困難

### 目標狀態
1. **完全可控**：自主託管，數據完全歸屬於自己
2. **專注寫作**：支持 Markdown，無需關心排版
3. **可被發現**：良好的 SEO，被搜索引擎收錄
4. **易於維護**：部署簡單，無需頻繁運維

### 約束條件
- 時間：2 周內完成 MVP
- 技術：使用 Next.js 16 + Prisma
- 成本：服務器費用控制在 $10/月以內
- 人力：一人開發
```

### 驗證問題陳述

問自己幾個問題：

1. **這是真問題嗎？** - 是否有足夠多的人遇到這個問題
2. **問題描述具體嗎？** - 能否用數據量化
3. **目標可衡量嗎？** - 如何判斷問題已解決
4. **約束合理嗎？** - 是否能在約束內達成目標

### 實用建議

1. **先問"爲什麼"**：挖掘問題背後的真正原因
2. **用數據說話**：儘量用數字描述現狀
3. **明確優先級**：哪些問題必須解決，哪些可以接受
4. **考慮權衡**：有些問題的解決可能帶來新問題
