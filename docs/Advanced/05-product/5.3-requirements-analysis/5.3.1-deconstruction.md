# 5.3.1 如何看透題目要求——AI 輔助需求拆解

### 一句話破題

讓 AI 幫你**從一句話需求中挖出十個具體問題**。

### 需求拆解的價值

一個看似簡單的需求，背後往往藏着大量細節：

```
原始需求："做一個用戶登錄功能"

隱藏的問題：
- 用什麼方式登錄？郵箱？手機號？第三方？
- 密碼有什麼要求？長度？複雜度？
- 忘記密碼怎麼辦？
- 需要驗證碼嗎？什麼情況下觸發？
- 登錄失敗幾次會鎖定賬號？
- Session 還是 JWT？有效期多久？
- 需要"記住我"功能嗎？
```

### 讓 AI 幫你提問

**Prompt 模板**：

```
我要實現 [功能描述]。

請扮演一個資深產品經理，從以下角度幫我列出需要澄清的問題：
1. 用戶角度：誰會使用？使用場景是什麼？
2. 功能角度：具體包含哪些子功能？
3. 邊界角度：什麼情況不支持？
4. 技術角度：有什麼技術約束？
5. 驗收角度：怎麼判斷做完了？
```

**示例對話**：

```
你：我要實現用戶登錄功能

AI（產品經理視角）：

## 需要澄清的問題

### 用戶角度
- 目標用戶是誰？普通用戶還是企業用戶？
- 用戶主要在什麼場景下登錄？PC/移動端/都有？

### 功能角度
- 支持哪些登錄方式？
  - [ ] 郵箱+密碼
  - [ ] 手機號+驗證碼
  - [ ] 第三方登錄（微信/Google/GitHub）
- 是否需要"記住登錄狀態"？
- 是否需要多設備登錄限制？

### 邊界角度
- 密碼要求：最小長度？需要包含特殊字符？
- 登錄失敗處理：幾次失敗後鎖定？鎖定多久？
- 是否需要驗證碼？什麼情況下觸發？

### 技術角度
- 認證方式：Session 還是 JWT？
- Token 有效期多久？
- 是否需要 refresh token？

### 驗收角度
- 正常登錄成功後跳轉到哪裏？
- 登錄失敗要顯示什麼錯誤信息？
```

### 拆解後的輸出

經過 AI 輔助拆解後，你應該能得到：

1. **確定的決策**：郵箱+密碼登錄，不支持第三方
2. **明確的規則**：密碼 8-20 位，必須包含字母和數字
3. **清晰的邊界**：失敗 5 次鎖定 30 分鐘
4. **驗收標準**：登錄成功返回 token，失敗返回錯誤碼

### 實用建議

1. **先讓 AI 提問，再自己回答**：AI 提的問題比你想的全面
2. **不確定的先記錄**：暫時回答不了的問題標記爲"待定"
3. **形成決策文檔**：把問答整理成文檔，作爲後續開發的依據
