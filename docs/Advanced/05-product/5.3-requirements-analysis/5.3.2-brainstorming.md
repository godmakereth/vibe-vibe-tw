# 5.3.2 我該做什麼功能——AI 輔助頭腦風暴

### 一句話破題

讓 AI 幫你**從一個想法生成十種可能**，然後你來選最好的。

### 頭腦風暴的困境

一個人想方案時，容易陷入思維定式：

```
你的想法："做一個博客"
你的方案：文章列表 + 詳情頁 + 編輯器

問題：這是你見過的博客的樣子，但不一定是最好的方案
```

### AI 如何幫你發散思維

**Prompt 模板**：

```markdown
我想做 [產品描述]。

請幫我進行頭腦風暴：
1. 列出 5 種不同的產品形態
2. 每種形態說明：核心特點、適合場景、實現難度
3. 推薦最適合獨立開發者的方案
```

**示例對話**：

```
你：我想做一個個人博客系統

AI：以下是 5 種不同的博客形態：

## 1. 傳統博客
- 核心特點：文章列表 + 分類標籤 + 評論
- 適合場景：內容創作者、技術分享
- 實現難度：⭐⭐（中等）

## 2. 知識庫/Wiki 風格
- 核心特點：雙向鏈接、知識圖譜、全文搜索
- 適合場景：知識管理、學習筆記
- 實現難度：⭐⭐⭐（較難）

## 3. 微博客/說說
- 核心特點：短內容、時間線、快速發佈
- 適合場景：碎片化記錄、日常分享
- 實現難度：⭐（簡單）

## 4. Newsletter 訂閱制
- 核心特點：郵件訂閱、定期推送、付費牆
- 適合場景：內容變現、精品內容
- 實現難度：⭐⭐⭐（較難）

## 5. 靜態博客 + 評論服務
- 核心特點：靜態生成、CDN 加速、第三方評論
- 適合場景：技術博客、追求性能
- 實現難度：⭐⭐（中等）

## 推薦
對於獨立開發者，推薦方案 1 或 5：
- 方案 1 功能完整，適合學習全棧開發
- 方案 5 性能好，維護成本低
```

### 功能發散：挖掘隱藏需求

當確定產品形態後，可以繼續發散功能點：

```
你：我選擇做傳統博客。除了基本的文章發佈，還有哪些功能可以考慮？

AI：按優先級分類：

## 核心功能（必須有）
- 文章 CRUD
- Markdown 編輯器
- 分類和標籤

## 增強功能（建議有）
- 全文搜索
- 文章歸檔（按時間）
- RSS 訂閱
- SEO 優化（sitemap、meta）

## 進階功能（錦上添花）
- 評論系統（或集成 Giscus）
- 文章閱讀統計
- 深色模式
- 多語言支持

## 高級功能（可選）
- 草稿自動保存
- 定時發佈
- 文章協作編輯
- 會員訂閱
```

### 技術方案發散

同一個功能，可能有多種實現方式：

```
你：文章編輯器應該怎麼做？

AI：有 3 種主流方案：

## 方案 1：純 Markdown
- 實現：textarea + marked.js 預覽
- 優點：實現簡單、存儲方便
- 缺點：非技術用戶學習成本高

## 方案 2：富文本編輯器
- 工具：TipTap、Quill、Editor.js
- 優點：所見即所得、用戶友好
- 缺點：存儲格式複雜、跨平臺麻煩

## 方案 3：塊編輯器（Notion 風格）
- 工具：Editor.js、Plate.js
- 優點：靈活、現代
- 缺點：實現複雜

## 推薦
初期用方案 1，快速上線後再迭代
```

### 發散後的收斂

頭腦風暴的目的不是收集所有可能，而是**找到最適合當前階段的方案**：

```mermaid
graph LR
    A["一個想法"] --> B["AI 發散\n5+ 種可能"]
    B --> C["你來篩選\n2-3 個候選"]
    C --> D["深入分析\n比較優劣"]
    D --> E["確定方案\n1 個最優解"]
```

### 實用建議

1. **先發散再收斂**：不要一開始就否定可能性
2. **讓 AI 對比**：給出候選方案，讓 AI 分析優劣
3. **考慮資源約束**：獨立開發 vs 團隊開發的方案不同
4. **分階段實現**：選擇可漸進式擴展的方案
