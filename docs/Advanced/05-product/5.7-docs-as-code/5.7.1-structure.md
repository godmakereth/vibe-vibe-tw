# 5.7.1 文檔該放哪裏——文檔目錄結構

### 一句話破題

文檔目錄應該**與代碼結構對應**，讓人一看就知道去哪找。

### 推薦的目錄結構

```
my-project/
├── docs/                   # 文檔目錄
│   ├── README.md           # 文檔首頁
│   ├── getting-started.md  # 快速開始
│   ├── api/                # API 文檔
│   │   ├── auth.md         # 認證相關
│   │   └── posts.md        # 文章相關
│   ├── guides/             # 使用指南
│   │   ├── deployment.md   # 部署指南
│   │   └── development.md  # 開發指南
│   └── architecture/       # 架構說明
│       └── overview.md     # 架構概覽
├── src/                    # 源代碼
│   ├── app/
│   │   └── api/
│   │       ├── auth/       ← 對應 docs/api/auth.md
│   │       └── posts/      ← 對應 docs/api/posts.md
│   └── ...
└── README.md               # 項目入口
```

### 文檔與代碼的對應關係

| 代碼路徑 | 文檔路徑 | 內容 |
|----------|----------|------|
| `src/app/api/auth/` | `docs/api/auth.md` | 認證接口說明 |
| `src/app/api/posts/` | `docs/api/posts.md` | 文章接口說明 |
| `prisma/schema.prisma` | `docs/architecture/database.md` | 數據庫設計 |
| `src/lib/` | `docs/architecture/libs.md` | 工具函數說明 |

### 最小化文檔結構

對於個人項目，可以更簡化：

```
my-project/
├── docs/
│   ├── README.md      # 項目說明 + 快速開始
│   ├── api.md         # API 文檔（所有接口）
│   └── deployment.md  # 部署說明
├── src/
└── README.md          # 項目入口（指向 docs/）
```

### 文檔命名規範

```
✅ 好的命名
- getting-started.md   # 小寫 + 連字符
- api-authentication.md
- deploy-to-vercel.md

❌ 不好的命名
- GettingStarted.md    # 避免駝峯
- api authentication.md # 避免空格
- 部署說明.md          # 避免中文文件名
```

### 文檔入口設計

項目 README.md 應該提供清晰的導航：

```markdown
# My Blog

一個使用 Next.js 構建的個人博客系統。

## 快速開始

\`\`\`bash
pnpm install
pnpm dev
\`\`\`

## 文檔

- [開發指南](./docs/development.md)
- [API 文檔](./docs/api.md)
- [部署說明](./docs/deployment.md)

## 技術棧

- Next.js 16
- Prisma
- PostgreSQL
```

### 實用建議

1. **就近原則**：相關的文檔放在一起
2. **扁平優先**：不要嵌套太深
3. **命名直觀**：看文件名就知道內容
4. **及時歸檔**：過時文檔移到 archive 目錄
