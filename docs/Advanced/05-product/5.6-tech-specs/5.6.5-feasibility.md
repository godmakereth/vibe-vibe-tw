# 5.6.5 我一個人搞得定嗎——單人自評

### 一句話破題

獨立開發者需要誠實地評估：**這個項目我能在預期時間內完成嗎？**

### 可行性自檢清單

在開始之前，問自己這些問題：

| 問題 | 是 | 否 |
|------|----|----|
| 我理解業務需求嗎？ | 繼續 | 需要澄清 |
| 技術棧我熟悉嗎？ | 繼續 | 需要學習時間 |
| 有類似項目經驗嗎？ | 更準確 | 估算要加 buffer |
| 依賴的服務都瞭解嗎？ | 繼續 | 需要調研 |
| 時間充足嗎？ | 繼續 | 需要縮減範圍 |

### 工作量估算方法

#### 方法一：功能分解法

把項目拆成最小的功能單元，分別估算：

```markdown
## 博客系統工作量估算

### 後端開發
| 模塊 | 功能點 | 預估時間 |
|------|--------|----------|
| 用戶認證 | 登錄/註冊/登出 | 4h |
| 文章管理 | CRUD + 發佈狀態 | 6h |
| 分類標籤 | CRUD | 3h |
| 文件上傳 | 圖片上傳 | 3h |

### 前端開發
| 模塊 | 功能點 | 預估時間 |
|------|--------|----------|
| 首頁 | 文章列表 | 4h |
| 詳情頁 | 文章渲染 | 3h |
| 後臺 | 文章編輯器 | 6h |
| 認證 | 登錄表單 | 2h |

### 其他
| 項目 | 預估時間 |
|------|----------|
| 項目初始化 | 2h |
| 部署配置 | 3h |
| 測試調試 | 8h |
| 文檔 | 2h |

### 合計
- 開發時間：46h
- 加 30% buffer：60h
- 按每天 4h 有效時間：15 天
```

#### 方法二：類比法

參考類似項目的實際時間：

```
上次做類似項目：10 天
這次更復雜 × 1.5：15 天
新技術學習 + 2 天：17 天
```

### 獨立開發者時間陷阱

| 常見低估點 | 實際耗時 |
|------------|----------|
| "這個很簡單" | 往往不簡單 |
| "查個文檔就行" | 可能查半天 |
| "調試應該很快" | 經常最耗時 |
| "部署幾分鐘" | 環境問題多 |

**經驗法則**：初步估算後乘以 1.5 ~ 2

### 有效工作時間

獨立開發者一天的真即時間分佈：

```
一天 8 小時
├── 有效編碼時間：4h（最理想）
├── 調試/查資料：2h
├── 休息/中斷：1h
└── 溝通/雜事：1h
```

所以估算時用**有效時間**而不是總時間。

### AI 輔助開發的時間調整

使用 AI 輔助後，某些任務會變快：

| 任務類型 | 傳統耗時 | AI 輔助後 |
|----------|----------|-----------|
| 寫樣板代碼 | 1h | 0.2h |
| 查文檔/示例 | 0.5h | 0.1h |
| 調試錯誤 | 1h | 0.5h |
| 學習新 API | 2h | 0.5h |

但要注意：
- 審查 AI 代碼需要時間
- 複雜邏輯仍需自己思考
- 集成和測試不會變快

### 讓 AI 幫你估算

```
你：幫我估算這個博客系統的開發時間

技術棧：Next.js + Prisma + PostgreSQL
功能：文章 CRUD、分類標籤、用戶認證、Markdown 編輯
我對 Next.js 熟悉，Prisma 第一次用

AI：## 工作量估算

### 開發時間
| 階段 | 時間 | 說明 |
|------|------|------|
| 環境搭建 | 2h | 項目初始化、數據庫 |
| 數據模型 | 3h | Prisma Schema 設計 |
| 用戶認證 | 4h | next-auth 集成 |
| 文章模塊 | 8h | CRUD + 編輯器 |
| 分類標籤 | 4h | 基礎 CRUD |
| 前端頁面 | 10h | 列表/詳情/後臺 |
| 測試調試 | 6h | 各模塊聯調 |
| 部署上線 | 3h | Vercel + 數據庫 |

### 合計
- 基礎時間：40h
- Prisma 學習 buffer：+8h  
- 總計：48h
- 按每天 4h：12 天
- 建議預留：2 周
```

### 何時該縮減範圍

如果估算結果超出預期，優先考慮：

1. **砍功能**：先做核心功能
2. **用現成方案**：評論用第三方
3. **降低標準**：先能用，再好看
4. **分階段**：MVP → V1 → V2

### 實用建議

1. **誠實面對自己**：不要自欺欺人
2. **記錄實際時間**：下次估算更準
3. **預留 buffer**：總會有意外
4. **及時調整範圍**：發現不對勁就縮減
