# 5.4.4 爲什麼 AI 偏愛 Markdown——Markdown 格式優化

### 一句話破題

Markdown 是**人類可讀、機器可解析**的最佳平衡點。

### 爲什麼用 Markdown

| 格式 | 人類可讀性 | 機器可解析 | 版本控制友好 |
|------|-----------|------------|--------------|
| Word | ⭐⭐⭐ | ❌ | ❌ |
| PDF | ⭐⭐⭐ | ❌ | ❌ |
| HTML | ⭐ | ⭐⭐⭐ | ⭐⭐ |
| **Markdown** | ⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ |

Markdown 的優勢：
- **純文本格式**：可以用任何編輯器打開
- **結構清晰**：標題、列表、代碼塊層次分明
- **AI 友好**：大模型在訓練時大量使用 Markdown 格式
- **版本控制**：Git diff 清晰展示修改

### AI 如何"看"Markdown

AI 模型對 Markdown 的結構有天然的理解：

```markdown
# 一級標題 → AI 識別爲主題
## 二級標題 → AI 識別爲子主題
- 列表項 → AI 識別爲並列關係
1. 有序列表 → AI 識別爲步驟/順序

| 表頭 | 表頭 |
|------|------|
| 數據 | 數據 | → AI 識別爲結構化數據

```code``` → AI 識別爲代碼/配置
```

### 讓文檔更易解析的技巧

#### 1. 使用清晰的標題層級

```markdown
❌ 不好
功能一
這是功能描述
功能二
這是另一個功能

✅ 好
# 用戶管理模塊

## 功能一：用戶註冊
這是功能描述

## 功能二：用戶登錄
這是另一個功能
```

#### 2. 用表格代替長段落

```markdown
❌ 不好
郵箱字段是必填的，類型是字符串，格式要求是有效的郵箱格式。
密碼字段也是必填的，長度要求 8-20 字符。

✅ 好
| 字段 | 類型 | 必填 | 約束 |
|------|------|------|------|
| email | string | 是 | 有效郵箱格式 |
| password | string | 是 | 8-20 字符 |
```

#### 3. 用代碼塊展示數據結構

```markdown
❌ 不好
返回用戶 id、郵箱、姓名和創建時間

✅ 好
```json
{
  "id": "user_123",
  "email": "user@example.com",
  "name": "張三",
  "createdAt": "2024-01-15T10:00:00Z"
}
```
```

#### 4. 用列表展示步驟

```markdown
❌ 不好
首先驗證輸入，然後查詢數據庫，接着創建用戶，最後返回結果

✅ 好
1. 驗證輸入參數
2. 查詢郵箱是否已存在
3. 創建用戶記錄
4. 返回用戶信息
```

### Markdown 格式規範

#### 標題規範
```markdown
# H1 只用於文檔標題
## H2 用於主要章節
### H3 用於子章節
#### H4 用於細分內容
```

#### 列表規範
```markdown
- 無序列表用於並列項目
- 用 `-` 而不是 `*`

1. 有序列表用於步驟
2. 或者有先後順序的內容
```

#### 代碼塊規範
```markdown
行內代碼用 `反引號`

多行代碼塊指定語言：
```typescript
const user = await prisma.user.create({
  data: { email, password }
})
```
```

#### 表格規範
```markdown
| 左對齊 | 居中 | 右對齊 |
|:-------|:----:|-------:|
| 數據 | 數據 | 數據 |
```

### 常用 Markdown 元素

```markdown
## 強調
**粗體** 用於重要內容
*斜體* 用於術語定義
`代碼` 用於變量名、命令

## 鏈接和引用
[鏈接文字](url)
> 引用塊用於注意事項

## 任務列表
- [x] 已完成
- [ ] 未完成

## 分隔線
```

### 在 AI 對話中使用 Markdown

向 AI 提供需求時，使用 Markdown 格式效果更好：

```markdown
## 任務
實現用戶註冊 API

## 技術約束
- Next.js 16 App Router
- Prisma ORM
- PostgreSQL

## 輸入
| 字段 | 類型 | 必填 |
|------|------|------|
| email | string | 是 |
| password | string | 是 |

## 輸出
成功返回用戶信息和 JWT token
```

### 實用建議

1. **統一風格**：整個項目使用相同的 Markdown 風格
2. **善用工具**：VS Code + Markdown 插件可以即時預覽
3. **保持簡潔**：不要過度使用格式，內容清晰最重要
4. **注意兼容性**：不同平臺對 Markdown 擴展支持不同
