# 5.4 你寫的文檔 AI 看得懂嗎——AI 可讀的 PRD

### 爲什麼文檔格式很重要

在 Vibe Coding 中，文檔不僅要讓人能讀懂，更要讓 AI 能"理解"。

```mermaid
graph LR
    A["模糊的描述"] --> B["AI 猜測你的意圖"]
    B --> C["生成的代碼可能偏離"]
    
    D["結構化的文檔"] --> E["AI 精確理解"]
    E --> F["生成的代碼符合預期"]
```

**好文檔 vs 差文檔**：

| 差文檔 | 好文檔 |
|--------|--------|
| "做一個好看的頁面" | "使用 Tailwind，卡片圓角 8px，間距 16px" |
| "用戶登錄後跳轉" | "登錄成功返回 200，跳轉到 /dashboard" |
| "處理錯誤情況" | "密碼錯誤返回 401，格式錯誤返回 400" |

### 結構化文檔的核心要素

一份 AI 友好的 PRD 應該包含：

1. **明確的輸入輸出**：數據從哪來，到哪去
2. **具體的約束條件**：技術棧、格式、邊界
3. **清晰的異常處理**：各種錯誤情況如何響應
4. **可驗證的成功標準**：怎麼判斷"做對了"

### 本節目標

學完本節，你將掌握：

1. **結構化設計**：讓文檔模塊清晰、層次分明
2. **輸入輸出定義**：明確數據流轉和接口契約
3. **邊界條件處理**：預判異常情況的系統響應
4. **格式優化**：使用 AI 偏愛的 Markdown 格式

**記住**：寫文檔就是寫 Prompt。文檔越清晰，AI 理解越準確，生成的代碼就越符合預期。
