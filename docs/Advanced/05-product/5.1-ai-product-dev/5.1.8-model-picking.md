# 5.1.8 哪家 AI 最懂我——模型選擇與應用

### 一句話破題

不同的 AI 模型有不同的特長，**選對工具比用力更重要**。

### 主流 AI 工具對比

| 工具 | 特長 | 適用場景 | 注意事項 |
|------|------|----------|----------|
| **Claude** | 長文本理解、代碼質量 | 複雜需求分析、代碼審查 | 上下文窗口大 |
| **GPT-4** | 通用能力均衡 | 日常開發任務 | 廣泛的知識面 |
| **Cursor** | IDE 集成、代碼補全 | 即時編碼輔助 | 需要訂閱 |
| **GitHub Copilot** | 代碼補全 | 寫代碼時的即時建議 | VS Code 集成好 |

### 場景化選擇指南

#### 場景 1：理解複雜需求

**推薦**：Claude / GPT-4

```
場景：拿到一份 10 頁的產品需求文檔，需要提取關鍵信息

做法：
1. 把文檔餵給 AI
2. 讓它總結核心功能點
3. 讓它識別技術風險
4. 讓它生成開發任務列表
```

#### 場景 2：即時寫代碼

**推薦**：Cursor / GitHub Copilot

```
場景：邊寫代碼邊需要 AI 提供建議

做法：
1. 寫註釋描述意圖
2. AI 自動補全代碼
3. Tab 接受或修改
```

#### 場景 3：調試和修復

**推薦**：任意對話式 AI

```
場景：遇到報錯，需要理解原因和解決方案

做法：
1. 複製完整報錯信息
2. 附上相關代碼片段
3. 讓 AI 解釋並提供解決方案
```

#### 場景 4：代碼審查

**推薦**：Claude

```
場景：寫完一個功能，想讓 AI 幫忙檢查問題

做法：
1. 提供完整代碼
2. 說明預期功能
3. 讓 AI 從安全、性能、可讀性等角度審查
```

### 模型版本選擇

| 任務複雜度 | 推薦模型 | 原因 |
|------------|----------|------|
| 簡單問題 | GPT-3.5 / Claude Instant | 快速、便宜 |
| 中等複雜 | GPT-4 / Claude 3 Sonnet | 平衡成本和質量 |
| 高度複雜 | GPT-4 Turbo / Claude 3 Opus | 最強能力 |

### 實用建議

#### 1. 建立個人工作流

```
我的日常開發工作流：
- 寫代碼：Cursor（Tab 補全）
- 複雜問題：Claude（對話分析）
- 快速查詢：GPT-4（即問即答）
```

#### 2. 不要過度追求"最強"

```
❌ 每個問題都用最強的模型

✅ 簡單問題用快速模型，複雜問題才用強模型
```

#### 3. 多個工具配合使用

```
組合使用示例：
1. [Cursor] 寫代碼框架
2. [Claude] 審查代碼質量
3. [GPT-4] 優化特定函數
4. [Cursor] 繼續開發
```

### 成本控制

| 策略 | 說明 |
|------|------|
| **分級使用** | 簡單問題用免費/便宜模型 |
| **批量處理** | 多個小問題合併成一個請求 |
| **緩存結果** | 常見問題的答案記錄下來 |
| **善用免費額度** | 各平臺都有免費使用限制 |

### 保持更新

AI 領域發展很快，建議：

1. **關注新模型發佈**：Claude、GPT 會定期更新
2. **嘗試新工具**：可能有更適合你的工具出現
3. **加入社區**：學習他人的最佳實踐
